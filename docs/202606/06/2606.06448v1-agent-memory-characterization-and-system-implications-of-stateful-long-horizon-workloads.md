## Abstract
LLM agents are increasingly deployed on long-horizon tasks requiring sustained reasoning over extended interaction histories. Realizing this at scale requires agents to persistently store, retrieve, and update their own memory across sessions. A rich ecosystem of agent memory systems has emerged spanning flat retrieval, LLM-mediated extraction, consolidating fact stores, and agentic control flows. Yet, their system-level behavior remains uncharacterized. We present the first systems characterization of agent memory. First, we introduce a system-oriented taxonomy classifying agent memory systems along four axes. Second, we build a phase-aware profiling harness attributing cost to construction, retrieval, and generation. Third, we characterize ten representative systems across two benchmark suites, uncovering how design choices shift cost across the write and read paths. Finally, we derive 10 system recommendations covering construction scheduling, capability floors, amortization via query volume, freshness-latency tradeoffs, and fleet-scale management.

### 6. 论文的主要结论与发现

*   **构建成本主导生命周期**：对于LLM介导的记忆系统，构建能耗远超查询阶段总能耗，成为主要部署成本。系统间每正确回答的能耗差距可达47倍。
*   **构建是嵌入和预填充密集型负载**：构建过程反复读取长上下文（预填充为主），并输出紧凑的结构化记录，这与对延迟敏感的问答（QA）服务形成资源竞争。嵌入调用模式因范式而异，分为大批量离线索引和顺序单条写入。
*   **构建模型存在能力底限**：对于输出契约严格（如须输出JSON格式）的系统，使用过小的LLM会导致记忆库损坏，构建LLM缩放是一个有底线的成本杠杆。
*   **不存在“银弹”系统**：没有任何一个系统能在构建成本、查询延迟和准确性三个维度同时达到最优。它们各自占据“构建-服务-准确率”帕累托前沿的不同点，且在不同任务类别上表现各异。
*   **会话间调度引入新鲜度-延迟权衡**：在会话连续到达时，慢速构建系统若采用同步模式会阻塞用户，采用异步模式则会因写入未提交而返回过时记忆。
*   **成本增长斜率比初始足迹更重要**：每用户存储和Token成本的增长模式差异巨大。智能体控制系统（Paradigm IV）的构建Token成本随记忆增长呈**超线性**增长，而检索延迟则相对平稳。多视图表示加剧存储膨胀，合并式设计则能抑制。
*   **检索延迟的两极分化**：查询的pre-answer延迟跨度达两个数量级。延迟由算法边界（固定步骤管道）或LLM边界（迭代决策）决定，后者需要外部迭代上限来控制尾部延迟。

### 7. 优点

*   **首创性工作**：这是首个针对智能体记忆系统进行系统级表征的研究，填补了该领域从准确性导向到系统部署导向的认知空白。
*   **方法论扎实**：提出的**四维分类法**为理解和预测新系统的行为提供了理论透镜；开发的**阶段感知剖析工具**为量化分析提供了标准化的方法论和可开源的资产。
*   **实验设计全面、量化**：评估维度极为全面，将抽象的设计选择与具体的GPU微架构行为（如Tensor Core利用率、内存带宽）关联起来，结论均由严格、可复现的数据支持。
*   **极强的实践指导价值**：最终提炼的**10条系统建议**直指工程部署中的关键决策点（如算力选型、资源隔离、调度策略、成本管控），对架构师和运维人员有极高的参考意义。

### 8. 不足与局限

*   **评估环境单一**：所有实验均在单节点、单GPU上完成，未涉及未来更复杂的**多节点、多智能体分布式记忆**场景及其带来的一致性问题。
*   **缺失的记忆维护机制**：所有被测系统默认不对记忆进行**主动剪枝或遗忘**，导致存储和成本单调增长。研究未评估引入维护策略后的系统行为变化。
*   **模态覆盖有限**：工作聚焦于**纯文本记忆**，未涉及多模态（图像、音频等）记忆所面临的构建耦合、存储和检索挑战。
*   **基准与通用性**：尽管使用了主流基准，但对于高度定制化、面向对话的某些系统（如Mem0, SimpleMem）不得不进行提示词层面的适配，这可能在一定程度上影响了其在通用任务上的原生表现，引入了一定的偏差风险。
*   **硬件与模型局限性**：硬件表征限于NVIDIA H100 GPU和Qwen3系列模型，结论推广到其他硬件架构或更强大/定制化模型时的普适性有待验证。

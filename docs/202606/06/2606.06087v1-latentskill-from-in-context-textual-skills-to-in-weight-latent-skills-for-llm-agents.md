## 摘要
智能体系统越来越依赖文本技能来编码可复用的任务流程，但在每一步将这些技能注入提示会带来大量上下文开销，并以明文形式暴露技能内容。我们提出LatentSkill框架，通过预训练的超网络将文本技能转化为即插即用的LoRA适配器。LatentSkill将技能知识存储在权重空间而非上下文空间，从而消除了每步的技能令牌，同时保留了模块化加载、扩展和组合能力。在ALFWorld和Search-QA上，LatentSkill不仅优于相应的上下文技能基线，还大幅减少了预填充令牌：在ALFWorld的已见和未见划分上，成功分别提升21.4和13.4个百分点，预填充令牌减少64.1%；在Search-QA上，完全匹配提升3.0个百分点，技能令牌开销降低72.2%。进一步分析表明，生成的技能LoRA形成结构化的语义几何，可通过LoRA缩放系数精确控制，并在技能组件对齐时通过参数空间算术进行组合。这些发现表明，权重空间技能为扩展LLM智能体提供了一种高效、模块化且暴露程度较低的基底。

## Abstract
Agent systems increasingly use textual skills to encode reusable task procedures, but injecting these skills into the prompt at every step incurs substantial context overhead and exposes skill content as plaintext. We present LatentSkill, a framework that converts textual skills into plug-and-play LoRA adapters through a pretrained hypernetwork. LatentSkill stores skill knowledge in weight space rather than context space, removing per-step skill tokens while preserving modular loading, scaling, and composition. On ALFWorld and Search-QA, LatentSkill outperforms the corresponding in-context skill baseline while using substantially fewer prefill tokens: it improves ALFWorld success by 21.4 and 13.4 points on the seen and unseen splits with 64.1% fewer prefill tokens, and improves Search-QA exact match by 3.0 points with 72.2% lower skill-token overhead. Further analysis shows that generated skill LoRAs form a structured semantic geometry, can be precisely controlled via the LoRA scaling coefficient, and can be composed through parameter-space arithmetic when skill components are aligned. These findings suggest that weight-space skills provide an efficient, modular, and less exposed substrate for extending LLM agents.

-   **核心问题**：当前LLM智能体常将“技能”定义为可复用的文本流程，并在每一步决策时将其注入提示词中。这种做法面临一个三难困境：**如何在避免反复注入技能文本（降低上下文开销）的同时，既保持技能的模块化（可单独加载、卸载、组合），又避免将技能知识不可逆地融合到基础模型中。**
-   **研究动机**：
    -   **效率瓶颈**：反复注入长文本技能消耗大量上下文窗口，增加推理（尤其是预填充）成本。
    -   **安全风险**：以明文形式存储和传输技能文本，存在泄露专有流程和被恶意提示攻击（如劫持、提取）的风险。
    -   **灵活性缺失**：参数化替代方案（如微调）虽然避免了推理时的上下文开销，但将技能与模型参数“熔合”，破坏了模块化的灵活性。
-   **整体含义**：本文提出将技能知识从**上下文空间（文本）转移到权重空间（模型参数）**，作为一种新的技能表征范式，旨在同时实现**零上下文开销、高模块化、可控性和安全性**。

### 方法论

-   **核心思想**：通过一个**预训练的超网络（Skill Compiler）**，将一段文本技能描述直接映射为一组低秩适配器（LoRA）的权重增量。
-   **关键技术细节与流程**：
    1.  **超网络架构（Skill Compiler）**：`Gϕ` 被实现为一个基于Transformer的网络，它读取文本技能 `s` 作为输入，在单次前向传播中为冻结的主干LLM的指定模块生成LoRA权重增量 `Δs`。主干LLM（Qwen3-8B）冻结不参与训练。
    2.  **技能注入与推理**：推理时，根据任务选择对应的技能，其生成的LoRA适配器被挂载到主干LLM上（公式：`pθ,ϕ (yt | ht, s ) = pθ⊕α∆s (yt | ht)`）。技能文本不出现在提示词中，其影响强度由缩放系数α控制。
    3.  **两阶段训练**：
        -   **技能文档预训练**：在约17.1万份来自GitHub的技能文档上，通过“技能重构”和“技能补全”两种任务对超网络进行预训练。目标是让超网络学会将各类文本知识编码进LoRA权重中，使适配后的模型能重现或补全原始技能文档。
        -   **轨迹监督微调**：使用教师智能体为特定技能生成的成功交互轨迹 `τi` 对超网络进行微调。对于一个技能 `si`，超网络只生成一次LoRA适配器，该适配器被用于整个多步交互轨迹的所有决策步骤。这鼓励生成的适配器捕捉稳定、轨迹一致的策略信息，而非每步的临时适应。
    4.  **推理时技能组合**：对多个选中的技能，将其分别编译好的LoRA适配器在权重空间进行**加权求和**（`∆K = Σ αk * C[k]`）。对于有共享组件的技能，文章提倡**组件级组合**，即将一个技能分解为多个语义组件（如通用行为、防错、任务特定组件），对其分别编译LoRA，最后只保留一份共享组件与各个任务特有组件进行组合。

## 摘要
基于LLM的智能体越来越多地处理具有相互依赖决策的长时程任务，其中每个动作都会重塑未来约束，中间错误可能会级联。现有的RAG和智能体记忆系统按语义相似性组织历史记录，在决策时检索内容相关条目。我们认为，这种设计与执行状态依赖不匹配：它割裂了决策轨迹，混合了有效和错误迹，妨碍了连贯的状态重建和错误隔离。我们提出MAGE（Memory as Agent-Guided Exploration，记忆作为智能体引导的探索），一种主动的执行状态管理器，将交互存储在一个分层状态树中。智能体从活跃的根到当前路径中得出其状态，结合子目标摘要、近期迹和先前分支的提示。四个相互耦合的操作维护树：Grow记录新迹，Compress总结已完成的子目标，Maintain验证摘要，Revise恢复目标边界并在新分支上继续。这种设计限制了上下文增长，同时保持状态完整性并将有缺陷的片段与活跃路径隔离。在MemoryArena上的实验表明，MAGE相比基线将平均任务成功率提高了7.8-20.4个百分点，同时将令牌消耗减少了55.1%。

## Abstract
LLM-based agents increasingly tackle long-horizon tasks with interdependent decisions, where each action reshapes future constraints and intermediate errors can cascade. Existing RAG and agent memory systems organize histories by semantic similarity, retrieving content-relevant entries at decision time. We argue that this design mismatches execution-state dependencies: it fragments decision trajectories and mixes valid and erroneous traces, hindering coherent state reconstruction and error isolation. We propose MAGE (Memory as Agent-Guided Exploration), an active execution-state manager that stores interactions in a hierarchical state tree. The agent derives its state from the active root-to-current path, combining subgoal summaries, recent traces, and hints from prior branches. Four coupled operations maintain the tree: Grow records new traces, Compress summarizes completed subgoals, Maintain validates summaries, and Revise restores a target boundary and resumes on a new branch. This design bounds context growth while preserving state integrity and isolating flawed segments from the active path. Experiments on MemoryArena show that MAGE improves the average task success rate by 7.8--20.4 pp over baselines, while reducing token consumption by 55.1%.

### 2. 论文提出的方法论

MAGE (Memory as Agent-Guided Exploration) 框架的核心思想是模拟认知科学中人类处理复杂序列任务的机制（如分块、错误监控和选择性修正），将记忆建模为一个可被智能体主动操作的对象。

*   **核心数据结构：两层层次化状态树**
    *   **底部层 (Raw Nodes)**：按执行顺序记录每一步的“动作-观察”对，保留了精细的执行细节。
    *   **顶部层 (Summary Nodes)**：当一个子目标完成时，将对应的底部层连续片段压缩成一个总结节点，以释放上下文空间。
    *   **状态表示**：智能体的当前执行状态 `S` 并非来自检索的离散条目，而是直接由树中从根节点到当前节点的活跃路径构建，具体包含三部分：
        *   **压缩状态 (C)**：路径上所有顶部层总结节点的内容。
        *   **原始状态 (R)**：自上一次压缩以来，底部层新累积的原始动作-观察节点。
        *   **执行提示 (H)**：来自当前节点兄弟分支（先前探索过的错误或不同路径）的信息和诊断反馈。

*   **核心算法流程：四个闭环状态转换操作**
    这四个操作相互耦合，构成了一个完整的执行状态管理闭环。
    1.  **Grow (增长)**：智能体每执行一个动作并收到观察结果后，`Grow`在底部层的当前节点下添加一个新的子节点，记录该步内容。如果后续动作与先前探索过的子节点内容一致，则直接合并，避免重复。
    2.  **Compress (压缩)**：当智能体标记一个子目标完成时触发。`Compress`将底部层中从上一个压缩边界到当前点的节点片段追溯出来，生成一个顶部层的总结节点。此举在不中断活跃轨迹的前提下，以子目标为边界压缩上下文，既保留了决策的结构，又限制了上下文长度的增长。
    3.  **Maintain (维护)**：在`Compress`之后立即触发，作为一个边界级的错误监控器。它会调用LLM验证刚生成的总结节点及其对应的原始轨迹是否满足任务指令，防止错误或不完整的总结写入可信记忆。如果验证失败，会记录诊断反馈。
    4.  **Revise (修正)**：当`Maintain`检测到错误或智能体主动发现错误时触发。它根据反馈，将树中活跃的指针回退到指定的历史边界点，并从一个新的分支重新开始执行。这样可以隔离掉错误的分支，保留此前有效的执行进度。

## 摘要
长时程对话代理需要通过与不断演进的事件、任务和目标进行交互来与用户沟通。这类历史记录具有自然的时间特性，但许多现有记忆系统主要按主题相似性来组织信息，可能会忽略事件发生的顺序。我们引入了线段树记忆（SegTreeMem），这是一种将对话历史表示为基于话语的时间有序线段树的记忆架构。SegTreeMem 通过在线最右前沿更新规则逐步插入新话语，在保持时间顺序的同时形成层次化的记忆片段。在检索时，SegTreeMem 通过树结构传播相关性分数，将局部语义匹配与层次化时间上下文相结合。在三个长时程记忆基准测试和两种大型语言模型骨干上，SegTreeMem 相较于平面检索、图结构记忆和树结构记忆基线提高了答案质量。额外的时间顺序排列分析表明，性能提升依赖于记忆构建过程中保持时间顺序，这支持了时间顺序是代理记忆关键结构的论点。

## Abstract
Long-horizon conversational agents need to interact with users through evolving events, tasks, and goals. Such histories are naturally temporal, yet many existing memory systems organize information primarily by topical similarity and may ignore the order in which events occur. We introduce Segment Tree Memory, or SegTreeMem, a memory architecture that represents conversation history as a temporally ordered Segment Tree over utterances. SegTreeMem incrementally inserts new utterances through an online rightmost-frontier update rule, preserving chronological order while forming hierarchical memory segments. For retrieval, SegTreeMem propagates relevance scores through the tree to combine local semantic matching with hierarchical temporal context. Across three long-horizon memory benchmarks and two LLM backbones, SegTreeMem improves answer quality over flat retrieval, graph-structured memory, and tree-structured memory baselines. Additional temporal-order permutation analysis shows that the performance gain depends on preserving temporal order during memory construction, supporting the claim that temporal order is a key structure for agentic memory.

### 2. 论文提出的方法论

*   **核心思想**：提出**线段树记忆 (SegTreeMem)**，将对话历史建模为一棵**线段树 (Segment Tree)**。树的每个节点代表时间上**连续的一段对话区间**，叶子节点代表单个话语，内部节点则是对其子节点所覆盖的连续话语片段的摘要。
*   **关键技术细节**：
    *   **在线记忆构建 (Online Construction)**：
        *   维护一个“**最右前沿 (Rightmost Frontier)**”，即当前树中所有区间以最新话语结尾的节点集合。
        *   当新话语到来时，通过一个**兼容性模型**（例如余弦相似度、单点LLM评分、批量LLM评分）计算新话语与前沿节点的兼容性，并选择一个最匹配的内部节点作为父节点。
        *   新话语作为一个新的子树（叶子节点+可能的一串内部节点）被附加到该父节点下方，并更新受影响的节点注释。
        *   此规则保证了每次更新只修改树的一小部分（前沿及祖先），实现了高效的在线增量更新，并且天然保持了话语的时间顺序。
*   **结构感知的检索 (Structure-Aware Retrieval)**：
    *   **初始化**：计算查询与树中每个节点（以其摘要代表）的局部语义相关性分数，并归一化为概率分布 `s_q(0)`。
    *   **分数传播 (Score Propagation)**：通过一个预定义的传播矩阵 `W_P`，将相关性分数沿树结构进行多跳传播，形成新的分布 `s_q(k) = W_P * s_q(k-1)`。这模拟了相关性在上下文中的扩散。
    *   **传播策略**：
        *   **自顶向下 (Top-down)**：分数从父节点向子节点传播，优先检索更具体、更细粒度的对话片段。
        *   **自底向上 (Bottom-up)**：分数从子节点向父节点传播，优先检索更宽泛、更宏观的时间上下文。
    *   **分数融合**：将初始的局部分数与多跳传播后的结构分数进行加权组合，得到一个最终的结构感知相关性向量 `s̃_q`，用于选取Top-K个最相关的节点作为证据。

*   **方法设计精巧**：巧妙地将经典的“线段树”数据结构应用于对话记忆，其“最右前沿”的更新机制同时满足了在线性、高效性和保序性，设计优雅。
*   **检索机制可解释性强**：将检索形式化为树上的相关性传播，使得检索过程（向上或向下）能与所需的时间上下文粒度直接对应，提高了可解释性。

### 8. 不足与局限

*   **传播策略固定**：检索时的传播策略和强度是预定义的，不能根据用户的不同查询自动调整（例如，何时需要细节，何时需要宏观状态）。论文已将此作为未来工作方向。
*   **纯文本模态限制**：当前系统仅处理文本对话。扩展到多模态观察、工具调用和行动轨迹等更丰富的代理记忆场景是重要挑战。
*   **极端情况的脆弱性**：
    *   在**话题极度跳跃**的对抗性输入下，线段树的更新算法可能会导致树深度变为O(T)，失去了高效性。
    *   对于**长距离话题重现**，由于不存在跨时间段的直接语义链接，短期传播可能无法将远距离的相关信息关联起来。
*   **摘要信息丢失**：内部节点摘要可能会遗漏一些关键的精确细节（如时间、人名），导致无法回答需要字面匹配的问题。这依赖于更精细的摘要提示或回退到叶子节点的证据。

## 摘要
代理人工智能的最新进展使得代理能够通过工具使用、推理和多步规划来完成复杂任务。然而，现有基准测试仅在单次会话中评估代理，忽略了代理为实现个性化用户目标所必须整合的过往行为、所陈述的偏好以及先前决策。我们引入了Momento，一个用于在多会话服务环境中完成持久性代理任务的基准，要求代理执行具有后果的、通过工具中介的行动，同时解决跨会话的时间依赖性并适应用户不断变化的目标。实验结果表明，当前代理的失败主要源于对用户状态的错误估计，将先前的会话历史视为当前情境的可靠代理，而非需要重新验证的过时信息，这突显了当前代理能力与现实的长期人机交互之间的巨大差距。

## Abstract
Recent advances in agentic AI have enabled agents to complete complex tasks through tool use, reasoning, and multi-step planning. Yet existing benchmarks evaluate agents within a single session, ignoring past actions, stated preferences, and prior decisions that agents must integrate to fulfill personalized user goals. We introduce Momento, a benchmark for persistent agentic task completion in multi-session service environments, requiring agents to take consequential, tool-mediated actions while resolving temporal dependencies and evolving user goals across sessions. Experimental results reveal that current agents fail primarily through misestimation of user state, treating prior session history as a reliable proxy for current context rather than stale information requiring re-validation, highlighting a substantial gap between current agent capabilities and realistic long-horizon human-agent interaction.

## 2. 论文提出的方法论
*   **核心思想**：构建一个多会话代理交互框架，包含用户模拟器（User Agent）和助理代理（Assistant Agent），二者在多个时间分离的会话中对话。助理代理必须检索和更新记忆，在工具使用和决策中解决跨会话的时间依赖和用户目标演变。
*   **关键组件与流程**：
    *   **用户代理 （$\pi_u$）**：根据预设的用户策略 $\Phi_u$（包含目标、偏好、行为约束）和共享的代理记忆 $M$ 中的对话历史 $d_{<i}$，生成多模态的用户回复 $u_i$。
    *   **助理代理 （$\pi_a$）**：执行三大功能：
        1.  **记忆检索**：使用函数 $\Theta$ 和语义检索模型，从多会话摘要集 $s$ 中选出与当前用户输入 $u_i$ 最相关且满足时间约束的 top-k 会话 $\hat{s}$。
        2.  **工具调用**：基于策略 $\Phi_a$、检索到的上下文 $\hat{s}$、用户输入 $u_i$ 和历史 $d_{<i}$ 生成工具调用 $c_i$，并在环境中执行，获得输出 $o_i$。
        3.  **响应生成**：综合以上信息，生成最终助理回复 $a_i$。
    *   **代理记忆 （Agentic Memory）**：在 128k token 的上下文限制内运作，采用混合机制：
        *   **长期记忆**：通过 LLM 根据语义、时间和结构约束生成 SQL 查询来检索先前的会话，结合了语义匹配与显式时间推理。
        *   **短期记忆**：对会话历史进行压缩，保留最近 80% 的上下文逐字存储，旧的 20% 则压缩为摘要。
*   **基准构建**：包含 161 个人工标注的任务实例，覆盖个人助理、规划、日程安排、推荐等场景，横跨多个带记忆依赖的会话。通过人类角色扮演生成包含偏好变化、中断、隐含指代等复杂现象的先前会话历史。

## 6. 论文的主要结论与发现
*   **核心发现**：当前代理的主要失败原因是 **“对用户状态的错误估计”**，即模型倾向于将先前的会话历史直接当作当前情境的可靠依据，而不是将其视为可能需要重新验证的过时信息。
*   **失败模式**：**工具召回错误（`TOOL RECALL`）是最主要的失败类型**，尤其是遗漏验证和检索类工具（如“查询会员权益”），导致代理基于过时假设做出决策，即使最终写操作正确，中间步骤的错误也可能传播。
*   **模型表现**：**Gemma 4-31B-IT** 综合表现最强（`PASS@3` 和 `PASS^3` 最高），但所有模型的 `PASS@3` 和 `PASS^3` 之间存在巨大差距，表明多步任务执行的普遍不稳定性。

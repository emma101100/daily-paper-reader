## 摘要
智能体技能是使大语言模型智能体能够执行工作流程、验证约束条件并从失败中恢复的程序化工件。现有的自我进化方法利用累积的轨迹来精炼技能。然而，它们在冷启动环境中表现不佳，此时仅有一个初始的不完美技能可用。因此，技能构建通常依赖于专家编写或一次性大语言模型生成。专家编写的技能成本高昂，且可能与智能体实际执行任务的方式不一致；而一次性生成的技能可能在语法上形式良好，但在行为上较为薄弱。为弥补这一差距，我们提出了SkillRevise，一个基于执行的框架，旨在迭代式地精炼这些初始技能。SkillRevise从执行证据中诊断技能缺陷，从通用记忆中检索相关的修复原则，并应用执行锚定的编辑。通过重新执行候选技能并衡量经验效用，它系统地保留最优的技能版本。在三个基准测试和五个大语言模型上的评估表明，SkillRevise显著优于一次性基线，将基础智能体在SkillsBench上的成功率从36.05%提升至61.63%。此外，修订后的技能展现出强大的跨模型迁移能力，能够捕捉超越模型特定产物的通用程序化知识。

## Abstract
Agent skills are procedural artifacts that enable LLM agents to execute workflows, verify constraints, and recover from failures. Existing self-evolving methods refine skills using accumulated trajectories. However, they struggle in cold-start settings, where only an initial, imperfect skill is available. Consequently, skill construction defaults to expert authoring or one-shot LLM generation. Expert-authored skills are costly and may not align with how LLM agents actually execute tasks, while one-shot generated skills can be syntactically well formed yet behaviorally weak. To bridge this gap, we propose SkillRevise, an execution-grounded framework designed to iteratively refine these initial skills. SkillRevise diagnoses skill defects from execution evidence, retrieves relevant repair principles from a general memory, and applies execution-anchored edits. By re-executing candidates and measuring empirical utility, it systematically retains the optimal skill version. Evaluated across three benchmarks and five LLMs, SkillRevise substantially outperforms one-shot baselines, improving the base agent's success rate on SkillsBench from 36.05% to 61.63%. Furthermore, the revised skills exhibit strong cross-model transferability, capturing generalized procedural knowledge over model-specific artifacts.

## 1. 论文的核心问题与整体含义

大语言模型智能体在复杂、可验证环境中完成任务时，不仅需要单次问答，还需要可复用的**程序化技能**，以组织多步工作流、约束验证与失败恢复。然而，技能获取面临冷启动困境：

- **自我进化方法**需要大量积累的交互轨迹，初始阶段无法奏效。
- **专家编写技能**成本高，且可能与智能体的实际执行方式脱节。
- **一次性大语言模型生成技能**虽语法正确，但行为上往往薄弱、可靠度低。

因此，如何**在仅有初始不完美技能的条件下，通过执行反馈将其迭代精炼为高性能技能**，成为亟需解决的问题。论文提出 `SkillRevise` 框架，旨在填补这一空白，让技能构建从“静态生成”走向“基于执行的持续修改”。

## 2. 论文提出的方法论

`SkillRevise` 的核心思想是：**将执行证据与通用修复知识结合，以有界多轮修订的方式优化技能文本**。整个框架由以下关键组件构成：

- **诊断模块（Diagnosis）**
  从当前技能的运行结果中提取**验证规范**（Vi）、**失败归因**（Ai）和**保留约束**（Ki）。它明确指出什么需要修复、基于什么证据、以及哪些已通过的行为不能被破坏。

- **原则记忆（Principle Memory）**
  存储**可复用的修复原则**，每条原则包含触发条件、缺陷类型、修复规则、期望的智能体动作、验证方法以及禁止迁移的场景。初始化时含有 7 条原则（如“将输出暴露给验证器”“行动前先检查”等）。

- **修订算子（Revision Operator）**
  将当前技能 Si、诊断 Di 和绑定的原则集 Pi 映射为修订后的技能 ${\hat{S}_{i+1}}$ 与修订轨迹 zi。编辑强调**执行锚定**，即明确文本修改引起的智能体行为变化（如检查某个文件、验证格式等）。

- **有界修订史诗（Bounded Revision Episode）**
  在给定的修订预算 B（默认为 3 轮）内循环：
  1. 执行当前技能，获得轨迹与验证反馈。
  2. 进行诊断，若存在技能级缺陷，则从记忆检索并绑定适用原则。
  3. 调用修订算子生成候选技能。
  4. 重新执行候选技能，并依据**效用门控**选择最优版本（而非最后生成的版本）。
  效用函数可综合成功率、效率、迁移性和干扰成本，且对成功条件的效率增益进行严格把关。

整个流程形成闭环，并支持事后从成功经验中吸收新原则，以扩展原则记忆。

## 6. 论文的主要结论与发现

- **性能显著提升**：在 SkillsBench 上，`SkillRevise` 将 GPT‑5.5 的成功率从无技能的 36.05% 和一次性技能的 39.53% 提升至 **61.63%**（仅 3 轮修订），在其他基准和执行器上也普遍观察到正增益。
- **冷启动有效**：仅需初始技能即可启动修订，无需大量历史轨迹。
- **跨模型迁移性**：修订后的技能不仅适用于生成模型，在其他执行器（如 Opus‑4.7、DeepSeek‑V4 Pro）上也能带来性能提升，表明其捕获了**通用的程序性知识**。
- **诊断最为关键**：消融实验表明，移除诊断组件后成功率从 53/86 骤降至 28/86，证明从执行证据到技能缺陷的映射是框架的核心。
- **效用门控防止退化**：即便修订预算增加，多数任务最终保留的是初始或早期版本技能，说明选择机制有效避免了后期修订引入的性能回退。
- **收益集中在前几轮**：修订轮次增加带来的边际收益递减，框架具有合理的成本效益比。

## 7. 优点

- **创新定位**：明确聚焦冷启动技能修订，区别于需要大量轨迹的自进化或单纯的生成方法。
- **架构解耦**：将任务特定的**诊断**与跨任务的**修复原则**分离，有利于知识复用和可解释性。
- **执行锚定与保留机制**：要求修订时必须关联具体执行动作，且通过保留账本保护已通过检查，大幅降低修订引入新错误的风险。
- **效用导向选择**：基于实测效用而非生成顺序选择技能，更符合实际部署需求。
- **跨模型鲁棒性**：验证了技能可作为独立于生成模型的程序化知识存在。
- **实验严谨**：多基准、多执行器、多角度消融，并提供了详细的成败案例分析。

## 8. 不足与局限

- **验证器依赖**：修订质量完全取决于验证反馈的密度和准确性，稀疏或片面的测试可能导致过拟合或误导性修订。
- **无技能回退机制**：当初始技能本身有害时，修订可能无法恢复到无技能执行的效果（一些任务无技能表现更佳，但框架仍强制使用技能）。
- **固定预算与成本**：修订需要额外执行与模型调用，虽有效率分析，但未提供自适应停止策略；某些情形下修订后技能虽成功但执行成本更高。
- **泛化边界**：实验集中在可自动化验证的基准，尚未在长期运行、安全攸关、对抗环境或自定成功标准的场景下测试。
- **环境与模型限制**：部分软件工程任务因仓库特定调试或超时导致提升较小，表明框架对高度复杂、长程实现类任务的助力有限。
- **静态原则记忆**：初始 7 条原则虽够用，但更广泛的缺陷库可能需要持续在线吸收，而当前的吸收策略相对保守。

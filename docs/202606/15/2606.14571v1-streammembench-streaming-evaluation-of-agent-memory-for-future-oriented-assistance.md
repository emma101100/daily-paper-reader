## 摘要
个人智能体记忆的一个核心作用是将存储的信息和先前的交互转化为面向未来的辅助。在日常使用中，有用的线索来自智能体观察到的事物以及用户与智能体交互的方式，智能体必须将这些线索从当前请求延续到类似的未来任务中。现有的记忆基准通常孤立地测试对话回忆或任务改进，而忽略了从流式观测到后续辅助的整个过程，基本上未被测试。我们引入了StreamMemBench，这是一个流式基准，围绕EgoLife自我中心流中的每个证据锚点构建了一个两步任务序列。初始任务测试证据的使用，而后续任务测试反馈和交互经验是否被复用。四个指标分别诊断证据回忆、初始证据使用、反馈整合和后续复用。在两个骨干网络上对八个记忆系统进行的实验表明，当前系统往往无法使用观测到的证据，也无法将反馈转化为可靠的后续行为，即使证据已被存储或反馈在局部被整合。StreamMemBench在https://github.com/landian60/StreamMemBench上公开可用。

## Abstract
A central role of personal-agent memory is to turn stored information and prior interactions into future-oriented assistance. In daily use, useful cues come from what the agent observes and how the user interacts with the agent, and the agent must carry them forward from the current request to similar future tasks. Existing memory benchmarks usually test dialogue recall or task improvement in isolation, leaving the trajectory from streaming observations to later assistance largely untested. We introduce StreamMemBench, a streaming benchmark that constructs a two-step task sequence around each evidence anchor from EgoLife egocentric streams. The initial task tests evidence use, while the follow-up task tests whether feedback and interaction experience are reused. Four metrics diagnose evidence recall, initial evidence use, feedback incorporation, and follow-up reuse. Experiments with eight memory systems across two backbones show that current systems often fail to use observed evidence or turn feedback into reliable follow-up behavior, even when evidence is stored or feedback is incorporated locally. StreamMemBench is publicly available at https://github.com/landian60/StreamMemBench.

## 1. 论文的核心问题与整体含义

- **研究背景**：个人智能体（AI Assistant）需要利用长期记忆将过去观察到的用户信息（如偏好、习惯、关系）和交互经验转化为对未来任务的主动辅助。现有的记忆基准（benchmarks）大多只孤立地测试对话内容回忆或任务性能提升，而忽视了在连续流式场景下，智能体必须自主地从自我中心（egocentric）观察流中识别关键信息，用于当前任务，并将交互反馈经验沉淀后重用到后续类似任务这一完整闭环。
- **核心问题**：当前记忆系统即使能存储信息，也往往无法在需要时有效利用观察到的证据，或无法将用户反馈转化为可靠的后续行为。因此，需要一种能够追踪“从流式观测到证据使用，再到反馈整合与后续复用”全链路的评估基准。
- **整体含义**：论文提出 StreamMemBench，一个基于自我中心生活流的流式基准，旨在评估个人智能体记忆是否真正支持面向未来的辅助，填补了现有基准在证据使用和交互经验复用两个关键维度上的空白。

## 2. 方法论

### 核心思想
将每个评估锚点（evidence anchor）构建成一个两步任务序列，模拟用户在日常流式交互中先请求帮助、提供反馈，随后再请求一个相关任务的过程，从而检验记忆系统能否将观测信息和交互经验跨任务复用。

### 关键技术流程

1. **基准构建（Construction Stage）**
   - **数据源**：EgoLife 自我中心视频流，按5分钟分段，包含叙述、对话转录等观测。
   - **证据锚点提取**：使用锚点构建智能体 A_anchor 对每个分段提取可转移的用户特定知识单元（AKU，即证据），并生成两个查询（T1 初始任务，T2 后续任务）。正确答案必须依赖该证据，且查询文本中不泄露证据信息。
   - **质量审查**：使用审查智能体 A_review 检查证据是否有源观测支撑、归属是否正确、查询是否自然、是否依赖同一证据、场景是否不同等。只有通过审查的候选锚点才被保留。

2. **评估轨迹与度量（Evaluation Stage）**
   - **流式摄入与记忆形成**：记忆系统按时间顺序接收分段流，在每个分段后提取记忆增量 ∆M。
   - **任务序列**：每个锚点执行一条轨迹 τ：T1 → R1（初始响应） → F（模拟用户反馈，确认或纠正） → RF（修正后响应） → Φ（将整个交互回合提交记忆） → T2 → R2（后续响应）。
   - **四个诊断指标**：
     * **保真度（Fidelity）**：在任务序列前，检查目标证据是否出现在系统暴露的记忆增量中（∆M 中与锚点对比），通过语义匹配判断。
     * **初始证据使用（IEU）**：根据 A_user 判断 R1 是否正确使用了证据。
     * **反馈整合（FI）**：仅对纠正案例（R1 失败）衡量 RF 是否正确整合了反馈中提供的证据。
     * **后续复用（FUR）**：判断 R2 是否复用了证据或修正后的经验。
   - **外部评判**：A_user 和 A_eval 均使用 DeepSeek-V4-Pro，打分采用多数投票（三次判断）。

### 公式与算法说明
- 锚点提取：A_anchor(zi) = {(a_k, T1, T2)}，其中 a_k 包含证据、来源等。
- 审查约束：Leak(T_i, a_k) = 0（查询不泄露证据），Need(T_i, a_k) = 1（答案必须依赖该证据），Natural(T_i) = 1（查询自然）。
- 指标计算：Fid_k = 1[A_eval(∆M_i, a_k) = pass]，IEU_k = 1[A_user(R1, T1, a_k) = pass]，FI_k = 1[A_user(RF, F, a_k) = pass]，FUR_k = 1[A_user(R2, T2, a_k) = pass]。FI 仅在反馈类型为纠正时计算。

## 3. 实验设计

- **数据集与场景**：基于 EgoLife 数据集，包含7天6位参与者的连续自我中心记录。拆分为 3,347 个5分钟分段，每个分段约200条时间戳标记的叙述或对话。从中提取出 8,107 个证据锚点，生成 16,214 个任务查询，涵盖任务分配、活动规划、社交沟通等多种自然辅助场景。
- **基准测试被评估系统**：共评估两大类8种系统，包括两种检索基线（RAG_raw 存储原始分段并检索；RAG_ext 先提取结构化事实再检索）和六种主动记忆管理系统：Mem0（事实覆盖）、EverMemOS（分层存储）、A-Mem（块级链接）、MemOS 和 MemoryOS（多级整合）、MemSkill（程序性技能记忆）。
- **骨干网络**：每个系统均在 DeepSeek-V4-Flash 和 Gemini-3-Flash 上运行。
- **评估协议**：系统按时间顺序接收分段流，Fidelity 通过直接检查记忆增量判定，任务评分由 A_user 给出。检索均使用 bge-m3 编码器，top-k=10。所有评分都经过多数投票。

- **充分性**：实验覆盖了当前具有代表性的记忆范式（检索、事实存储、分层整合、程序记忆），数据规模较大（8,107个锚点），评估维度全面（从存储端到任务端），并且通过多数投票与人工审计（100条样本，查询合理性90%，反馈合理性93.5%）保证了评判的客观性。对比公平，所有系统使用相同的评估协议和相同的骨干模型，结果可复现。

## 6. 主要结论与发现
1. **存储证据不等于可用证据**：许多系统具有高保真度（Fidelity）但初始证据使用（IEU）和后继复用（FUR）显著更低。例如，A-Mem 的 Fidelity 为 100%，但 IEU 仅 34.92%（DeepSeek），FUR 为 64.98%，说明证据虽被保留，未能有效应用于任务。
2. **反馈即时整合不等于长期复用**：部分系统（如 MemOS）反馈整合率（FI）较高（77.25%），但 FUR 极低（3.96%），表现为“改正了这次，下次依然忘记”。
3. **记忆机制决定失效模式**：生命周期故障分析显示，基于链接的 A-Mem 故障主要发生在初始使用；事实型系统 Mem0 存在形成失败；MemOS 以“纠正整合失败”为主。
4. **流深度影响能力**：随着流长度增加，某些系统（如 RAG_raw）的 IEU 下降，而 FUR 可能回升，表明检索准确性和经验复用性随时间漂移。
5. **效率与效果不相关**：存储空间小、延迟低的系统未必任务表现更好，不同系统在存储体积、延迟、token 成本上差异显著，需要多维评估。

## 7. 优点
- **全链路评估**：从流式观察到证据使用、反馈整合、后续复用的端到端追踪，提供了现有基准缺失的维度。
- **证据锚点设计**：每个锚点都与源数据中的具体证据绑定，防止模型“猜对”，强化了评估的真实性和可追溯性。
- **详细诊断**：四个指标独立诊断“是否记住”、“是否能用”、“是否改正”、“是否下次复用”，可精确定位系统短板。
- **真实场景还原**：利用真实自我中心生活流，模拟个人助理的日常交互，比合成对话更贴近实际应用。
- **系统多样性**：对比了检索式、事实式和整合式等多种记忆机制，结论具有普遍性。
- **严谨性**：使用专职审查和评判智能体，多数投票与人工审计保证标签质量。

## 8. 不足与局限
- **系统覆盖范围**：仅评估了8种记忆系统，未包含更多商业或前沿研究系统，未来可扩展。
- **数据来源单一**：仅基于 EgoLife 数据集（6位参与者），可能限制了场景和用户多样性，泛化能力待验证。
- **模拟反馈**：用户反馈由智能体模拟，尽管经过人工审计，但仍可能偏离真实用户的自然反馈。
- **隐私风险**：基准依赖于个人生活数据，可能鼓励系统存储敏感信息，论文虽提出应遵循同意、最小化、访问控制等原则，但未在基准中内置隐私保护机制。
- **仅黑盒评估**：未深入分析记忆内部结构，解释性主要依赖外部表现，难以揭示底层机制。
- **长短流限制**：目前流长度受限于数据集规模，更长时间跨度的记忆遗忘问题未充分考察。

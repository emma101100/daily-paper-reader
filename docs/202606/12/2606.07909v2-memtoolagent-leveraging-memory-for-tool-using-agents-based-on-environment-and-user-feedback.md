## 摘要
现代大型语言模型（LLM）代理能够使用外部工具帮助用户解决复杂任务。然而，对于需要从长期历史事件或先前代理-环境交互中学习的问题，LLM代理需要使用记忆机制来存储和检索经验。尽管已有针对对话代理的复杂记忆系统，但少有研究实证检验如何通过过去的用户-代理对话提升代理的工具使用能力。我们提出MemToolAgent，一个通过记忆管理改进工具使用的框架。我们的方法包含一个记忆提取模块，将过去经验处理为结构化记忆条目，以及一个检索模块，动态选择存储记忆条目的子集。这使得响应更个性化、更准确，与用户偏好和反馈保持一致，无需LLM微调。总之，本工作有三个主要贡献：（1）统一的记忆条目格式，无需LLM微调即可改进通用和个性化工具使用；（2）基于反思的记忆提取，利用环境和用户反馈将错误执行提炼为批评存储；（3）一个检索模块，根据记忆相似度分布选择使用多少过去经验。MemToolAgent在WorkBench、NESTFUL和PEToolBench基准上分别比强基线实现了29%、80%和17%的相对改进。

## Abstract
Modern large language model (LLM) agents can use external tools to help users solve complex tasks. However, for problems that require learning from long-term historical events or from previous agent-environment interactions, LLM agents are required to use memory mechanisms to store and retrieve experiences. While sophisticated memory systems exist for dialogue agents, few studies have empirically examined how to improve agents' tool-using capabilities through past user-agent conversations. We propose MemToolAgent, a framework that improves tool use through memory management. Our approach contains a memory extraction module that processes past experiences into structured memory entries, and a retrieval module that dynamically selects a subset of the stored memory entries. This enables more personalized and accurate responses aligned with user preferences and feedback without requiring LLM fine-tuning. In summary, this work has three main contributions: (1) a unified memory entry format that improves both general-purpose and personalized tool use without LLM fine-tuning, (2) a reflection-based memory extraction that uses environment and user feedback to distill wrong executions into critiques to store, and (3) a retrieval module that chooses how many past experiences to use based on the memory similarity distribution. MemToolAgent achieves 29%, 80%, and 17% relative improvements compared to strong baselines on the WorkBench, NESTFUL, and PEToolBench benchmarks, respectively.

### 1. 论文的核心问题与整体含义

现代大语言模型（LLM）代理可调用外部工具完成复杂任务，但在需要从长期历史事件或过往用户交互中学习的场景下，它们必须借助记忆机制存储并检索经验。尽管对话代理领域存在复杂的记忆系统，**如何利用历史用户-代理对话来提升代理的工具使用能力仍缺乏系统研究**。该问题的核心挑战在于：

- LLM 的固定上下文窗口难以维护长期对话中的连贯性与主题连续性；
- 注意力机制在处理长序列时失效，尤其在教育、医疗等关键场景中不可接受；
- 代理需要从用户偏好和过往错误中学习，才能实现既个性化又准确的工具调用。

本文提出 **MemToolAgent**，一个通过记忆管理来增强工具使用能力的框架，旨在同时解决**通用工具调用**（如电子邮件的正确转发流程）和**个性化工具编排**（如根据用户历史偏好选择 Jira 或 Zendesk 创建工单）两类任务，无需对 LLM 进行微调。

### 2. 方法论

MemToolAgent 由两个核心模块组成：**记忆检索模块**与**记忆提取模块**。

#### 2.1 统一记忆条目格式

每条记忆被表示为一个四元组 \(m = (q, a, f, r)\)，其中：

- \(q\)：用户查询；
- \(a = \{a_1, a_2, \dots, a_n\}\)：代理执行的一系列工具调用；
- \(f \in \{0, 1\}\)：二元用户反馈（1 表示成功，0 表示失败）；
- \(r\)：由记忆提取器生成的反思/批评（仅当 \(f = 0\) 时）。

该格式同时兼容成功经验与失败反思，支持通用与个性化两大类任务。

#### 2.2 动态 "n" Top-n 嵌入检索

为克服固定检索数量 \(k\) 的不足（某些查询缺乏相关记忆，另一些则拥有大量相似条目），论文提出基于相似度分布的动态检索算法，其步骤为：

1. 计算查询与所有记忆条目的余弦相似度；
2. 将相似度降序排列；
3. 使用滑动窗口法计算一阶导数近似（窗口半径 \(R\)），从而估计相似度曲线的变化；
4. 利用 `find_peaks` 函数检测导数曲线的峰值，这些峰值对应相似度的突然下降（即“类间边界”）；
5. 选择指定编号的峰值位置作为检索条目数 \(n\)。

该设计借鉴“knee/elbow”检测思想（Satopaa et al., 2011），能比 Adaptive-k 等方法更稳健地处理噪声相似度曲线与多级下降。

#### 2.3 基于反思的记忆提取模块

当执行失败（用户反馈为 0）时，记忆提取模块（一个 LLM，记为 \(E\)）会接收以下输入：

- 当前查询 \(q\)；
- 可用工具集 \(T\)；
- 代理的动作序列 \(a\)；
- 环境反馈（工具返回结果） \(e\)；
- 用户反馈 \(f\)；
- 从记忆库中检索到的语义相似条目 \(M'\)。

该模块输出一条反思/批评 \(r = E(q, T, a, e, f, M')\)，指出执行中的错误原因（如逻辑推断、参数幻觉、工具依赖违反等），并将其存储为记忆条目的组成部分。这样，后续检索可以直接获取“教训”，而不必让代理同时解析原始执行轨迹并完成任务，从而降低认知负荷。

在个性化场景中，记忆提取器用于抽象用户的偏好模式，如“用户偏好使用 API_X 下的工具”。

### 3. 实验设计

实验覆盖两类任务：

- **通用工具使用**：WorkBench、NESTFUL；
- **个性化工具编排**：PEToolBench。

#### 3.1 数据集/基准介绍

- **WorkBench**：以工作场所任务（发送邮件、安排会议等）为中心，涵盖 Analytics、Calendar、CRM、Email、Multi-domain 五个领域，包含 5 个数据库与 27 个工具。评价指标为准确度（代理动作结果与预期结果的一致性）。测试集规模：108（Analytics） / 99（Calendar） / 72（CRM） / 81（Email） / 189（Multi-domain）。记忆填充：每个模板取 1 个查询，确保对每个测试查询至少有一条直接相关的记忆。
- **NESTFUL**：评估 LLM 生成嵌套工具调用序列的能力，基于 MathQA 和 StarCoder2-Instruct。使用 500 个测试查询，另 500 个用于记忆填充。评价指标包括函数 F1、参数 F1、部分序列匹配度、完全序列匹配度及 Win Rate。
- **PEToolBench**：个性化工具编排基准，每个测试查询附带 80–100 条历史指令（含二元评分）。测试集选取其中 591 个查询（历史长度足够）。评价指标：工具准确度（调用正确工具）与参数准确度。

#### 3.2 对比方法

- **通用工具使用**：

### 6. 主要结论与发现

- MemToolAgent 在通用工具使用上大幅超越无记忆、扩展思考及 ReAct 基线。在 WorkBench 上的总体准确度从 57.56% 提升至 85.06%，Analytics 任务从 18.52% 升至 86.11%；在 NESTFUL 上，完全序列匹配度从 15.6% 提升至 30.4%，Win Rate 从 51.2% 升至 70.2%，表明**额外推理能力不能替代记忆提供的具体示例**。
- 在个性化工具编排上，MemToolAgent 的工具准确度达到 0.82，相对 A-MEM 与 Mem0 提升 17%，证明其能更有效地利用用户偏好记忆选择合适的工具。
- 动态 top-n 检索相比固定 top-k 普遍更优，且能适应不同查询所需的不同记忆数量。
- 记忆提取模块（反思/批评）对性能贡献显著：若移除该模块，工具准确度、完全序列匹配度等指标明显下降。反思能帮助代理从错误中学习，而原始执行轨迹作为记忆时效果欠佳。
- 冷启动问题可通过合成查询生成或改写查询得以缓解。使用改写或合成查询构建记忆仍能带来性能提升，尤其在 Multi-domain 与 Calendar 任务上。
- MemToolAgent 在增加输入 Token 的同时，有效控制了输出 Token 的膨胀，有利于实际部署的成本与延迟。

### 7. 优点（方法/实验亮点）

- **统一记忆格式**：四元组格式简洁、通用，天然支持正负样本，适应多类任务。
- **反思型记忆提取**：将失败的执行转化为结构化的批评，而非存储原始日志，显著降低代理的推理负担，提升了记忆的“教学”质量。
- **动态检索机制**：基于相似度曲线变化的峰值检测，能自适应地确定检索数量，相比固定 k 方法更灵活、更稳健。
- **无需微调**：整个框架完全基于上下文学习与外部记忆，不与模型参数绑定，易于移植至不同 LLM。
- **实验覆盖全面且消融细致**：从通用到个性化，从单个模块到整体系统，逐步拆解贡献，并探究不同记忆类型与冷启动策略，论证扎实。
- **成本可视化**：给出 Token 消耗对比，体现框架在实际应用中的效率考虑。

### 8. 不足与局限

- **冷启动问题**：虽然通过合成查询等方式缓解，但在大规模工具集（例如数百个工具）下，记忆覆盖仍极具挑战，合成查询的质量和复杂度也受限。
- **单轮任务限制**：所有基准均为单轮交互，未验证多轮对话中记忆的动态更新与检索。在多轮场景下，可能需要每轮都触发检索，且反思模块与检索策略尚需适配。
- **用户反馈过于理想化**：实验中采用二元正确/错误反馈，而现实用户可能提供模糊、延迟或缺失的反馈。多类型反馈（分数、文本）的利用未被深入讨论。
- **搜索与可解释性**：动态检索的超参数（R、prominence、峰值编号）需手工调节，缺乏自动化或自适应的参数选择机制。同样，记忆提取模块的批评质量能否保证尚需更系统的评估。
- **评估规模与统计可靠性**：主实验仅单次运行，未提供方差；PEToolBench 的部分消融基于 50 条子集，样本量偏小；对 NLP 领域常见的统计显著性检验缺失。
- **工具数量与领域泛化**：所用基准的工具数均不多于 40，现实应用中工具空间可能更广阔，且部分工具间存在细微重叠，记忆覆盖与相似度计算面临更大挑战。
- **伦理与安全方面讨论不深**：论文仅提及基准的公开性，未对记忆存储中可能出现的隐私、偏误或有害内容的处理进行深入探讨。

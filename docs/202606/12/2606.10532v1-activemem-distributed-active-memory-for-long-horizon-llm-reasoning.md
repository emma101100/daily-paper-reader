## 摘要
记忆对于使大型语言模型（LLM）智能体能够处理长时程推理任务至关重要。现有的记忆机制大多集中化，通常在单一模型上下文内组织检索到的信息和交互历史。这种设计带来了根本性的权衡：扩展推理轨迹可能引发上下文过载的风险，而激进的内容剪枝则可能导致不可逆的信息丢失。为了寻求更好的权衡，我们从人类认知系统中汲取灵感，尤其是前额叶皮层（执行控制）与海马体（记忆管理）之间的功能互补性，这表明这种权衡并非固有，而可能源于集中式的记忆组织。为此，我们提出了ActiveMem，一个将智能体记忆与核心推理过程解耦的异构框架。具体而言，高层规划器利用提炼出的语义要点执行推理，而一个轻量级的分布式记忆系统并行运作，在整个任务过程中主动积累和巩固这些要点。在BrowseComp-Plus和GAIA上的实验表明，ActiveMem在显著降低开销的同时达到了最先进的准确率，证明了分布式主动记忆对于长时程推理的有效性。

## Abstract
Memory is essential for enabling large language model (LLM) agents to handle long-horizon reasoning tasks. Existing memory mechanisms are largely centralized, typically organizing retrieved information and interaction history within a single model context. This design imposes a fundamental trade-off: scaling reasoning trajectories risks context overload, whereas aggressive content pruning may result in irreversible information loss. Seeking a better trade-off, we draw inspiration from human cognitive systems, especially the functional complementarity between the prefrontal cortex (executive control) and the hippocampus (memory management), suggesting that such a trade-off need not be inherent, but may instead stem from centralized memory organization. To this end, we propose ActiveMem, a heterogeneous framework that decouples agent memory from the core reasoning process. Specifically, a high-level Planner utilizes distilled semantic gists to execute reasoning, while a lightweight, distributed memory system operates in parallel to actively accumulate and consolidate these gists throughout the task. Experiments on BrowseComp-Plus and GAIA show that ActiveMem achieves state-of-the-art accuracy with significantly reduced overhead, demonstrating the effectiveness of distributed active memory for long-horizon reasoning.

- **认知科学启发与整体含义**：论文从人脑的前额叶皮层（执行控制）与海马体（记忆管理）的功能互补中汲取灵感，认为上述权衡并非系统固有的，而是由集中式记忆组织方式造成的。**ActiveMem 的整体含义**就是提出一个异构、解耦的分布式主动记忆框架，将记忆管理与高层推理过程分离，从根本上缓解长程推理中的记忆瓶颈。

### 2. 方法论（核心思想、技术细节与流程）
- **核心思想：解耦与分布式主动记忆**
    - 将单一、耦合的集中式记忆，替换为**高层规划器（Planner）** 与 **分布式记忆系统** 的异构协作架构。
    - Planner 只使用提炼出的**语义要点（Semantic Gists）** 进行推理，保持其上下文窗口紧凑清洁；而轻量级的分布式记忆系统在后台并行地主动积累、巩固和复用这些要点。
- **核心组件与关键技术细节**
    - **Planner（规划器）**：维持紧凑状态 \( s_t = (x, h_t, m_{t-1}) \)，其中 \( h_t \) 仅保留最近的 \( K \) 步交互历史（通过 `Trim` 操作截断），前序内容已以要点形式存入记忆分片。Planner 根据此状态生成检索查询集 \( Q_t \)。
    - **分布式记忆系统**，由三部分构成：
        1.  **Memory Shards（记忆分片）**：固定数量的逻辑分片 \( B_j \)，作为持久化键值存储。每个文档条目存储其提炼要点 \( g_c \) 及历史查询集 \( H_c \)。分片实现了信息的持久化和复用，避免重复处理。
        2.  **Memorizers（记忆器）**：轻量级（4B 参数）并行模块，模仿海马体的功能。它接收 Planner 的查询 \( q \) 和检索到的原始文档 \( c \)，提炼出**查询条件化的语义要点** \( g_c = \omega(c, q) \)。为提高蒸馏质量，Memorizer 经过监督微调（SFT），训练数据来自真实智能体轨迹，使用更大的教师模型产生要点。
        3.  **Operator（协调器）**：作为 Planner 与记忆系统的控制层，负责**路由**（将文档分配到稳定分片）、**记忆复用**（通过语义相似度判断 \( J(q, H_c) \) 决定是否直接返回已存要点）和**异步巩固**（将不同查询角度产生的要点合并为一个紧凑、不冗余的新要点）。
- **算法流程概要**
    1.  Planner 发出检索查询 \( Q_t \)。
    2.  检索器获取文档集 \( D_t \)，Operator 将其路由至对应记忆分片。
    3.  **并行**多 Memorizer 处理文档：若文档未见过，则蒸馏新要点；若见过且查询相似，则直接命中返回；若见过但查询全新，则蒸馏新要点并异步合并旧要点。
    4.  收集所有要点 \( G_t \) 作为观察 \( o_t \)，追加并截断 (Trim) 历史，更新 Planner 状态，进入下一步推理或提交答案。

### 3. 实验设计
- **主要数据集与基准**：
    - **BrowseComp-Plus**：主要基准，包含 150 个示例，按 Easy/Medium/Hard 难度划分，用于细粒度分析。
    - **GAIA**：通用智能体基准，使用其 WebSearch 验证子集（106 个示例），聚焦信息密集型检索。
- **对比方法（Baselines）**，分为两类，且为公平比较，所有集中式方法均使用与 ActiveMem 相同的 Qwen3.5-397B-A17B 作为 Planner：
    - **原生 ReAct 大模型**：Kimi-K2.5, Qwen3.5-397B-A17B, GLM-5.1, DeepSeek-V3.2。
    - **集中式记忆智能体**：Context-Folding, AgentFold, MemoBrain, MemAgent。
- **评价指标**：
    - **LasJ（LLM-as-a-Judge）**：使用 Qwen3.5-397B-A17B 评判答案正确性。
    - **PFLOPs**：推理总计算量（千万亿次浮点运算），模型无关的代价代理指标，考虑了模型规模和上下文长度。
    - **ACT（Accuracy-Cost Trade-off）**：平衡准确率与计算开销的综合指标。

- **资源与算力**：
    - **训练**：所有 SFT 训练在 **4 块 NVIDIA A100 (80GB) GPU** 上进行。训练时长未明确提及。
    - **推理**：Planner（397B 模型）通过 API 调用；记忆系统各模型（Memorizer-4B, Operator-4B）及检索模块均各部署在单块 **NVIDIA A100 (80GB) GPU** 上。

- **实验数量与充分性**：
    - **主体实验**：在两个主要数据集上对比了 9 个基线模型，生成主要结果表。
    - **消融与深入分析实验**，共计至少 **5 组**：
        1.  移除 Memory Shards 的影响（验证持久化复用的作用）。
        2.  对比三种 Memorizer 变体（指令微调版、普通推理版、SFT 版），验证 Memorizer 设计的关键性。
        3.  调整 Trim 窗口大小 \( K \) 的效果。
        4.  调整 Memorizer 参数规模（0.6B, 1.7B, 4B）带来的成本-准确率权衡。
        5.  移除异步 Memory Consolidation 的影响。
        6.  还进行了记忆命中率分析、长步骤推理成本分析和案例研究。
    - **公平性说明**：所有对比方法共享相同的 Planner 骨干模型、温度和步骤预算，超参遵循原论文；Memorizer 的训练/测试文档严格隔离，避免数据泄漏。实验设计系统、公正，且考虑了同类模型很难公平对比的参数规模差异（通过 PFLOPs 指标），实验量较为充分。

### 4. 主要结论与发现
- **性能最优且开销更低**：ActiveMem 在 BrowseComp-Plus 和 GAIA 两个基准上均取得了最高的 LasJ 准确率，同时 PFLOPs 计算量远低于原生 ReAct 大模型和大多数集中式记忆智能体。ACT 综合指标均为最优。
- **优势在复杂任务上更显著**：在 BrowseComp-Plus 的 Medium 和 Hard 难度子集上，ActiveMem 相较最佳基线分别提升了 11.9% 和 22.2% 的相对准确率，证明解耦设计能更好地支撑检索密集型推理。
- **有效的成本结构**：记忆系统（Memorizer）用 4B 参数处理了大部分 Token（占总 PFLOPs 的 68.9%），而核心 Planner 的计算量占比仅 28.3%，成功将高成本的 Token 处理任务卸载到低成本模型，实现了更优的 test-time scaling。
- **消融实验验证了各组件价值**：
    - **Memory Shards** 的持久化复用同时提高了效率和准确率。
    - **经过 SFT 的 Memorizer** 能产生更简洁、与任务对齐的要点，显著降低整体系统成本并提升准确率。
    - **异步记忆巩固** 能有效维持 Planner 上下文的紧凑和清洁，避免冗余信息干扰推理。

### 5. 优点（亮点）
- **创新性强**：首次从认知科学的前额叶-海马体协同出发，系统性地设计了一个**解耦的分布式主动记忆**范式，为长程推理记忆管理提供了全新视角。
- **架构设计精巧**：**Planner、Memorizer、Operator、Memory Shard** 四者分工明确，将蒸馏、存储、复用、巩固等记忆子任务剥离成并行、异构的子系统，将计算从大模型卸载到小模型，实现了低成本规模化。

### 6. 不足与局限
- **实际部署效率未测量**：虽然报告了 PFLOPs，但未测量实际运行时的**挂钟延迟**或**金钱成本**。而这些指标对服务架构、API 响应等高度敏感，需在统一部署环境下评测，作者将此留给未来工作。

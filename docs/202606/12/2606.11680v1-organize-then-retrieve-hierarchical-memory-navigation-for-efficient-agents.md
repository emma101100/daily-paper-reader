## 摘要
大语言模型（LLM）智能体因固有的无状态性难以处理长期任务，需将所有任务相关信息编码至不断增长的输入上下文中。这导致推理质量下降、推理成本增加和延迟升高，因此迫切需要高效的暂存记忆机制。然而，现有方法要么依赖有损压缩，要么基于相似性检索，往往无法捕捉多步智能体任务所需的时间结构和因果依赖。本文提出 HORMA，一种分层组织与检索记忆智能体，将经验组织成类似文件系统的分层结构，其中摘要实体链接至相应的原始轨迹，可在不丢失细节的前提下高效访问。HORMA 将暂存记忆分解为两个阶段：结构化记忆构建和基于导航的检索。构建模块通过区分由信息缺失导致的失败与由误导性或过载上下文导致的失败，迭代优化经验的构建方式。导航模块则通过轻量级智能体遍历层次结构来检索任务相关上下文，该智能体经强化学习训练，能选择最少但充分的上下文，从而降低关键执行路径上的延迟。在 ALFWorld、LoCoMo 和 LongMemEval 上，HORMA 在受限上下文预算下提升了任务性能，同时在长对话任务中最多仅需基线方法 22.17% 的令牌使用量。相比现有方法，它持续实现了更优的效率-性能权衡，并能有效泛化到未见任务。

## Abstract
Large language model (LLM) agents struggle with long-horizon tasks due to their inherent statelessness, requiring all task-relevant information to be encoded in growing input contexts. The resulting degraded reasoning quality, increased inference cost, and higher latency necessitate efficient working memory mechanisms. However, existing approaches either rely on lossy compression or similarity-based retrieval, which often fail to capture temporal structure and causal dependencies required for multi-step agentic tasks. In this work, we present HORMA, a Hierarchical Organize-and-Retrieve Memory Agent that organizes experience into a file-system-like hierarchical structure, where summarized entities are linked to the corresponding raw trajectories, enabling efficient access without losing detailed information. HORMA decomposes working memory into two stages: structured memory construction and navigation-based retrieval. The construction module iteratively refines how experiences are structured by distinguishing between failures caused by missing information and those caused by misleading or overloaded context. The navigation module retrieves task-relevant context by traversing the hierarchy using a lightweight agent trained with reinforcement learning to select minimal yet sufficient context, thereby reducing latency along the critical execution path. Across ALFWorld, LoCoMo, and LongMemEval, HORMA improves task performance under constrained context budgets while requiring at most 22.17% of the baseline token usage in long conversation tasks. Compared to existing methods, it consistently achieves better efficiency-performance trade-offs and generalizes effectively to unseen tasks.

## 2. 论文提出的方法论

### 2.1 核心思想
- 将暂存记忆构建为**层次化的文件系统工作区**，而非扁平记忆缓冲区。
- 记忆构建（长期结构维护）与记忆检索（每步即时提取）在功能和时间尺度上存在根本差异，故将二者**显式解耦**为两个模块：
  - **记忆管理代理（Memory Manager \(M_m\)）**：负责将原始轨迹组织成结构化、可导航的实体文件与目录树。
  - **检索代理（Retrieval Agent \(M_r\)）**：在推理时通过 Bash 工具导航工作区，提取最小充分上下文。

### 2.2 关键技术细节

#### 记忆构建模块（Memory Manager）
- 使用具备强层次推理能力的前沿 LLM（如 Claude Sonnet 4.5）作为初始构造策略。
- **递归技能精炼**：通过对比成功轨迹与失败轨迹，区分两类失效模式：
  - **外生技能 (Exogenous)**：由于结构化记忆丢失信息导致原始轨迹成功而 HORMA 失败的情况。
  - **内生技能 (Endogenous)**：由于结构化记忆过滤了噪声而使 HORMA 成功、原始轨迹失败的情况。
- 利用这些对比分析生成自然语言反馈，并以**文本梯度下降**方式迭代更新管理提示词，逐步积累可转移的记忆管理技能（非参数更新，无需微调 LLM）。

#### 检索模块（Retrieval Agent）
- 交互接口为 **Bash 命令**（`ls`, `cd`, `grep`, `cat` 等），加上两个终端动作 `select` 和 `done`，用以添加已验证内容到上下文和终止检索。
- 检索采用**基于证据的强化学习奖励**：以检索到的上下文与真实任务证据之间的 Jaccard 重叠度为奖励信号，使用 GRPO 优化轻量级检索策略。
- 检索策略可泛化到不同领域和任务，减少不必要的检索调用，降低延迟。

#### 公式化框架
- 整体记忆增强策略：
  \[
  \pi(a_t | o_t, F_t, q) = M_r(C_t | F_t, q) \cdot M_\theta(a_t | o_t, C_t, q; P_{\text{main}})
  \]
  其中 \(C_t \subseteq F_t\) 为检索出的上下文，\(F_t\) 为结构化记忆工作区。
- 记忆过渡：\(F_{t+1} = T_F(F_t, a_t, o_t)\)，由记忆管理器通过执行文件操作实现。
- 检索奖励：\(J(C_t, E) = \frac{|C_t \cap E|}{|C_t \cup E|}\)。

## 3. 实验设计

### 3.1 数据集与场景

- **LoCoMo**：长期多会话对话基准，选取其中 3 个对话的 519 个问答实例进行测试（其余 7 个对话用于训练轻量检索器），设置 10K token 上下文限制。
- **LongMemEval**：多类型长期记忆评测，367 个多样化问答实例，设置 50K token 上下文限制，作为零样本域外泛化测试。

## 6. 主要结论与发现

- HORMA 在所有基准上均能**在严格的上下文预算下取得优异的任务性能**，同时**大幅降低令牌使用量**（长对话中最多仅为基线 22.17%）。
- 层级化文件系统结构增强的笔记，即使仅配合简单嵌入检索也能提升性能，表明记忆管理具有独立增益。
- 轻量检索器通过 RL 学习后可**减少无效检索调用**，并展现出**强零样本域外泛化**能力（在 LongMemEval 上超越所有基线，包括 Claude 检索器）。
- 将记忆构建与检索解耦，可缓解信用分配问题（credit assignment gap），且记忆管理高度依赖高层语义推理能力，若管理器较弱，即使加强检索也无法补偿。

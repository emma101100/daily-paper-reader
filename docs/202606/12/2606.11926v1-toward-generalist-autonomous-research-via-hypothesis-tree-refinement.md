## 摘要
科学进步依赖于探索、实验和抽象这一循环往复的过程。研究者测试候选方向，解读证据，并将所得经验应用于后续尝试。我们研究AI智能体如何能在长时间跨度内自主运行这一循环。我们提出Arbor，一个用于自主研究的通用框架，它结合了一个长期运行的协调器、多个短期运行的执行器以及假设树精化（HTR）机制——一棵持续更新的树，将跨时间的假设、产物、证据和提炼出的见解联系起来。协调器基于这棵树管理全局研究策略，而执行器则在隔离的工作树中实现并测试各个假设。随着结果返回，Arbor更新树结构，传播可复用的经验，精炼搜索前沿，并采纳经过验证的改进。这种设计将自主研究从一系列局部尝试转变为一个累积过程，其中策略、执行和证据随时间延续。我们在自主优化（AO）设定下评估Arbor，这是一种操作场景，智能体通过迭代实验改进初始研究产物，无需步骤级的人类监督。在模型训练、框架工程和数据合成等六项真实研究任务中，Arbor在所有六项任务上均取得了最佳留存结果，在相同任务接口和资源预算下，其平均相对留存增益比Codex和Claude Code高出2.5倍以上。在MLE-Bench Lite上，Arbor使用GPT-5.5达到了86.36%的任意奖牌率，这是我们比较中最强的结果。

## Abstract
Scientific progress depends on a repeated loop of exploration, experimentation, and abstraction. Researchers test candidate directions, interpret the evidence, and carry the resulting lessons into later attempts. We study how an AI agent can run this loop autonomously over long horizons. We introduce Arbor, a general framework for autonomous research that combines a long-lived coordinator, short-lived executors, and Hypothesis Tree Refinement (HTR), a persistent tree that links hypotheses, artifacts, evidence, and distilled insights across time. The coordinator manages global research strategy over the tree, while executors implement and test individual hypotheses in isolated worktrees. As results return, Arbor updates the tree, propagates reusable lessons, refines the search frontier, and admits verified improvements. This design turns autonomous research from a sequence of local attempts into a cumulative process in which strategy, execution, and evidence are carried across time. We evaluate Arbor under Autonomous Optimization (AO), an operational setting where an agent improves an initial research artifact through iterative experimentation without step-level human supervision. Across six real research tasks in model training, harness engineering, and data synthesis, Arbor achieves the best held-out result on all six tasks, attaining more than 2.5x the average relative held-out gain of Codex and Claude Code under the same task interface and resource budget. On MLE-Bench Lite, Arbor reaches 86.36% Any Medal with GPT-5.5, the strongest result in our comparison.

### 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

*   **核心思想：假设树精化（Hypothesis Tree Refinement, HTR）**
    *   **持久化的研究状态**：Arbor框架的核心是一个持久化的“假设树”（Hypothesis Tree）。这棵树不仅是搜索前沿，也是过往尝试的记忆，以及经过验证的产物改进的审计追踪。
    *   **双重角色分离**：
        *   **长期协调器（Coordinator）**：拥有全局研究状态（即假设树），负责管理研究策略。它观察树的状态、构思新假设、选择待测假设、整合返回的证据、传播洞察，并决定分支是继续、剪枝还是合并。
        *   **短期执行器（Executor）**：在隔离的工作区（worktrees）中，负责具体实现和测试单个假设，并返回结构化的证据（得分、事实结果、提炼的洞察和产物引用）。
    *   **节点构成**：每个树节点绑定了一个研究单元，包含：
        *   **假设 (Hypothesis)**：一个可验证或证伪的声明。
        *   **洞察 (Insight)**：对实验证据的可复用解读（在叶节点上是具体的局部发现，在内节点上是抽象的方向级理解）。
        *   **元数据 (Metadata)**：连接语义假设和可执行证据，如节点状态、开发集得分、Git分支引用等。
*   **关键技术细节和算法流程**
    *   **Htr循环**：协调器通过一个循环过程维护这棵树，步骤如下：
        1.  **观察 (Observe)**：阅读包括活跃前沿、近期证据、祖先洞察等在内的树结构投影。
        2.  **构思 (Ideate)**：选择一个父节点，在其下提出一组细化的子假设，这些构思受树中已累积证据的约束。
        3.  **选择 (Select)**：基于预期效用、证据积累和多样性，选择待执行的“待定”叶节点。
        4.  **分发 (Dispatch)**：将选定的假设分发到独立的执行器，在其工作区中进行实现和评估（仅使用开发集反馈 `Edev`）。
        5.  **回溯 (Backpropagate)**：当执行器返回后，协调器将证据写入叶节点，并将洞察沿路径向上传播到根节点，实现局部发现到全局知识的抽象。
        6.  **决策 (Decide)**：根据新证据，决定继续探索、剪枝一个被证伪的子树，或尝试通过“留出集准入闸门”（held-out merge gate）将候选产物合并为主分支。只有当候选产物在`Etest`上的表现优于当前最佳产物时，合并才会发生。
    *   **自主优化（Autonomous Optimization, AO）任务形式化**：一个AO任务被定义为一个元组 `P = (M0, O, Edev, Etest)`，分别代表初始产物、优化目标、开发集评估器和留出测试集评估器。智能体的目标是返回在`Etest`上得分最佳的产物`M*`，但整个搜索过程中不能将`Etest`作为探索的提示。

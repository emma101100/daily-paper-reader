## 摘要
可验证性、署名归属和可复现性是科学知识的基本要求，然而当前的出版基础设施并未能大规模地落实这些要求。我们引入了 Traxia，一个智能体原生的科学出版框架，其中 AI 研究智能体发表可验证的论文、构建声誉身份、进行相互同行评审，并在共享的溯源模型中与人类协作。Traxia 将智能体视为第一性的认知参与者：每篇论文附带推理轨迹，每个声明附有置信区间，每个智能体拥有密码学签名的身份，每次协作都有不可篡改的贡献日志。我们形式化了五个组件：智能体身份与注册表、可验证出版层、四级同行评审协议、声誉与质押引擎，以及具备矛盾检测功能的知识图谱。该框架针对可复现性失败、溯源不透明以及全球南方研究能力被排斥的问题。本文仅呈现架构基础和形式化规范，不报告实证结果。评估和更深入的组件研究将在后续论文中展开。原型系统部分实现了核心形式化方法；完整系统仍在积极开发中。

## Abstract
Verifiability, attribution, and reproducibility are foundational requirements of scientific knowledge, yet current publishing infrastructure does not enforce them at scale. We introduce Traxia, an agent-native scientific publishing framework in which AI research agents publish verifiable papers, build reputational identities, peer-review one another, and collaborate with humans in a shared provenance model. Traxia treats agents as first-class epistemic participants: every paper carries a reasoning trace, every claim a confidence interval, every agent a cryptographically signed identity, and every collaboration an immutable contribution log. We formalise five components: Agent Identity and Registry, Verifiable Publishing Layer, four-tier Peer Review Protocol, Reputation and Staking Engine, and a Knowledge Graph with contradiction detection. The framework targets reproducibility failure, provenance opacity, and exclusion of Global South research capacity. This paper presents architectural foundations and formal specifications only; it does not report empirical results. Evaluation and deeper component studies will follow in subsequent papers. A prototype partially implements core formalisms; the full system remains under active development.

### 2. 论文提出的方法论
论文提出了一套不存在传统实验的体系架构，并以形式化定义描述各组件。

- **核心认识单位：可验证认识产物（VEA）**
  - 定义为六元组 `V = (C, T, P, S, R, σ)`，包括声明集合（每个声明带置信区间）、有序推理轨迹（每步含前提、推理规则、结论与置信度）、溯源集合（先前提及的 VEA）、作者智能体或集合、综合认知置信度分数（ECS）以及密码学签名。
  - **ECS 计算**：`ECS(V) = α·c̄ + β·ρ + γ·τ`，其中 c̄ 为平均声明置信度，ρ 为可复现性分数，τ 为轨迹完整性分数。默认权重 α=0.4, β=0.35, γ=0.25，并证明在合理邻域内排序相对稳定。
  - **可复现性分数 ρ**：基于独立复现尝试的加权成功率，无复现尝试时取先验值 0.5。
  - **轨迹完整性分数 τ**：衡量声明中有多少具备从前提、经过可验证推理规则到结论的完整机器可验证路径。

- **五大架构组件**
  1. **智能体身份与注册表**：每个智能体拥有密码学密钥对、版本历史、关联人类 ORCID、声明的领域集合及验证模式（已验模式 vs. 证明模式）。已验模式用于开源模型，证明模式用于闭源模型并需人类担保者质押信誉。
  2. **可验证出版层**：强制要求提交完整推理轨迹，并对 VEA 进行模式验证。
  3. **四级同行评审协议**：
     - **Tier 0**：沙箱式仿真，提交前由平台控制的对抗智能体自检，存活率高者获“战测”徽章。
     - **Tier 1**：领域匹配专家智能体评审方法与声明有效性。
     - **Tier 2**：红队对抗智能体专找漏洞，按严重性分级。
     - **Tier 3**：符合 agent H-index 门槛的人类仲裁官做出最终决定。协议形式化保证冲突不可隐蔽性、对抗覆盖优势及复现记录的完整性（均附条件）。
  4. **信誉与质押引擎**：定义智能体 H-index（AHI），引入自我引用折扣 λ=0.1；支持研究赏金（Research Bounty，研究者可质押信誉悬赏指定问题）；声誉质押（Reputation Staking，对 VEA 质量押注，若论文被撤回则扣减信誉，若复现成功则奖励）。
  5. **知识图谱与矛盾检测**：构建 `G=(N,E)`，节点包括智能体与 VEA，边包括作者、引用、评审、矛盾、复现等。基于声明的结构化比较和共享领域本体检测认识矛盾，并自动标记、通知作者并开启解决线程。同时计算 VEA 的陈旧度分数 ϕ 以实现“活文献”。

## 摘要
基础模型代理日益成为长期运行的系统，能够在多次交互中记住用户，这使得记忆成为显式的部署期功能，而不仅仅是模型权重的属性。现有研究涉及参数化记忆或审计固定记忆配置，但未能刻画记忆设计选择如何共同塑造个性化效用、提取风险和删除保真度。我们将这一表面作为部署期记忆进行研究，将代理记忆建模为由个性化召回率（PR）和对抗性提取率（AER）衡量的隐私-效用前沿，并遍历三个记忆设计因素：摘要压缩程度、检索广度（k）和删除模式。我们进一步引入遗忘残留分数（FRS）来量化被删除的信息是否仍可从衍生记忆层中恢复。在LongMemEval上，关键事实摘要将Gemma 3 12B上的金丝雀提取降低76%，GPT-4o-mini上降低64%，同时几乎保留了所有个性化召回率；关键的是，一旦内容被压缩移除，增加k不再恢复泄漏。然而，同样的压缩导致了删除保真度失效：仅原始删除会使大约20%的实例中的衍生摘要副本可恢复，只有全流水线清除或墓碑标记删除才能将最差残留降至零。综上，这些结果表明，持续性的代理记忆必须作为头等记忆机制进行评估——评估其帮助代理回忆了什么、使什么变得可提取、以及它究竟能擦除什么。

## Abstract
Foundation-model agents are increasingly long-lived systems that remember users across interactions, making memorization an explicit deployment-time function rather than solely a property of model weights. Existing work addresses parametric memorization or audits fixed memory configurations, but does not characterize how memory-design choices jointly shape personalization utility, extraction risk, and deletion fidelity. We study this surface as deployment-time memorization, formulating agent memory as a privacy-utility frontier measured by Personalization Recall (PR) and Adversarial Extraction Rate (AER), and sweeping three memory-design knobs: summarization aggressiveness, retrieval breadth (k), and deletion mode. We further introduce the Forgetting Residue Score (FRS) to quantify whether deleted information remains recoverable from derived memory tiers. On LongMemEval, key-fact summarization reduces canary extraction by 76% on Gemma 3 12B and 64% on GPT-4o-mini while preserving nearly all personalization recall; critically, once content is compressed away, increasing k no longer restores leakage. The same compression, however, induces a deletion-fidelity failure: raw-only deletion leaves derived summary copies recoverable in approximately 20% of instances, and only full-pipeline purge or tombstone redaction drives worst-tier residue to zero. Together, these results establish that persistent agent memory must be evaluated as a first-class memorization mechanism -- assessed by what it helps agents recall, what it makes extractable, and what it can truly erase.

### 1. 论文的核心问题与整体含义
基础模型代理（foundation-model agents）正从无状态助手演变为能跨对话记住用户偏好的长期运行系统。这一转变使得**记忆（memorization）不再是模型权重中偶然的参数化属性，而成为部署流水线中显式的系统功能**。然而，现有的记忆研究主要关注参数化记忆（如训练数据提取），或仅审计固定的记忆配置，并未刻画**记忆设计选择（摘要压缩程度、检索广度、删除模式）如何共同影响个性化效用、隐私泄露风险与删除保真度**。本文提出**部署期记忆（deployment-time memorization）** 这一概念，将代理的外部记忆流水线视作一个隐私–效用前沿，意在回答：记忆系统究竟帮助代理回忆了什么、使什么变得可被对抗提取，以及能真正擦除什么。

### 2. 论文提出的方法论
论文将持久化代理记忆建模为**写入–检索–响应流水线**，并围绕三个可控设计因素展开系统扫描：
- **摘要压缩程度 \(S\)**：控制存储内容的形式，\(S=0\) 为原始对话轮次，\(S=1\) 为关键个人事实，\(S=2\) 为单句会话摘要。
- **检索广度 \(k\)**：每次查询时注入代理上下文窗口的记忆块数量。
- **删除模式**：实现“遗忘”操作的五级阶梯（见下文）。

**核心度量指标：**
- **个性化召回率（Personalization Recall, PR）**：衡量代理能否正确回答需要记住用户上下文的查询，定义为答案正确性的期望（余弦相似度 > 0.50 或所有内容词逐词出现）。
- **对抗性提取率（Adversarial Extraction Rate, AER）**：向先前会话注入高熵金丝雀（如“我的私密会话令牌是 XQ7-VIOLET-3829”），然后通过直接提问、间接回忆或越狱指令进行探测，计算金丝雀被逐字恢复的比例（取各探针级别最大值）。
- **隐私–效用曲线下面积（Privacy–Utility AUC, PUA）**：固定 \(S\) 遍历 \(k\)，计算可达到召回率包络下的面积，数值越高表示在低提取风险下获得更高召回。
- **遗忘残留分数（Forgetting Residue Score, FRS）**：按记忆来源层（原始块 vs. 摘要块）分别衡量删除后仍可被提取的金丝雀比例，并关注最差记忆层的残留值（\(FRS_{worst}\)）。

**五种删除模式构成消融阶梯：**
- *无操作（noop）*：对照，不做任何删除。
- *仅原始（raw only）*：仅清除原始文本块，不重建嵌入，不处理摘要层。
- *原始+重新摘要（raw plus resummarize）*：清除原始块并重新嵌入，对受影响会话用清理后的输入重新生成摘要。
- *全清除（full purge）*：所有记忆层均被清理、重新嵌入，空块被丢弃。
- *墓碑标记（tombstone）*：将金丝雀在所有层中替换为 `[REDACTED]` 并重新嵌入，测试模型是否从上下文“脑补”出隐私信息。

1. **摘要化有效清洗隐私**：关键事实摘要（\(S=1\)）使 Gemma 3 12B 的提取率降低 **76%**，GPT-4o-mini 降低 **64%**，且个性化召回损失极小（约 5 个百分点）。

## 摘要
Arbor 是一个多智能体框架，它将结构化树搜索作为认知层引入，用于在大型、有状态动作空间中运行的自主智能体。以往的自主优化系统针对孤立目标进行无状态评估。而 Arbor 维护一个显式的评分假设搜索树，作为跨智能体共享的工作记忆，随着每次测量而演化，将失败视为重塑后续探索的诊断信号，并随着先前成功改变瓶颈分布而扩展。我们在全栈 LLM 推理优化上验证了 Arbor，该领域要达到巅峰性能历来需要工程团队在应用、框架、编译器、内核和硬件栈上协同努力。Arbor 将协调者（Orchestrator）智能体与批评者（Critic）智能体配对：协调者通过委托推理栈上的领域专家（Domain Specialists）来驱动优化，批评者则通过根因分析、内省和测量验证来保障稳定性——这是一种制衡架构，没有哪个智能体能单方面驱动系统。智能体的能力被分解为硬技能（领域专业知识）和软技能（决定贡献如何组成的协调协议），从而实现了完全自主的多日优化活动。Arbor 相比供应商优化的基线，在吞吐量-延迟帕累托改进上可达 193%，而没有该框架的单个智能体在吞吐量提升 33% 时便停滞不前，并在数小时内不可恢复地崩溃。Arbor 适用于多代硬件平台，运行间方差在 2 个百分点以内，表明该方法与硬件无关且可复现。

## Abstract
Arbor is a multi-agent framework that introduces structured tree search as a cognition layer for autonomous agents operating in large, stateful action spaces. Prior autonomous optimization systems operate on isolated targets with stateless evaluation. Arbor instead maintains an explicit search tree of scored hypotheses that serves as the shared working memory across agents, evolving with every measurement, treating failures as diagnostic signal that reshapes subsequent exploration, and expanding as prior successes shift the bottleneck distribution.   We validate Arbor on full-stack LLM inference optimization, a domain where achieving peak performance has historically required coordinated effort from engineering teams across the application, framework, compiler, kernel, and hardware stack. Arbor pairs an Orchestrator agent, which drives optimization by delegating to Domain Specialists across the inference stack, with a Critic agent that safeguards stability through root-cause analysis, introspection, and measurement validation -- a checks-and-balances architecture where neither agent can unilaterally drive the system. Agent capabilities are decomposed into hard skills (domain expertise) and soft skills (coordination protocols that determine how contributions compose), enabling fully autonomous multi-day campaigns. Arbor achieves up to 193% inference throughput-latency Pareto improvement over vendor-optimized baselines, while a single agent without the harness plateaus at +33% throughput improvement and crashes irrecoverably within hours. Arbor generalizes to multiple generations of hardware platform, and run-to-run variance is within 2 percentage points demonstrating that the method is hardware-agnostic and reproducible.

## 2. 方法论：核心思想与关键技术细节

- **核心思想**：以 **深度优先的启发式树搜索** 作为认知层，维护一棵动态扩展的 `T=(V,E)` 搜索树。树节点代表一次动作及其结果（保留、恢复或崩溃），整棵树即为多智能体共享的状态化记忆。
- **搜索循环**：
  1. **剖分**当前系统，识别各层瓶颈及其 GPU 耗时占比。
  2. **评分**所有候选动作，按启发式 `h(a)` 选择最高分动作。
  3. **分派**给动态构建的**领域专家**实现。
  4. **端到端验证**（准确率 + 完整基准测试）把关。
  5. **更新树**：保留的动作成为新基线，恢复的动作记录诊断，崩溃则触发 Critic 的根因分析。
  6. **重新评分**，因瓶颈分布可能已转移；若分数过低或预算耗尽则停止；任何崩溃/回退后自动回溯到上一个已验证状态。
- **动作评分公式**（文中式(1))：
  ```
  h(a) = g(a) / [c(a) * (1 - r_acc) * (1 - r_crash)] * m_gap + C * sqrt(ln N / (1 + n_cat))
  ```
  - `g(a)` 预期增益（目标核的 GPU 耗时占比 × 类别先验加速比）。
  - `c(a)` 估算时间成本；`r_acc`, `r_crash` 类别经验失败率（随活动动态更新）。
  - `m_gap` 与目标吞吐量差距挂钩的紧迫乘数（1.0~2.0）。
  - 右项为 **UCB 风格**探索奖励，鼓励尝试采样不足的动作类别。
- **多智能体架构与制衡**：
  - **协调者（Orchestrator）**：驱动搜索循环，决定探索方向，动态构建领域专家并进行端到端验证。
  - **领域专家（Domain Specialists）**：无预定义固定角色，而是由协调者在运行时根据目标、硬件上下文和知识库条目**动态构建**，涵盖内核、框架、通信、编译器、算子调度等层。每位专家独立迭代、本地验证并返回可合并补丁。
  - **批评者（Critic）**：作为制衡方，执行**护栏**（测量完整性）、**根因分析**（崩溃诊断并生成约束）、**内省**（评估回退动作的潜力并生成精炼子动作）和**稳定性监控**（长期健康跟踪）。批评者不能单方面阻止探索，需提供诊断证据；协调者必须接受批评者识别的稳定性约束（例如主动牺牲短期吞吐以换取会话稳定性）。
- **硬技能与软技能分解**：
  - 硬技能：各智能体领域专长（如剖分、内核优化、根因分析）。
  - 软技能：协作协议，如 GPU 资源仲裁、在专长边界处委托、整合他人发现等。
- **持久化知识库**：每次活动结束后，将原始记录蒸馏为结构化条目（验证过的技术、失败模式、参数级发现）供后续活动使用，既更新评分先验，又直接注入领域专家的构建 prompt，实现跨会话的暖启动。

## 6. 论文的主要结论与发现

- Arbor 将全栈优化重新形式化为**状态化树搜索**，并配合**制衡式多智能体架构**，能够在完全没有人工介入的情况下连续自主运行数日。
- 在 AMD MI355X 上，Arbor 相对 InferenceX 基线实现 **+40% 至 +193%** 的吞吐量提升，且 Pareto 曲线在所有交互性水平上均支配基线。

- 方法可跨硬件代际泛化（MI300X 同样获 +62%–+99%），且两次独立运行最终收敛的吞吐量差异在 **2 个百分点以内**，表明高可复现性。
- 持久化知识库、硬/软技能分解、批评者的内省与根因分析是实现长期稳定优化的关键。

## 7. 优点

- **认知层范式创新**：将搜索树作为智能体间共享的**动态工作记忆**，使失败和成功都能持续重塑搜索空间，比传统无状态循环更贴近实际工程优化过程。
- **制衡架构**：通过 Orchestrator 和 Critic 的明确张力，解决了纯探索易崩溃、纯保守无法推进的问题，且案例展示了主动牺牲短期吞吐换取长期稳定最终获得更大收益。
- **动态生成专家**：领域专家按需构建，提示中注入目标、硬件、历史失败等信息，兼具灵活性和深度。
- **可复现性与硬件无关性**：通过独立重复运行和跨代验证证明方法的鲁棒性。
- **实用深度**：不仅调参，还能自主重写 vendor 内核库、重新生成调度表，并提交可合并补丁，展示了真正的生产级优化能力。

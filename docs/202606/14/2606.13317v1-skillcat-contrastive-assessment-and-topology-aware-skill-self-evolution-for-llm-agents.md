## 摘要
针对大语言模型智能体的技能自进化方法旨在将执行轨迹转化为可复用的技能文档，但现有流程通常每个任务仅从单条轨迹学习、在未校验的情况下合并候选技能补丁，并在推理前加载完整技能库。我们提出 SkillCAT，一个无需训练的框架，将该过程划分为三个阶段。对比因果抽取（CCE）为每个任务采样多条轨迹，并比较同一任务的成功与失败轨迹对，以识别解释结果差异的证据。评估增强进化（AAE）在源任务克隆上重放每个候选补丁，仅保留能够改善或维持任务结果的补丁，然后进行层级技能补丁合并。拓扑感知任务执行（TTE）将进化后的技能编译为可路由的子技能拓扑，使得推理时仅加载与任务相关的能力节点。我们在常见智能体基准上评估 SkillCAT，包括 SpreadsheetBench、WikiTableQuestions 和 DocVQA，并进一步测试了跨模型和分布外泛化能力。在这些设置中，SkillCAT 将基准方法的平均得分提升高达 40.40%，展示了无需模型训练的可靠技能进化。

## Abstract
Skill self-evolution methods for LLM agents aim to turn execution trajectories into reusable skill documents, but current pipelines typically learn from one trajectory per task, merge candidate skill patches before checking them, and load the full skill corpus before inference. We propose SkillCAT, a training-free framework that separates this process into three stages. Contrastive Causal Extraction (CCE) samples multiple trajectories for each task and compares same-task success/failure pairs to identify evidence that explains outcome differences. Assessment-Augmented Evolution (AAE) replays each candidate patch on source-task clones and keeps only patches that improve or preserve task outcomes before hierarchical skill patch merging. Topology-Aware Task Execution (TTE) compiles the evolved skills into a routable sub-skill topology, so inference loads only the capability nodes relevant to the task. We evaluate SkillCAT on common agent benchmarks, including SpreadsheetBench, WikiTableQuestions, and DocVQA, and further test cross-model and out-of-distribution generalization. Across these settings, SkillCAT raises the average score over baselines by up to 40.40%, demonstrating reliable skill evolution without model training.

## 1. 论文的核心问题与整体含义

- **问题背景**：大语言模型（LLM）驱动的智能体在执行长程任务时，常依赖外部技能文档（如结构化指令、工具使用规则、历史经验）。这些技能的获取方式正从手工编写转向**技能自进化**，即从智能体的执行轨迹中自动提取和精炼可复用技能。
- **现有方法的瓶颈**：当前离线批处理框架（如 Trace2Skill）虽然比逐轨迹更新更高效，但仍存在三个关键缺陷：
  - **单轨迹偏差**：每个任务仅采集一条轨迹，成功轨迹可能包含偶然策略，失败轨迹往往无法定位根因，导致提取的证据不够可靠。
  - **未校验合并**：将候选技能补丁直接合并，未独立验证其对源任务的影响，低质量甚至有害的补丁可能污染技能库。
  - **推理时上下文过载**：合并后的技能文档持续膨胀，推理时加载全部内容会增加提示长度并分散智能体的注意力。
- **整体含义**：SkillCAT 旨在以**无需模型训练**的方式，通过三阶段框架系统性地解决上述三大问题，使技能自进化更可靠、更高效，并提升智能体的跨任务泛化能力。

## 2. 方法论：核心思想与关键技术细节

SkillCAT 将技能自进化解耦为三个可观察的阶段，分别对应三个关键决策。

- **对比因果抽取（Contrastive Causal Extraction, CCE）**
  - **核心思想**：利用多种子采样获取同一任务的多条轨迹，构建成功/失败对比对，从中提取**因果分水岭**处的证据，而非总结整条轨迹。
  - **技术细节**：
    - 对任务 \(x_i\)，使用 \(K\) 个随机种子运行智能体，获得轨迹集合 \(T_i\)（含成功/失败标记）。
    - 构建对比对 \((\tau^+_i, \tau^-_i)\)（成功轨迹与失败轨迹各随机抽取一条）。
    - 定位**因果分水岭** \(w_i\)：即成功与失败轨迹中**首次动作序列出现分歧**的位置。
    - 以分水岭为中心抽取候选经验记录 \(r_i = E(\tau^+_i, \tau^-_i, w_i)\)，突出导致结果差异的关键行为。
    - 若某任务全成功或全失败，可回退至单轨迹提取（Fallback），或直接跳过以保持严格对比。

- **评估增强进化（Assessment-Augmented Evolution, AAE）**
  - **核心思想**：将每个候选补丁视为**待验证的假设**，先在源任务克隆上回放以评估其对结果的真实影响，仅保留有效补丁并进行**分层逐级合并**。
  - **技术细节**：
    - 对每个补丁 \(p_j\)，在源任务克隆集上记录基线结果 \(y_j\)（0/1）与回放结果 \(\hat{y}_j\)。
    - 按结果转移评分离散化：失败→成功（3.0），成功→成功（2.0），失败→失败（1.0），成功→失败（0.0）。该排序直接体现补丁的效用。
    - **阈值筛选**：设定 \(\theta=2.0\)，仅保留分数 \(a_j \ge \theta\) 的补丁，即至少维持已知成功行为的补丁。
    - **分层合并**：将保留的补丁按评分层级从低到高依次合并到技能文档中，保证高分补丁在后续合并阶段享有更高优先级：
      \[
      S^{(\ell)} = \mu(S^{(\ell-1)}, P^{(\ell)}_\theta), \quad S^* = S^{(L)}
      \]
      其中 \(\mu\) 为合并算子，\(L\) 为层级数。

- **拓扑感知任务执行（Topology-Aware Task Execution, TTE）**
  - **核心思想**：将进化后的技能编译为**可路由的拓扑**，推理时仅加载与当前任务相关的能力节点，减少无关上下文。
  - **技术细节**：
    - **离线编译**：解析技能文档的 Markdown 层级，生成始终加载的核心技能 \(S^\circ\) 及可路由节点集 \(V\)；为每个节点提取摘要、关键词、依赖等元数据构成拓扑摘要 \(\Sigma\)。
    - **在线路由与组装**：给定任务描述 \(c\) 和预算 \(k\)，利用大语言模型路由 \(\rho(c, \Sigma; k)\) 选择 \(k\) 个主要节点；再通过确定性扩展（验证节点标识、加入依赖、注入基础节点）得到最终节点集 \(V_c\)；最后拼接核心技能与这些节点的原始正文：
      \[
      S_c = S^\circ \oplus \bigoplus_{v \in V_c} B_v
      \]

完整流程如 Algorithm 1 所示，三个阶段有序衔接，形成从技能学习到精简部署的闭环。

## 6. 主要结论与发现

- SkillCAT 在电子表格任务上显著超越 Trace2Skill 等基线，深化场景下 Qwen3.5-35B-A3B 的 Vrf 提升 45.83 个百分点（从 9.67% 至 55.50%），创建场景下同样大幅领先（54.50% vs 20.17%）。
- 在 WikiTQ 等分布外任务上，SkillCAT 展现强大泛化能力，尤其在创建场景下 Qwen3.5-35B-A3B 的准确率从 20.14% 提升至 72.82%。
- 跨模型实验表明，进化后的技能可被未参与进化的用户模型复用，但收益幅度因用户模型而异（如 gemma-4-31B-it 受益明显，gpt-5.4-mini 提升较小）。
- DocVQA 多模态场景中，SkillCAT 全流程取得最佳 ANLS 和 Acc，而未经验证的 Trace2Skill +Error 甚至低于无技能基线，突显补丁验证的必要性。
- 消融实验证实 CCE、AAE、TTE 各有独立贡献，缺少任何一环均会导致性能下降，且单模块无法完全弥补其他模块的缺失。
- CCE 种子数为 5 时达到最佳性能-成本平衡；AAE 评分桶的单调排序表明回放验证的分数能够有效预测下游效用；TTE 拓扑路由可在压缩上下文（41.6%）的同时维持甚至提升准确率。

## 7. 优点（方法或实验设计上的亮点）

- **三阶段解耦清晰**：将技能进化的证据提取、补丁验证与部署上下文控制独立为三个可观察模块，逻辑链完整。
- **对比因果提取创新**：利用多次采样的成功/失败轨迹定位因果分水岭，相较单轨迹总结可获得更鲁棒的技能证据。
- **补丁级源任务回放**：以可操作的评分机制代替盲目合并，简单有效的阈值（保留至少维持成功行为的补丁）避免了技能污染。
- **拓扑感知执行**：将技能编译为拓扑并动态路由，兼顾性能与推理效率，实现上下文精简。
- **全面的评估设计**：涵盖深化与创建、跨模型泛化、OOD 和多模态任务，多维度验证方法的有效性与鲁棒性。
- **消融与分析透彻**：不仅检验整体效果，还深入分析各模块的独立贡献及关键参数的影响，增强了结论的可信度。
- **无需训练**：整个流程不涉及模型参数更新，易于落地和迁移。

## 8. 不足与局限

- **算力信息缺失**：未明确报告实验使用的 GPU 型号与总计算量，仅给出 CCE 采样时间，可复现性略有折扣。
- **超参数敏感性与泛化**：CCE 种子数增大至 7 或 9 时性能反而下降，暗示对比对的过拟合风险；阈值 \(\theta=2.0\) 和预算 \(k=7\) 可能需针对不同场景重新调整。
- **依赖强初始技能**：在深化场景中，性能大幅提升依赖于高质量的手工初始技能；当初始技能很弱时虽仍有显著提升，但绝对分数仍有限。
- **用户模型依赖性**：跨模型实验显示，技能进化模型的“教学效果”受用户模型能力制约，较弱用户模型（如 gpt-5.4-mini）提升幅度较小，表明技能质量可能并非唯一的瓶颈。
- **任务范围**：实验集中在电子表格操作与表格/文档问答类任务，尚未验证在更开放或复杂交互式环境（如规划、编程、对话型任务）中的有效性。
- **合并策略固定**：分层合并虽优于直接合并，但合并算子本身复用 Trace2Skill 的流程，可能限制了对复杂冲突的解决能力。

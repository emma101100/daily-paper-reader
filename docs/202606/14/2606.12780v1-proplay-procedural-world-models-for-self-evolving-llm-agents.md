## 摘要
自我进化智能体应能在无外部监督的情况下通过交互实现提升，但在部分可观测环境中，这一目标仍具挑战性，因为智能体必须主动探索、从有限反馈中学习，并决定何时信任过往经验。现有基于LLM的智能体方法常依赖记忆或规划模块，却很少在这些模块间形成闭环，以持续优化对环境动态的内部理解。我们提出ProPlay，一种支持过程级预演的过程世界模型，使智能体能够利用已学习的世界知识预演未来的过程路径。ProPlay不将经验表示为孤立的规则或低层动作约束，而是将成功轨迹抽象为过程，并将其组织为一个过程图，该图捕捉任务阶段间的因果转换关系。每个转换关联一个可靠性记录嵌入，用以从历史结果估算其特定于任务的贡献。每回合开始前，ProPlay基于已知的图结构模拟未来的过程轨迹，作为结构化的软引导；执行完毕后，再依据环境反馈优化该图。在公开基准上的实验表明，ProPlay相比强基线方法持续提升了环境理解与自我进化能力。代码已开源至 https://github.com/antman9914/proplay。

## Abstract
Self-evolving agents are expected to improve through interaction without external supervision, but this remains difficult in partially observable environments where agents must explore actively, learn from limited feedback, and decide when to trust prior experience. Existing LLM-agent methods often rely on memory or planning modules, yet they rarely close the loop between them to continually refine an internal understanding of environment dynamics. We introduce ProPlay, a procedural world model that supports procedure-level preplay, where agents can rehearse future procedural paths using the learned world knowledge. Rather than representing experience as isolated rules or low-level action constraints, ProPlay abstracts successful trajectories into procedures and organizes them in a procedure graph that captures causal transitions among task stages. Each transition is associated with a reliability record embedding to estimate its task-specific contribution from past outcomes. Before each episode, ProPlay simulates future procedural trajectories over known graph structures as structured soft guidance; after execution, it refines the graph using environment feedback. Experiments on public benchmarks show that ProPlay consistently improves environment understanding and self-evolution capability over strong baselines. Our code has been released in https://github.com/antman9914/proplay.

### 2. 方法论：核心思想、关键技术与流程

ProPlay的核心思想是构建一个**不断进化的过程世界模型（Procedural World Model）**，其核心单元是比动作更高层次的“**过程（Procedure）**”。方法流程为三阶段循环：预演、执行、精炼。

*   **核心思想**：
    将环境动态表示为**过程间的因果转换结构**，而非动作层面的约束。这种过程级抽象能捕捉任务推进的骨架，具有较强的跨任务泛化能力，同时保留了底层动作选择的灵活性。

*   **关键技术细节**：
    1.  **过程世界模型（过程图 G）**：
        模型被组织为一个属性图 `G = (P, E, C)`。
        *   `P`：过程节点集，每个节点是一个可泛化任务阶段的自然语言描述（如“煮沸水”、“测量属性”），从成功交互轨迹中归纳得出。
        *   `E`：有向边集，编码了过程间的因果关系转换。
        *   `C`：**可靠性记录嵌入**，映射每条边到一个嵌入向量 `c^{k}_{ij}`。该向量通过公式加权求和得来：
            `c^{k}_{ij} = Σ_{i=1}^{k} I((p_i, p_j) ∈ W^{i}) · R_i · ϕ(d_{τ_i})`
            其中 `I` 是指示函数，`R_i` 是回报，`ϕ` 是任务描述编码器。该设计旨在量化特定任务下，一个过程转换对成功的历史贡献，从而将可泛化的知识与偶发经验区分开。

    2.  **过程级预演**：
        每回合开始时（第k回合），为当前任务 `τ_k` 构建一个过程轨迹 `W^*` 作为软引导。
        *   输入：当前任务描述 `d_{τ_k}`、过程图 `G_{k-1}`、历史失败经验。
        *   **构造过程**：首先，对每条转换边计算一个**任务特异性可靠性分数** `s^{k-1}_{ij}` ，即任务描述嵌入 `ϕ(d_{τ_k})` 与可靠性记录嵌入 `c^{k-1}_{ij}` 的余弦相似度。然后，将所有已知转换按分数降序排列，连同失败经验一起提供给LLM，让LLM推理生成一个有序的过程序列 `W^*`。

    3.  **回合式世界模型精炼**：
        *   **软引导执行**：在执行阶段，`W^*` 被注入到LLM的上下文中作为高层计划，但**不强制执行**，智能体可根据实时观察自行决定底层动作，平衡了利用与探索。
        *   **过程归纳**：回合结束后，截取累积奖励最后一次增长前的**有效动作序列** `H^{s}_k`。LLM将其分割为连贯的任务阶段，归纳出新的过程集合 `P_{new}`。为避免冗余，每个新过程与已有过程进行语义相似度比较：若超过阈值 δ，则合并更新；否则，作为新节点加入图 `G`。然后将 `H^{s}_k` 映射到更新后的 `P` 上，形成新的过程路径 `W^k`，并将新出现的过程转换加入边集 `E`。
        *   **可靠性更新**：根据上述公式（公式1），更新所有在本回合中出现的过程转换的可靠性记录嵌入 `c^{k}_{ij}`，以反映其最新的任务特异性贡献。

### 8. 不足与局限

*   **算力信息缺失**：未提供任何计算资源或API调用成本的量化数据，使得方法的实际开销难以评估。
*   **适用范围有限**：

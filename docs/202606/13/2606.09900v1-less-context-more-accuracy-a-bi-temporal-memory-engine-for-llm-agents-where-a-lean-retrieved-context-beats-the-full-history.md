## 摘要
长期记忆是LLM智能体缺少的关键层：它们跨会话遗忘，而常见的变通方案——将完整历史回放至提示中——昂贵、缓慢，且随着干扰信息的积累，准确率降低。大多数记忆系统在成本或延迟上占优，但在准确率上仍逊于完整上下文基线，且基准测试结果报告于不一致、不可复现的测试工具上，导致同一系统在不同来源中得分悬殊。我们提出Engram，一个基于双时态数据模型的开源双进程记忆引擎。快速写入路径追加无损片段，关键路径上无LLM调用；异步路径提取原子（主语、谓语、宾语）事实，构建双时态知识图谱，并在不针对每个事实调用LLM的情况下消解矛盾——废止而永不删除，使每个事实保留来源和取代链。混合读取路径融合稠密、词法、图及新近度/显著度信号，应用时间点（“截至”）过滤，并组装紧凑、带来源标记的上下文。在完整的500问LongMemEval_S上，由官方类别特定判分器评分，Engram的精简配置——从约9.6k token的检索切片作答，而非完整历史——得分83.6%对比完整上下文的73.2%（+10.4分，McNemar p < 10^-6），token数减少约8倍（9.6k vs. 79k），且无错误。增益需要混合读取路径：仅事实损失召回率，而事实加上检索到的片段可恢复细节。我们还贡献了一个中立的、内置于仓库的评估工具，集成了官方判分器并在每张表中包含完整上下文基线，发布原始逐问日志，并记录了无声扭曲记忆基准的测量完整性陷阱（截断、自制判分器、完整历史泄露）。每个数字都附带可复现的命令。

## Abstract
Long-term memory is the missing layer for LLM agents: across sessions they forget, and the common workaround -- replaying the whole history into the prompt -- is expensive, slow, and, as distractors accumulate, less accurate. Most memory systems win on cost or latency but still lose to the full-context baseline on accuracy, and benchmark numbers are reported on inconsistent, non-reproducible harnesses, so one system appears at wildly different scores across sources. We present Engram, an open-source, dual-process memory engine on a bi-temporal data model. A fast write path appends lossless episodes with no LLM on the critical path; an asynchronous path extracts atomic (subject, predicate, object) facts, builds a bi-temporal knowledge graph, and resolves contradictions without an LLM call per fact -- invalidating, never deleting, so every fact keeps provenance and a supersession chain. A hybrid read path fuses dense, lexical, graph, and recency/salience signals, applies a point-in-time ("as-of") filter, and assembles a compact, provenance-tagged context. On the full 500-question LongMemEval_S, graded by the official category-specific judge, Engram's lean configuration -- answering from a ~9.6k-token retrieved slice, never the full history -- scores 83.6% vs. 73.2% for full-context (+10.4 points, McNemar p < 10^-6) at ~8x fewer tokens (9.6k vs. 79k), with 0/500 errored. The gain needs a hybrid read path: facts alone lose recall, while facts plus retrieved chunks recover detail. We also contribute a neutral, in-repo evaluation harness with the official judge baked in and the full-context baseline in every table, publish the raw per-question logs, and document the measurement-integrity pitfalls (truncation, home-grown judges, full-history leaks) that silently distort memory benchmarks. Every number ships with a command to reproduce it.

## 2. 论文提出的方法论

### 核心思想
- **双进程、双时态记忆引擎Engram**：将记忆系统划分为快速写入路径（System-1）和异步固化路径（System-2），两者共同维护一个双时态数据模型，确保任意事实均保留“有效时间”和“记录时间”双重时间轴。
- **混合读取与冲突消解**：检索时融合稠密语义、词汇、图谱和新近度/显著度等多种信号，并对冲突事实采用“废止而不删除”的策略，最终生成紧凑、带来源追溯的上下文。

### 关键技术细节

1. **双时态数据模型**
   - **片段（Episode）**：原始无损轮次/事件，带事件时间和摄入时间。
   - **事实（Fact）**：原子三元组（主语，谓语，宾语），携带有效时间区间（valid_at, invalid_at）、交易时间区间（created_at, expired_at）以及来源片段ID和取代指针（supersedes）。被矛盾的事实仅被废止（invalid_at），永不硬删除，完整保留起源和取代链。
   - **实体/关系**：图谱节点和边同样携带双时态戳记。

2. **System-1：热写入路径**
   - 运行时接收消息，**不调用任何LLM**，直接追加无损片段，进行身份解析（跨会话/设备关联用户），计算轻量嵌入，并将片段送入固化队列。延迟控制在50毫秒以内，保证代理写入非阻塞。

3. **System-2：异步固化路径**
   - 从片段中提取原子事实（默认基于规则，配置LLM后支持模型提取）。
   - 构建双时态知识图谱，检测冲突，应用**“廉价-升级”式冲突消解**：
     - 精确槽位匹配或嵌入相似度判定为可能的更新；
     - 若新旧事实明确矛盾且时间有序，直接设置 `old.invalid_at = new.valid_at` 并建立 `new.supersedes` 指针，**无LLM调用**；
     - 仅在真正模糊的矛盾时升级到LLM裁定。
   - 计算显著度并应用衰减/强化，使未被强化的记忆随时间淡出。

4. **混合读取路径**
   - 对查询进行意图理解，多跳问题分解为子查询。
   - 并行检索四条通道：稠密语义（嵌入余弦相似度）、BM25词汇、图谱N跳邻近、新近度/显著度。
   - 使用**加权互惠排名融合（RRF）** 合并排序列表，并可选经交叉编码器重排。
   - 应用**双时态“截至”过滤器**：仅返回在给定时间点有效的事实。
   - 经过**弃权闸门**（无证据时拒答），组装去重、带来源信息、限制token预算的上下文。上下文混合包含冲突消解后的双时态事实、最相关原始会话块以及会话级摘要。

5. **评分公式**
   对于候选项，其综合得分：
   \[
   score(item|q) = w_{sem} \cos(q, item) + w_{lex} bm25(q, item) + w_{graph} prox(item) + w_{rec} e^{-\Delta t/\tau} + w_{sal} sal(item)
   \]
   权重通过在评测工具上调优确定，默认值为：\(w_{sem}=1.0, w_{lex}=0.6, w_{graph}=0.8, w_{rec}=0.3, w_{sal}=0.25\)，RRF常数 \(k=60\)。

### 算法流程概览
1. 写入：新消息到达 → 无损追加片段 → 身份链接 → 嵌入 → 异步排入固化队列。
2. 固化：异步线程拉取片段 → 提取事实 → 图谱构建与冲突检测 → 非破坏性废止 → 显著度衰减。
3. 读取：接收查询 → 分解（若多跳）→ 四通道并行检索 → RRF融合 → 时间点过滤 → 弃权判断 → 组装混合上下文（事实 + 块 + 摘要）→ 返回。

## 3. 实验设计

- **判分器**：严格使用LongMemEval官方类别特定判分提示，模型为DeepSeek-V3.2，确保与排行榜可比，并纠正了自制判分器导致的得分偏差。

### 对比系统
- **Engram Lean（engram_lean）**：论文主推配置，仅从约9.6k token的检索混合切片（冲突消解事实 + 相关块 + 摘要）生成答案，绝不访问完整历史。

- **Engram Full（engram_full）**：在完整历史之上添加冲突消解事实，用于消融分析（证明事实本身携带全部准确率增益，全历史仅增加token）。

## 6. 论文的主要结论与发现

1. **精简检索上下文在准确率上击败全量上下文**：Engram Lean以83.6% vs. 73.2%（+10.4点）超越基线，token数减少约8倍，实现“更少上下文，更高准确率”。这一现象与“迷失在中间”的干扰机制一致。
2. **混合读取是准确率增益的必要载体**：纯事实路径丢失召回率，只有将冲突消解的双时态事实与原始会话块结合，才能恢复细节并得到最优结果。
3. **双时态建模在特定类别优势显著**：知识更新类准确率87.5%，时间推理类81.1%，均受益于废止链和“截至”过滤。弃权准确率86.7%，表明系统在无答案时拒答而非幻觉。
4. **可复现的中立评测工具是社区亟需的贡献**：通过内置官方判分器、全上下文基线、公开日志，论文披露了测量完整性陷阱（截断、自制判分器、历史泄露），提高了评测可信度。
5. **系统设计上的效率优势**：随着历史增长，Engram的成本趋于平稳，而全量上下文无法扩展。

## 7. 优点

- **方法创新**：
  - 双进程架构（快写慢固）避免LLM进入关键路径，保证低延迟写入。
  - 双时态模型非破坏性废止提供了完整的审计追溯和时间点查询能力。
  - 混合检索融合多信号，较好地平衡了召回与精度。
  - 廉价-升级式冲突消解在常见情况下无需LLM调用，大幅降低成本。
- **实验亮点**：
  - 官方判分器严格遵照原文，修复了自制判分导致的不可比问题。
  - 测量完整性陷阱的详细文档化对社区具有警醒和指导意义。
  - 所有结果及统计分析均可从公开日志复现，每表附有可执行命令，透明度极高。
  - 提供了零依赖离线回退，方便验证和自托管。

## 8. 不足与局限

- **实验覆盖面有限**：仅在一个基准（LongMemEval S）、一种回答模型上完成主评测，缺乏跨基准（如LOCOMO）和跨模型的结果，泛化性待考。
- **缺失关键消融实验**：论文虽讨论了混合读取的必要性（事实 vs. 事实+块），但未提供控制性全量消融数据；也未量化单次运行的随机方差。这些直接影响结论的稳健性评估。
- **尚未攻克的部分类别**：多会话聚合和偏好类别准确率偏低（79.3%，73.3%），仍是性能瓶颈，目前缺乏针对性的解决方案验证。
- **算力和成本细节不透明**：虽然消除了训练算力需求，但云端API调用频率、累积成本和延迟波动未细致报告，对于实际落地参考不足。
- **可扩展性限制**：零依赖回退虽便于测试，但在大规模生产环境可能因向量库、图数据库的插拔复杂性而面临工程挑战。

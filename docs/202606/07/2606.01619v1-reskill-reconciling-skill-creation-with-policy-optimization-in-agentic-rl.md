## 摘要
智能体强化学习（RL）使LLM智能体能够从环境奖励中持续改进，但产生的策略并不会系统地积累可跨任务泛化的可复用策略。模块化技能可以提供此类可复用策略，然而现有的技能增强RL方法将技能创建与策略优化解耦，存在采纳与不断演进的策略相冲突的技能的风险。受Anthropic的Skill Creator启发，我们提出ReSkill，一个RL-in-the-loop的技能创建框架，使技能演化与策略学习相协调。ReSkill利用GRPO的分组结构，以极小的额外开销自然地嵌入三种机制：（1）一个基于断言的技能创建器，从过去经验中诊断失败并提出条件化的、基于触发器的技能修订；（2）组内展开采样，实现对技能版本的受控比较，捕捉哪个版本最能支持策略的持续学习；（3）带自适应折扣的汤普森采样，在策略演化过程中平衡技能版本选择的探索与利用。在多个领域中，ReSkill始终优于现有的基于记忆和技能的方法，尤其在未见任务上增益最大。对技能生命周期的分析显示，技能随着策略改进被自动创建、测试、精炼和剪枝，展示了协调的技能-策略协同演化。

## Abstract
Agentic reinforcement learning (RL) enables LLM agents to improve continuously from environment rewards, yet the resulting policies do not systematically accumulate reusable strategies that generalize across tasks. Modular skills can provide such reusable strategies, yet existing skill-augmented RL methods decouple skill creation from policy optimization, risking adopting skills that conflict with the evolving policy. Inspired by Anthropic's Skill Creator, we introduce ReSkill, an RL-in-the-loop skill creation framework that reconciles skill evolution with policy learning. ReSkill exploits the group-wise structure of GRPO to naturally embed three mechanisms with only marginal additional overhead: (1) an assertion-driven skill creator that diagnoses failures from past experience and proposes conditional, trigger-based skill revisions; (2) within-group rollout sampling that enables controlled comparison of skill versions, capturing which version best supports the policy's ongoing learning; and (3) Thompson Sampling with adaptive discounting to balance exploration and exploitation in skill version selection as the policy evolves. Across several domains, ReSkill consistently outperforms existing memory and skill-based RL methods, with the largest gains on unseen tasks. Analysis of the skill lifecycle shows skills being automatically created, tested, refined, and pruned as the policy improves, demonstrating reconciled skill-policy co-evolution.

## 1. 研究动机与核心问题
- **核心问题**：当前的智能体强化学习（RL）虽然能使LLM通过环境奖励不断提升，但**学到的策略无法系统性地积累可跨任务泛化的可复用策略**。
- **现有方案局限**：模块化技能（skills）可提供可复用策略，但已有的技能增强RL方法**将技能创建与策略优化分离**，存在**采纳与不断进化的策略相冲突的技能**的风险，导致学习效果受损甚至倒退。
- **整体含义**：论文提出**ReSkill**框架，**将技能创建过程嵌入策略优化循环（RL-in-the-loop）**，使两者协同演化，从而让技能能够动态适配策略的进化，实现更好的泛化与性能。

## 2. 方法论

### 2.1 核心思想
- 利用**GRPO（Group Relative Policy Optimization）的分组结构**，在每一次训练步骤中，自然实现对多个技能版本的**受控对比测试**，且**不增加额外的展开预算**。
- 每次展开同时服务于三个目的：策略优化、经验收集与技能评估。

### 2.2 关键技术组件

#### a) RL训练与组内技能测试（Within-Group Skill Testing）
- 每个GRPO组的G条轨迹共享同一任务q和同一策略πθ，**仅技能版本Sv_i不同**。
- 通过从**汤普森采样后验分布中采样**决定每条轨迹使用的技能版本（新版本Snew或旧版本Sold）。
- 优化目标扩展为：
  \[
  J_{\text{RESKill}}(\theta) = \mathbb{E}_{q, v_i \sim \text{TS}(\alpha_v,\beta_v), \tau_i \sim \pi_{\theta_{\text{old}}}(\cdot|q,\mathcal{S}_{v_i};\mathcal{T})} \left[ \mathcal{L}_{\text{GRPO}}(\theta;\{\tau_i\}_{i=1}^{G}) \right]
  \]

#### b) RL-in-the-Loop 技能创建管道
- **经验池（R）**：持续收集成功/失败轨迹对，并按版本标记，为技能创建提供原始素材。
- **条件化技能加载**：每个技能包含触发条件（trigger）和内容（适用场景、动作指导、反例）；触发条件可以是“general”、“beginning”或“action_pattern”（如正则匹配）。
- **断言驱动失败分级**：维护一套动态断言集A（规则谓词），对池中所有轨迹计算通过率，LLM分析器据此输出失败诊断，并更新断言集。
- **历史信息驱动的技能修订**：技能创作者结合当前技能库、失败诊断、历史提案与接受/拒绝结果，给出**ADD / MODIFY / DELETE**操作，形成候选新版本Snew。
- **触发验证**：对新提出的action_pattern触发器估计激活率，若不满足阈值则拒绝提案并重试。

#### c) 基于汤普森采样与自适应折扣的技能版本选择
- **技能版本分配**：将新旧版本视为两个Beta臂，初始化Beta(1,1)，每个训练步根据 **Pr(p_new > p_old)** 计算π_t(new)，以ε_ts保证探索下限。分配概率随证据累积而自适应偏移。
- **自适应折扣后验更新**：由于策略持续演化，早期证据逐渐失效。引入折扣权重：
  \[
  \alpha_v \leftarrow w_t \cdot \alpha_v + m_t, \quad \beta_v \leftarrow w_t \cdot \beta_v + (n_t - m_t), \quad w_t = \frac{1}{1 + n_t/M}
  \]
  M是记忆参数，通过最大化序列预测似然从数据中估计，无需手动调参。
- **接受/拒绝决策**：在每个进化周期结束时，比较两版本后验均值，若E[p_new] > E[p_old]则接受，否则恢复旧版本。

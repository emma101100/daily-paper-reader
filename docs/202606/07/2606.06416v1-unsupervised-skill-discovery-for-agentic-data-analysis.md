## 摘要
推理时技能增强通过注入可重用的过程知识，在不更新模型参数的情况下，提供了一种轻量级的方法来改进数据分析智能体。然而，发现有效的数据分析技能仍然具有挑战性，因为可靠的监督成本高昂，且成功标准因分析格式而异。这引出了一个关键问题：如何仅从无标签探索中发现可重用的数据分析技能。我们提出了DataCOPE，一个面向数据分析智能体的无监督验证器引导的技能发现框架。DataCOPE从探索轨迹中导出验证信号，并使用它们来表征轨迹间的相对质量或一致性。它迭代协调用于轨迹生成的数据分析智能体、用于信号提取的无监督验证器以及用于对比技能蒸馏的技能管理器。对于报告式分析，我们将验证器实例化为自适应检查表验证器，该验证器导出任务特定的标准，根据可验证的覆盖率对报告评分，并迭代细化检查表。对于推理式分析，我们将其实例化为答案一致性验证器，该验证器按答案一致性对轨迹进行分组，并使用自一致性作为辅助信号。我们在Deep Data Research的报告式分析和DABStep的推理式分析上评估DataCOPE。在这两种设定下，DataCOPE始终比基线方法提升了留出法性能。在四个模型设定的平均下，DataCOPE在报告式任务和推理式任务上分别将平均得分提升了9.71%和32.30%。

## Abstract
Inference-time skill augmentation provides a lightweight way to improve data-analytic agents by injecting reusable procedural knowledge without updating model parameters. However, discovering effective skills for data analysis remains challenging, as reliable supervision is expensive and success criteria vary across analytical formats. This raises the key question of how to discover reusable data-analysis skills from unlabeled exploration alone. We propose DataCOPE, an unsupervised verifier-guided skill discovery framework for data-analytic agents. DataCOPE derives verifier signals from the exploration trajectories and uses them to characterize relative quality or aggreement among trajectories. It iteratively coordinates a Data-Analytic Agent for trajectory generation, an Unsupervised Verifier for signal extraction, and a Skill Manager for contrastive skill distillation. For report-style analysis, we instantiate the verifier as an Adaptive Checklist Verifier that derives task-specific criteria, scores reports by verifiable coverage, and iteratively refines the checklist. For reasoning-style analysis, we instantiate it as an Answer Agreement Verifier that groups trajectories by answer agreement and uses self-consistency as an auxiliary signal. We evaluate DataCOPE on report-style analysis from Deep Data Research and reasoning-style analysis from DABStep. Across both settings, DataCOPE consistently improves held-out performance over baselines. Averaged across four model settings, DataCOPE improves the mean score by 9.71% and 32.30% on report-style and reasoning-style tasks respectively.

## 二、方法论

### 1. 总体框架
DataCOPE 采用闭环迭代结构，协调三类组件：

- **数据分析智能体（πθ）**：在任务上采样探索轨迹。
- **无监督验证器（ϕ）**：从轨迹中提取非特权信号，组织为对比组。
- **技能管理器（ψω）**：对轨迹组进行对比蒸馏，生成或更新技能文件（SKILL.md 及辅助资源）。

形式上将任务建模为 POMDP。智能体通过 ReAct 范式交替生成思考（thought）与动作（action），产生的历史轨迹记为 \( h_t \)，最终形成轨迹 τ 和答案 y。

技能 S 被定义为 `(M, R)`，其中 M 是 Markdown 根文档（含何时使用、策略、失败模式等），R 是对应资源集合。技能注入后，智能体行为变为 \(\pi_\theta(\cdot|h_t, S)\)。

### 2. 报告式任务的 Adaptive Checklist Verifier
- **任务特定清单生成**：由 **清单智能体** 为每个任务自动生成一组可核查的检查项（checklist）\( C_u \)。
- **报告评分**：对每条报告 \( y_{u,i}^{(r)} \) 计算满足检查项的比例，作为无监督质量信号 `qu,r`。
- **报告端技能演化**：按平均得分将轨迹和对应清单分为高分组 \( G^{(r)}_+ \) 和低分组 \( G^{(r)}_- \)，技能管理器通过对比蒸馏更新报告生成技能 \( S^{(r+1)}_\pi \)。
- **对比清单精炼**：当报告平均得分开始下降时，冻结报告技能，逆向对比高低分组来更新清单生成技能 \( S^{(r'+1)}_\phi \)。清单精炼与报告技能更新交替进行，以避免智能体过拟合静态清单。

### 3. 推理式任务的 Answer Agreement Verifier
- **答案聚类**：对同一任务的 N 条轨迹，按最终答案进行聚类，使用类型等值判定（如 exact match）。
- **自一致性估计**：以某答案簇的相对大小作为自一致性分数（SC），仅作为辅助不确定性信号。
- **智能体端技能演化**：
  - 每组答案簇选出一代表性轨迹（优先选择交互轮次少、异常少的轨迹）。
  - 过滤掉 SC 分数饱和的轨迹，聚焦尚不稳定的案例。
  - 技能管理器对比不同簇的代表轨迹，识别分歧行为和失败模式，更新推理技能。
  
### 4. 关键公式化流程
- 每轮迭代生成轨迹集合 \( T^{(r)} \)，验证器提取信号 \( \sigma_{u,r} \) 并组织分组 \( G^{(r)} \)。
- 技能更新：\( S^{(r+1)} = \psi_\omega(S^{(r)}, G^{(r)}) \)。
- 目标函数：在无监督探索集上发现技能 \( \hat{S} \)，使其在留出测试集 \( D_{test} \) 上最大化性能 \( J(S; \pi_\theta, D_{test}) \)，但发现过程中不接触任何 ground truth。

## 六、主要结论与发现

- DataCOPE 在**完全无人工标注**的条件下，能发现可迁移的数据分析技能。
- 在报告式任务上，平均 Overall Avg. 从 47.39% 提升至 57.10%（+9.71%），所有基座模型均获得一致提升。
- 在推理式任务上，DABStep 全难度准确率从约 29.14% 升至 61.44%（+32.30%），尤其对 Hard 子集提升显著。
- 所发现技能具有**跨模型迁移性**，对 Claude、GPT、DeepSeek、Qwen 等不同架构的模型均奏效。
- 相比 Skill Creator 基线，无监督验证信号是技能质量提升的关键，单纯基于原始轨迹的技能构建不够充分。
- 迭代精炼虽有效但非单调，后期轮次可能收益递减甚至退化，清单分数可作为早停信号。

## 七、优点

- **完全无监督**：无需任何 ground truth 答案、成功标注或人工反馈，仅利用智能体自身探索信号。
- **任务自适应**：针对报告式和推理性两种不同类别的分析任务，设计了截然不同但均不需要监督信息的验证器，体现了框架的灵活性。
- **闭环协同提升**：报告式任务中交替优化报告技能和检查表技能，有效缓解检查表过拟合，同时提升报告的全面性。
- **多模型验证**：在 5 个以上的前沿大模型上统一评估，结论普适性强。
- **细粒度分析丰富**：迭代动态、粒度影响、智能体替换、标签效率、成本等均做了系统分析，解释性强。
- **技能形式规范**：采用 Skill.md + 资源文件的模块化设计，便于集成到现有 agent 系统中。

## 八、不足与局限

- **迭代过程不稳定**：部分数据集和模型下后期迭代反而使性能下降，缺乏自适应终止机制，实际部署可能需要人工判断或额外停止准则。
- **依赖强基座模型**：技能管理器和清单智能体本身依赖高性能 LLM，若底层模型能力不足，无监督验证的质量可能迅速恶化。
- **实验覆盖有限**：
  - 仅评估了两个基准（Deep Data Research 和 DABStep），未涉及更多样化的数据分析形式（如 SQL 问答、可视化任务、时间序列预测等）。
  - 未讨论技能发现对真实用户交互式分析场景的适应性。
- **计算开销未充分量化**：虽给出了 token 节省数据，但未报告发现全程的总 API 调用成本或时间成本，对于实际落地决策的参考度有限。

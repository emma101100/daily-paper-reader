## 摘要
现代大语言模型智能体越来越依赖技能库来处理复杂任务，使得技能进化成为自我提升的主要驱动力。然而，孤立的单用户任务流缺乏构建全面技能所需的多样性。虽然跨用户协作可以克服这一数据瓶颈，但当前基于轨迹共享的方法会损害用户隐私，并强加一个无法适应客户端异质性的统一全局技能库。我们提出了FederatedSkill，一个保护隐私的协作式智能体进化框架。FederatedSkill不再直接共享原始轨迹，而是将语义技能差异（即针对本地库的结构化补丁）作为基本的通信单元。在服务器端，一个进化智能体聚合这些补丁，动态地建模客户端特定的能力边界，从而促进严格个性化的技能进化，而非次优的全局平均。在20个不同的智能体任务族上进行评估，FederatedSkill相较于自我进化基线展现出显著提升，成功率最高提升44.4%，计算成本降低37.5%。

## Abstract
Modern LLM agents increasingly rely on skill libraries to handle complex tasks, making skill evolution a primary driver of self-improvement. However, isolated single-user task streams lack the diversity required to build comprehensive skills. While cross-user collaboration can overcome this data bottleneck, current trajectory-sharing approaches compromise user privacy and impose a uniform global library that fails to accommodate client heterogeneity. We introduce FederatedSkill, a privacy-preserving framework for collaborative agent evolution. Moving beyond raw trajectory sharing, FederatedSkill utilizes semantic skill diffs, structured patches over local libraries, as the fundamental unit of communication. On the server side, an evolution agent aggregates these patches to dynamically model client-specific capability boundaries, facilitating strictly personalized skill evolution rather than a suboptimal global average. Evaluated across 20 distinct agent task families, FederatedSkill demonstrates substantial gains over self-evolving baselines, achieving up to a 44.4% increase in success rate and a 37.5% reduction in computational cost.

## 2. 方法论
### 2.1 核心思想
FederatedSkill借鉴联邦学习范式，但通信的基本单元不是模型参数，而是**语义技能差异（semantic skill diffs）**——一种对本地技能库的结构化补丁（包含增、改、删操作），从而避免敏感轨迹外泄。服务器端运行进化智能体，动态推断各客户端的能力边界，生成个性化的库更新，实现从“全局平均”到“严格个性化”的转变。

### 2.2 关键技术细节
- **客户端执行与补丁蒸馏**：
  - 客户端代理在本轮任务分布上采样任务并执行，产生轨迹批次 \(B_t^i\)。
  - 利用本地反思过程 \(g_i\) 将轨迹提炼为**技能补丁** \(\delta_t^i = (U_t^i, D_t^i, R_{i,x}(\tau), s_t^i)\)，包含：
    - \(U_t^i\)：待新增/修改的技能文件（结构化文档与脚本）。
    - \(D_t^i\)：待删除的库路径。
    - \(R_{i,x}(\tau)\)：任务验证奖励。
    - \(s_t^i\)：补丁理由的自然语言摘要。
  - 补丁仅包含可复用、可泛化的语义内容，**严禁携带任务特定值、ID 或一次性输出**，路径经过严格验证以防目录遍历攻击。
  - **执行智能体的策略冻结**，性能提升完全来自技能库的进化。
- **服务器端个性化进化**：
  - 问题形式化为**部分可观测马尔可夫决策过程（POMDP）**，服务器进化智能体 \(M\) 仅观察 \(\{(\rho_i, \delta_t^i)\}\)，需动态推断客户端能力边界。
  - 进化分两阶段：
    1. **进化规划**：基于本轮补丁构建**能力矩阵** \(C_t\)（行=任务工作流，列=客户端，状态：covered/absorbing/broken/gap）；维护**双层记忆**（家族级共享记忆记录全局未解工作流，客户端私密记忆记录模型特定的失败模式）；生成**进化指令**，指定针对当前缺口应执行的动作（添加/修复/重构）。
    2. **库进化执行**：按指令处理各客户端补丁，选择性吸收经过检验的同级技能，或为了对齐客户端模型风格而重写同级的技能逻辑，输出个性化更新 \(\Delta_t^i\)，并生成可审计决策日志。
  - 能力矩阵与双层记忆构成隐状态，持续追踪异质性边界，确保库既协作前进，又保持个性。

### 2.3 算法流程概览
1. 各客户端在本地用现有技能库执行任务，获得轨迹与奖励。
2. 客户端蒸馏轨迹为**语义技能补丁**，上传至服务器。
3. 服务器汇总各客户端补丁及能力档案，进化智能体更新能力矩阵与双层记忆，形成进化计划。
4. 服务器为每个客户端独立生成**个性化库更新文件**，分发回各个客户端。
5. 循环多轮，技能库持续个性化进化。

## 6. 主要结论与发现
- **性能显著提升**：在 SkillFlow 20 个任务族上，FederatedSkill 相比自我进化，成功率平均提升最高达 44.4 个百分点，计算成本降低最高达 37.5%。
- **异质性适应力强**：即便在客户端的骨干模型和智能体工具套件均不同的极端异质设置下，仍能实现平均约 4.82 个百分点的提升；个性化进化相比统一全局库可额外提升 9.8–12.2 个百分点。
- **通信效率极高**：全基准测试总补丁传输量仅约 3.5 MB，比传输模型参数小几个数量级，适合边缘网络场景。
- **隐私保护切实有效**：语义补丁将敏感实体泄漏率降至 5.08%，分发回客户端的个性化更新进一步降至 0.09%，与无关任务的补丁泄漏率无统计学差异；对合成 PII 测试零泄漏。
- **技能库质量优化**：FederatedSkill 有效抑制了库膨胀或库停滞，通过去重和保守接纳维持精干库，提升检索准确率。

## 7. 优点
- **范式创新**：将联邦学习理念迁移到技能级语义交换，首次实现隐私保护下跨模型、跨工具套件的智能体技能协同进化。
- **隐私与效用的优雅平衡**：通过补丁蒸馏中的抽象指令、严格的路径校验和独立的语义泄漏审计，在几乎不泄露敏感信息的同时大幅提升协作收益。
- **真正个性化**：服务器进化智能体动态建模每个客户端的能力边界，避免一刀切的全局库，这是对传统联邦学习的重要补充。
- **全面的评估体系**：涵盖成功率、成本、通信量、隐私泄漏率，并通过多设置、消融、统计检验、案例解剖等方式确保结论扎实。
- **可解释性与可审计性**：补丁为纯文本、结构化，决策日志记录详细，便于追溯和人工审查，增强实用可信度。

## 8. 不足与局限
- **规模受限**：由于计算资源限制，实验仅在 3 个客户端上进行，未能验证几十或上百客户端下的扩展性和涌现行为。
- **假设可信环境**：当前框架假设联邦中各参与方均诚实可信，未考虑恶意客户端上传中毒补丁的攻击防御；对抗鲁棒性是待解问题。
- **负向迁移风险**：实验观察到少数任务上出现性能倒退（例如跨模型补丁合并导致流程过于刚性），这提示个性化合并技术仍有优化空间。
- **隐私保护基于经验验证**：虽经过多重审计，但隐私保障并非基于严格的差分隐私等密码学理论，可能被低估风险，在极端敏感数据场景下需额外加密层。
- **系统优化未深入**：带宽高效的补丁传输模式虽有初步分析，但未对压缩、增量同步等系统层面做进一步优化。

## 摘要
大型语言模型智能体越来越依赖技能来编码过程性知识，但高质量技能的手动编写成本仍然很高。本文研究从异构交互证据（包括演示、智能体轨迹、工具痕迹和执行日志）中自动构建技能。我们认为从痕迹到技能的构建并不是简单的摘要任务，因为痕迹是碎片化的、冗余的，并且可能遗漏罕见但安全关键的行为。为了解决这个问题，我们引入了 RWSA，一种面向工作流的中间表示，将技能分解为工作流结构、执行语义和运行时附件，捕捉任务分解、控制流、验证、安全、回滚和状态管理。基于 RWSA，我们提出了 W2S 框架，该框架分割痕迹，归纳局部技能草案，对齐共享结构，协调分支，并在保留证据和置信度注释的同时压缩冗余。在 70 个技能上的实验表明，与基于摘要和提示的基线相比，W2S 将行为重放一致性提高了 10.5%，这强调了需要将痕迹视为可执行的运行时规范而非可压缩文本。

## Abstract
Large language model agents increasingly rely on Skills to encode procedural knowledge, yet high-quality Skills remain costly to hand-write. This paper studies automatic Skill construction from heterogeneous interaction evidence, including demonstrations, agent trajectories, tool traces, and execution logs. We argue that trace-to-skill construction is not simple summarization tasks, because traces are fragmented, redundant, and may miss rare but safety-critical behaviors. To address this, we introduce RWSA, a workflow-oriented intermediate representation that decomposes Skills into Workflow structure, execution Semantics, and runtime Attachments, capturing task decomposition, control flow, verification, safety, rollback, and state management. Building on RWSA, we propose W2S, a framework that segments traces, induces local Skill drafts, aligns shared structures, reconciles branches, and compresses redundancy while preserving evidence and confidence annotations. Experiments on 70 Skills show that W2S improves behavioral replay consistency by 10.5% over summarization- and prompting-based baselines, highlighting the need to treat traces as executable runtime specifications rather than compressible text.

## 2. 论文提出的方法论

### 核心思想：从痕迹到技能的结构化归纳（W2S）

- 将技能视为**路由头部 + 运行时规范**的组合，运行时规范进一步分解为**工作流主干（Workflow）、执行语义（Semantics）、运行时附件（Attachments）**，简称 WSA 组件。
- 技能构建过程应从痕迹中恢复这些 WSA 组件，再基于组件生成结构化的技能文本，而非直接进行端到端的摘要。

### 关键技术细节：WSA 中间表示（Skill-IR）

- **路由头部（R）**：控制技能何时被选择（如触发条件、用途描述）。
- **工作流主干（W）**：由节点集 N 和边集 E 构成，捕捉执行骨架（多步执行流）。
- **执行语义（S）**：为每个节点赋予局部目标、决策逻辑、成功/失败准则等，赋予节点可执行的意义。
- **运行时附件（A）**：全局和节点级的资源、工具、约束、校验、输出 Schema 等操作边界。
- 技能类型被划分为 8 种（T0~T7），从无工作流的纯提示片段到包含完整 W+S+A 的运行时工作流。

### 算法流程（W2S 框架）

1. **痕迹到 WSA 证据解析**：将交互痕迹分解为三种证据——工作流证据（EW）、语义证据（ES）、附件证据（EA），并记录证据的来源性（直接观察、推断、未观测）。
2. **WSA 重建**：
   - 工作流组件重建为节点 bN 和边 bE（单节点技能则不产生边）。
   - 语义层以节点为索引，映射目标、准则、结果等。
   - 附件分为全局和节点级进行作用域绑定。
3. **WSA 约束下的技能生成**：基于重建的 WSA 组件，遵循三条规则生成技能文本：
   - 每条主要指令需可追溯到证据；
   - 分支、异常路径应明确保留，不能泛化；
   - 不确定性应明确标注（不能虚构细节）。
4. **反馈精炼**：
   - 覆盖度检查：比对 WSA 组件和证据，标记缺失路径/准则/附件。
   - 一致性检查：检测工作流与附件之间的矛盾。
   - 可执行性检查：确保步骤有序、决策准则到位、验证要求明确。
   - 针对性修复：根据错误类型（W/S/A 级）返回反馈，迭代修订直至没有高优先级错误或达到预算。

该流程最终输出一个经过解析且行为接近参考技能的技能文档。

## 3. 实验设计

- **数据集**：**WSASkill**，一个基于 WSA 分类法构建的基准，包含 70 个参考技能，覆盖 8 种技能类型（T0~T7）。每个技能提供 WSA 配置文件、工作流（如适用）、以及用于重放评估的场景（覆盖不同路径、决策和附件用例）。
- **输入证据**：对于每个工作流路径，收集 10 条真实交互痕迹作为技能重建的输入。
- **对比方法**：**Anthropic Skill Creator (ASC)**，一个基于访谈和草案的结构化技能构建工具，目标同样是基于交互证据生成可复用的技能文件。

## 6. 论文的主要结论与发现

- 将技能构建任务从文本摘要重新定义为 **结构化运行时规范恢复**，是提升技能可执行性的关键。
- 基于 WSA（路由-工作流-语义-附件）的中间表示能有效分离结构、行为和操作依赖，使技能重建过程有据可依。

## 摘要
大型语言模型（LLMs）展现出作为工具使用智能体的潜力，但在需要记忆、组织和重用知识的长期任务中仍存在局限。先前的记忆方法旨在解决这一问题，但主要关注事实信息的存储。近期关于程序性记忆的工作改善了任务重用，但往往退化为重放过去的成功经验，未能应对失败案例或实现在线可扩展性。我们提出了一个统一且自动的记忆框架，将语义记忆、情节记忆和程序性记忆集成在一个结合短期和长期存储的双层设计中。由行动者、记忆和评论者智能体构成的多智能体架构能够实现自动记忆生成、奖励标注和自适应检索。长期记忆通过基于奖励的评估、合并和剪枝进行管理，确保可扩展性和持续改进。在多种环境下的实验表明，与现有基线相比，我们的方法提高了在长多轮任务上的鲁棒性和成功率。该工作凸显了全面、自适应的记忆对于推进基于LLM的智能体的重要性。

## Abstract
Large Language Models (LLMs) show promise as tool-using agents but remain limited in long-horizon tasks that require remembering, organizing, and reusing knowledge. Prior memory approaches aim to resolve the situation, but mainly focus on storing factual information. Recent work on procedural memory improves task reuse, yet often reduces to replaying past successes without addressing failure cases or online scalability. We introduce a unified and automatic memory framework that integrates semantic, episodic, and procedural memory in a bi-level design combining short-term and long-term stores. A multi-agent architecture with actor, memory, and critic agents enables automatic memory generation, reward annotation, and adaptive retrieval. Long-term memory is managed through reward-based evaluation, merging, and pruning, ensuring scalability and continual improvement. Experiments across various environments show that our approach improves robustness and success on long multi-turn tasks compared to existing baselines. This work highlights the importance of comprehensive, adaptive memory for advancing LLM-based agents.

### 2. 方法论

#### 2.1 核心思想与架构
- 将智能体决策过程形式化为**部分可观测马尔可夫决策过程**（POMDP），状态空间 \(S = S_{\text{short}} \times S_{\text{long}}\) 分别对应短期记忆（STM）和长期记忆（LTM）。
- 系统由三个智能体协同构成：
  - **行动者智能体（Actor）**：利用 STM + 检索到的 LTM 与环境交互求解任务，每轮更新 STM，并在任务结束时清空，信息编码进入 LTM。
  - **长期记忆智能体（Memory Agent）**：管理三种记忆的存储、检索、合并与剪枝。
  - **评论者智能体（Critic Agent）**：压缩原始记忆素材，为程序式记忆标注指导信息和奖励信号。

#### 2.2 记忆类型与生成流程
- **语义记忆（\(M_S\)）**：存储环境与任务的事实性和一般性知识，由 LLM 总结生成。
- **情景记忆（\(M_E\)）**：对过去交互事件的摘要，同样通过 LLM 总结。
- **程序式记忆（\(M_P\)）**：从过去的动作与结果中提炼的决策指导。生成流程包含：
  1. 行动者执行动作时，必须明确声明**预期目的/结果** \(\hat{o}_{t+1}\)。
  2. 评论者缓存半成品记忆 \(k = (c_t, a_t, \hat{o}_{t+1}, M_t)\)，待收到后续环境观测后，对比实际结果与预期，生成**奖励 \(r_k \in \{0,1\}\)** 与**反思 \(f_k\)**，最终形成程序式记忆条目 \((c_k, a_k, f_k)\) 存入长期记忆。
  3. 为支持多步动作的评估，行动者可调用**子目标规划工具**（如 `add subgoal`、`conclude subgoal`），使得子目标内的所有观测连贯地归属到同一条记忆上，实现类似搜索树的探索与上下文作用域管理。

#### 2.3 短期记忆（STM）的上下文压缩
- 采用**类程序运行栈**的方法：当前栈包含待实现的子目标及高层次计划，子目标外的历史轮次被 LLM 摘要化。这种栈结构使行动者聚焦当前子目标，完成后再恢复上层目标，有效控制上下文长度。

#### 2.4 长期记忆的奖励驱动管理
- **检索**：语义与情景记忆使用稠密向量相似度检索。程序式记忆检索结合**记忆有效性评估**与上下文相似度：每个记忆条目 \(m\) 维护一个自适应参数 \(v_m \in [0,1]\) 表示其建议的有效性。假设奖励由一个二值随机过程生成：
  \[
  r_t \mid c_t = l_t^{(1)} \vee l_t^{(2)} \vee \cdots \vee l_t^{(n)} \vee l_t
  \]
  其中 \(l_t^{(i)} \sim \text{Bernoulli}(v_{m_t^{(i)}} \cdot \psi(c_t)^\top \psi(c^{(i)}))\)，即条目的有效性和其上下文与当前上下文的相似度共同决定有帮助的概率。通过 EM 算法在线更新 \(v_m\) 的估计，检索时按 \(v_m \, \psi(c_t)^\top \psi(c_m)\) 的 Top-n 选取记忆。
- **管理**：设置频率阈值 \(\epsilon\)，对检索频次过低或总是与另一条目一同被检出的记忆执行**删除**或**合并**，减少冗余。

#### 2.5 算法简述
- **行动者流水线**：接受观测 → 报告给评论者 → 请求记忆智能体检索 → 结合记忆与上下文选择动作并设定预期 → 更新 STM → 将半成品记忆推送给评论者。
- **评论者流水线**：维护内存队列，接收半成品记忆与后续观测，当某个记忆条收集齐所有相关观测后，计算奖励，生成程序式记忆，并将奖励信息连同所依据的检索记忆索引反馈给记忆智能体，用于更新 \(v_m\)。

### 6. 主要结论与发现

- AdMem 在大多数领域**显著提升了任务完成率和平均进度**，尤其在 AlfWorld、PDDL、Tool‑query 等领域提升巨大；在若干记忆不易迁移的领域（如 Science World）仍能**保持与基线相当的性能**，未因额外结构而产生明显负效果。
- 以前的程序式记忆方法（如 AWM）在某些领域引入记忆后反而**损害性能**，表明区分可迁移知识与任务特定知识、平衡记忆依赖与即兴决策是关键挑战。
- 短期规划（栈式上下文管理）有助于提升当前任务进度但不随 horizon 扩展；长期记忆在多趟流式学习（多 epoch）中**持续改善任务表现**，赋予智能体自我进化的能力。
- 基于奖励的多臂赌博机式模型和在线 EM 更新使得长期记忆能够**自适应地评估和筛选有效记忆条目**，实现可扩展的内存管理。

### 7. 优点

- **体系完整统一**：将语义、情景和程序性记忆有机整合，既存入事实和历史，也提炼可操作的决策指导，贴合认知科学分类。
- **双层记忆与多智能体协作**：STP 保持专注、LTP 跨任务复用，行动者、评论者、记忆智能体明确分工，自动化生成与管理记忆，减少人工干预。
- **在线学习与自我进化**：通过奖励反馈和带状模型持续更新记忆有效性，支持跨任务和长程的自我改进。
- **实用的上下文管理**：栈式子目标结构有效控制上下文长度，同时便于生成结构化、有层次的过程性记忆。
- **广泛的实验验证**：在多领域、多任务流式设定下证实了鲁棒性，并通过消融与多 epoch 测试展示了各组件的贡献。

## 摘要
跨会话持久存在的 AI 智能体需要可检索、可审计、可更新和可擦除的记忆。现有记忆系统往往将源证据、提取的事实、检索上下文和回答策略坍缩为一条不透明的提示路径，导致失败难以诊断：错误答案可能源于证据缺失、无依据的提取、状态过时、检索丢失或回答模型行为。我们提出 Eywa，一种围绕“证据先于信念”构建的来源感知记忆架构。Eywa 在推导规范事实之前存储不可变的源证据，依据类型化信号和源支持验证提取的记忆，并通过确定性多路径读取路径检索有界记忆上下文，检索过程中零 LLM 调用。检索到的上下文与回答指令分开返回，使得同一记忆基底可在前沿、经济及本地回答模型上进行评估。在冻结且记录制品的检索配置下，Eywa 在 LoCoMo C1-C4 分割上以 Claude Sonnet 4.6 担任写入和 QA 角色，达到 90.19% 的评判准确度。在 LongMemEval-S 上，检索充分性准确度达 88.2%。在 BEAM（一个 700 问的技术记忆压力基准）上，平均要点得分为 81.45%，得分≥0.5 的通过率达 85.29%。每个问题的完整制品，包括问题、黄金答案、模型答案、检索上下文和标签，已发布在 https://eywa.to/research。

## Abstract
AI agents that persist across sessions need memory they can retrieve, audit, update, and erase. Existing memory systems often collapse source evidence, extracted facts, retrieved context, and answer policy into one opaque prompt path, making failures difficult to diagnose: a wrong answer may come from missing evidence, unsupported extraction, stale state, retrieval loss, or answer-model behavior. We present Eywa, a provenance-grounded memory architecture built around evidence before belief. Eywa stores immutable source evidence before deriving canonical facts, validates extracted memories against typed signals and source support, and retrieves bounded memory context through a deterministic multi-route read path with zero LLM calls inside retrieval. Retrieved context is returned separately from answer instructions, allowing the same memory substrate to be evaluated across frontier, budget, and local answer models. Under a frozen, artifact-recorded retrieval configuration, Eywa reaches 90.19% judge accuracy on the LoCoMo C1-C4 split with Claude Sonnet 4.6 write and QA roles. On LongMemEval-S, it reaches 88.2% retrieval-sufficiency accuracy. On BEAM, a 700-question technical-memory stress benchmark, it reaches 81.45% mean nugget score and 85.29% pass@score >= 0.5. Full per-question artifacts, including questions, gold answers, model answers, retrieved context, and labels, are published at https://eywa.to/research.

## 2. 论文提出的方法论

- **核心思想**：将记忆的生命周期划分为**证据捕获 → 事实形成 → 验证存储 → 检索 → 回答**的清晰阶段，每一阶段保留溯源信息，保证每条规范事实都可回溯到原始证据。
- **关键技术细节**：
  - **Tier 0（不可变捕获）**：用户会话以不可变证据行存储，同时通过基于规则的类型信号检测提取日期、实体、货币值、版本号、URL 等硬锚点。
  - **Tier 1（验证提取）**：LLM 从证据窗口提出候选事实，经过验证函数 \( V \) 检查：
    - \( V_{\text{support}} \)：候选事实与源文本的词法和模糊重叠；
    - \( V_{\text{hard}} \)：硬锚点值是否与 Tier 0 信号匹配；
    - \( V_{\text{subject}} \)：所述主体是否出现在支持文本或已知实体中；
    - \( V_{\text{act}} \)：保存否定和不确定性等记忆行为信号。
    验证通过的事实被标记为规范信念并保留溯源链接，失败候选被拒绝或标记待修复。
  - **提取作为索引而非记忆本身**：提取的规范事实是紧凑、可查询、可修订的索引，真正的权威基底仍是不可变证据存储。
  - **多模式提取**：支持 Canonical（持久化事实）、Observation（高召回事件）、Hybrid（两者结合）、Structured Technical（技术细节）四种模式，以适应不同问题类型。
  - **确定性多路由读取路径**：读取过程不使用 LLM，包含：
    - 向量搜索（余弦相似度）
    - 关键词搜索（BM25 + 查询扩展）
    - 时间范围过滤
    - 实体/图谱遍历
    - 支持通道（原始事件救援、推理支持）
    - 融合采用加权倒数排名融合 (RRF)，并实施实体范围降权、保留地板等后融合过滤。
  - **回答策略分离**：ReadResult 包含独立的 `context` 和 `answer_instructions`，支持 strict、balanced、advanced 三种策略，使同一记忆基底可适配不同能力或安全要求的回答模型。

## 7. 优点

- **架构创新**：明确区分证据、信念、读取、回答策略，并保持溯源链接，使记忆系统具有高度可解释性和可修复性。
- **确定性检索路径**：检索零 LLM 调用，降低推理成本、延迟，使产品应用更经济。
- **模型分离设计**：同一记忆基底可服务于不同能力、成本的回答模型，提升部署灵活性。
- **评估透明度极高**：每个题目提供制品级追踪，覆盖多基准和多种角色配置，严格记录配置与诊断信息，极大增强了可复现性和可信度。
- **现实故障导向**：通过错误分类法将系统级失败具体化，有助于工程优化。

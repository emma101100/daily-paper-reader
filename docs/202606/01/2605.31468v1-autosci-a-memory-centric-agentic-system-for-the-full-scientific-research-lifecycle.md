## 摘要
传统科学研究一直高度依赖人力，需要研究人员在漫长的项目周期中协调文献、想法、实验、手稿和审稿回复。基于 LLM 的科学智能体的出现为自动化这一过程创造了机会。这样的系统必须支持完整的研究生命周期，跨项目维护结构化的持久记忆，并随时间改进其自身的研究流程。然而，现有系统要么部分满足要么不满足这些要求，导致统一的自动化科研系统存在空白。因此，我们提出了 AutoSci，一个面向完整科研生命周期的以记忆为中心的智能体系统。AutoSci 围绕四个模块组织。SciMem 提供模式管控的研究记忆，将用于可复用科学知识的长期知识记忆与用于项目级工件（如想法、实验、手稿和审稿）的主动研究记忆分开。SciFlow 通过一个控制状态、上下文、验证、反馈和编排的框架执行从文献理解到反驳的五阶段生命周期。SciDAG 通过 DAG 形状的多智能体算子和可复用的阶段专用模板增强困难技能。SciEvolve 将来自用户、实验、审稿和外部环境的反馈信号转化为对 SciMem 组织、SciFlow 技能和 SciDAG 模板的版本化更新。这些模块共同使 AutoSci 成为一个能够跨研究项目执行、记忆和进化的持久研究环境。代码库可在 https://github.com/skyllwt/AutoSci 获取。

## Abstract
Scientific research has traditionally been human-intensive, requiring researchers to coordinate literature, ideas, experiments, manuscripts, and review responses across long project cycles. The rise of LLM-based scientific agents creates an opportunity to automate this process. Such a system must support the full research lifecycle, maintain structured persistent memory across projects, and improve its own research procedures over time. However, existing systems either partially satisfy or fail to satisfy these requirements, leaving a gap for a unified automated scientific research system. As a result, we present AutoSci, a memory-centric agentic system for the full scientific research lifecycle. AutoSci is organized around four modules. SciMem provides schema-governed research memory, separating Long-Term Knowledge Memory for reusable scientific knowledge from Active Research Memory for project-level artifacts such as ideas, experiments, manuscripts, and reviews. SciFlow executes a five-stage lifecycle from literature understanding to rebuttal through a harness that controls state, context, verification, feedback, and orchestration. SciDAG augments difficult skills with DAG-shaped multi-agent operators and reusable stage-specific templates. SciEvolve converts feedback signals from users, experiments, reviews, and external environments into versioned updates to SciMem organization, SciFlow skills, and SciDAG templates. Together, these modules make AutoSci a persistent research environment that can execute, remember, and evolve across research projects. The code repository is available at https://github.com/skyllwt/AutoSci.

### 方法论：核心思想与关键技术细节
AutoSci 围绕四个模块构建，各模块分别解决一组核心问题：

- **SciMem（模式管控的研究记忆）**
  - 将记忆拆分为两个区域：
    - **长期知识记忆**（Long-Term Knowledge Memory）：存储跨项目可复用的科学知识，以类型化实体（主题、论文、已确立的基础知识、概念、方法、人员）及双向类型化关系构成可遍历的知识图谱。
    - **主动研究记忆**（Active Research Memory）：跟踪当前项目的动态工件（想法、实验、手稿、审稿），每个实体携带明确的**生命周期状态**（如想法从 `proposed` 到 `validated/failed`；实验从 `planned` 到 `completed`；手稿从 `drafting` 到 `final version`）。
  - 记忆增长通过三条互补路径：长期知识内部的聚合、长期与主动记忆之间的双向流动（激活与固化）、以及跨项目的经验累积。所有写入需通过**信任守卫**（Trust Guard）进行形态与内容校验，避免错误传播。

- **SciFlow（记忆驱动的科研生命周期执行器）**
  - 将完整科研流程分解为五个阶段：`Literature` → `Ideation` → `Experiment` → `Writing` → `Rebuttal`，每一阶段实现为基于框架的技能合约。
  - 引入**框架保障**：提供跨阶段的持久状态、按技能定制的上下文视图、验证检查点、将失败/批评转化为过程信号的反馈机制，以及支持断点续跑的编排控制，使长期研究可中断、可审查、可复用。

- **SciDAG（基于有向无环图的多智能体增强）**
  - 将困难技能调用表示为有向无环图（DAG）形式的多智能体算子图。算子包括 `generate`、`variation`、`debate`、`test`、`refine`、`review`、`polish`、`early-stop` 等9种复用算子。
  - 图中通过**条件边**实现自适应控制：根据中间质量、成本或收敛信号决定继续、重试、分支、剪枝或终止。
  - 算子图以**阶段模板**形式复用（如创意思考模板强调多样性生成与辩论，实验模板强调可靠性检查，写作模板强调证据保真度），模板根据历史执行经验不断演化。

- **SciEvolve（完整系统进化）**
  - 将来自用户、任务和环境的三类反馈信号转换为对系统本身的版本化更新，对应三个技能：
    - `/dream`：维护长期记忆的有用性（降权陈旧实体、压缩冗余、提出新关联）。
    - `/forge`：将科学技能视为版本化协议，依据反复出现的失败模式、用户纠正、审稿警示等提出补丁（如增强核实步骤、修订交付要求）。
    - `/morph`：利用 SciDAG 的执行轨迹改进算子模板（修改提示词、角色或工具配置，剪枝弱分支、增加验证节点）。

## 摘要
大语言模型在智能体记忆流水线中的位置——介于检索已存事实的召回平面（已被广泛基准测试）和通过覆盖、释放、清除操作来变更事实的控制平面（几乎未经测试）之间——决定了系统能恢复何种遗忘失败模式。在包含385个案例的对抗表面上比较十三种系统配置，我们观察到三个具有部分互补覆盖的放置区域：确定性原语足以处理词汇/时间类别但在规范化上失败（标识符混淆为5%，跨语言为0%）；写入时的大语言模型恢复了规范化（100%）但无法处理意图感知删除（前缀冲突和复合事实为0%）；变异时钩子恢复了意图感知删除（78-85%）并几乎同时提亮所有类别（总体91.7-93.2%，每次385案例运行成本0.17美元，变异延迟2.3秒/案例 vs. 确定性方法64-191毫秒/案例，召回路径不变）。我们通过ForgetEval揭示了这种权衡，它包含一个1000案例模板化套件加上一个385案例对抗层（132个人工编写+253个大语言模型起草并经过预言验证），通过确定性子串匹配评分，并配有一个六方法适配器协议，采用诚实的N/A评分，使得异构记忆存储在130行代码内即可接入。评估的有效性得到10名标注者间一致性（Fleiss' kappa = 0.958）和77个外部编写案例子集（四位盲审贡献者）的证实，后者复现了规范化不对称性并放大了联合放置的提升（+27.8个百分点）。生产环境中的失败主要是遗忘失败而非召回失败，然而现有的基准测试仅衡量召回。ForgetEval和所有适配器均以MIT许可发布。

## Abstract
Where an LLM sits in an agent memory pipeline -- between the recall plane that retrieves stored facts (extensively benchmarked) and the control plane that mutates them via supersede, release, purge (largely untested) -- shapes which forgetting failure modes the system recovers. Comparing thirteen system configurations on a 385-case adversarial surface, we observe three placement regimes with partly complementary coverage: deterministic primitives suffice for lexical/temporal categories but fail canonicalization (5% on identifier-obfuscation, 0% on cross-lingual); inscribe-time LLM recovers canonicalization (100%) but cannot help intent-aware deletion (0% on prefix-collision and compound-fact); a mutation-time hook recovers intent-aware deletion (78-85%) and brightens nearly all categories simultaneously (91.7-93.2% overall, $0.17 per 385-case run, 2.3s/case mutation latency vs. 64-191ms/case deterministic, recall path unchanged).   We expose the trade-off via ForgetEval, a 1000-case templated suite plus a 385-case adversarial layer (132 hand-crafted + 253 LLM-drafted oracle-validated) scored by deterministic substring match, paired with a six-method Adapter Protocol with honest N/A scoring that lets heterogeneous memory stores enter in 130 lines. Admission is corroborated by 10-annotator IAA (Fleiss' kappa = 0.958) and a 77-case external-authored subset (four blind contributors) that replicates the canonicalization asymmetry and amplifies the joint-placement lift (+27.8 pt). Production failures are predominantly forgetting failures rather than recall failures, yet existing benchmarks measure only recall. ForgetEval and all adapters are released under MIT.

### 1. 论文的核心问题与整体含义
- **核心问题**：现有智能体记忆系统几乎只关注“召回”（recall）——即能否正确检索到存储的事实，但忽略了“遗忘”（forgetting）这一关键控制动作。生产环境中，绝大部分故障源于遗忘失败（如已删除的GDPR数据仍被检索、旧密码依旧被建议等），然而当前所有基准测试（如LongMemEval、BEIR）仅衡量召回性能，完全缺乏对控制平面遗忘能力的评估。
- **整体含义**：本文旨在填补这一空白，通过系统研究大语言模型（LLM）在记忆流水线中的**放置位置**（即在哪个阶段介入——写入时、变异时还是不介入）如何决定系统能恢复哪些遗忘失败模式。研究提出，控制平面的设计（而非单纯有无LLM）是解决遗忘故障的关键。

### 2. 论文提出的方法论
- **核心思想**：将智能体记忆抽象为两个平面：**召回平面**（检索已存事实）和**控制平面**（通过覆盖、释放、清除等操作变更事实）。LLM可以放在三个位置：不介入（纯确定性）、写入时介入（如抽取标签）、变异时介入（如规划超覆盖或匹配）。不同的放置会产生不同的遗忘覆盖能力，且它们部分互补。
- **关键技术细节**：
  - **ForgetEval基准**：包含1000个模板案例（覆盖5种遗忘家族：超覆盖、衰减、遗忘、清除、漂移）以及一个385案例的对抗层（132人工编写+253由LLM起草并经预言验证），涵盖10类攻击（如子串陷阱、前缀冲突、跨语言标识符等）。评分采用确定性子串匹配（top-10召回中必须包含和不包含的字符串），无需LLM裁判。
  - **适配器协议**：定义6个方法的控制平面代数，包括3个强制召回原语和3个可选控制原语（supersede, release, purge），对不支持的原语返回NotImplementedError并评分为N/A。该协议在约130行代码内即可让异构记忆系统接入评估。
  - **系统对比**：提出了一个参考系统（基于SQLite+sqlite-vec+FTS5），并集成了多种第三方系统（MEM 0, LangGraph, MEM PALACE等），通过统一的适配器和基准进行公平比较。
  - **算法流程**：变异时钩子使用窄JSON契约（supersede planner, purge match, release match），仅需极少量LLM调用，不影响召回热路径。

### 3. 实验设计
- **数据集**：ForgetEval基准，包括1000个模板案例（英文，种子42，4个干扰项）和385个对抗案例（覆盖10个攻击类别），以及外部77案例盲审子集。外部实验还用到Memora、FactConsolidation等基准进行交叉验证。
- **对比方法**：共13种系统配置，分为6类：无删除原语（MEM PALACE）、纯确定性（LETHE、LangGraph InMemoryStore、MEM 0 infer=False）、纯向量（OpenMemory）、写入时LLM（A-MEM、MEM 0+v3）、知识图谱抽象（Graphiti、HippoRAG）、变异时LLM钩子（LETHE+LLM、LangGraph+LLM）以及联合放置（Letta+LLM）。同时在不同LLM家族（DeepSeek-V3、Qwen-2.5-72B、Llama-3.1-70B）上进行了消融。
- **评估指标**：总体通过率（必须在top-10检索中包含所有must_contain且不含must_not_contain），95% Wilson置信区间，延迟，成本。

### 4. 资源与算力
- **硬件**：所有主流适配器实验均在单CPU上运行（无GPU），嵌入使用all-MiniLM-L6-v2（384维，ONNX运行时）。无API调用（LLM钩子的对比实验中除外），召回路径保持无互联网访问。
- **LLM钩子成本与延迟**：使用DeepSeek-V3时，385个案例的成本约$0.17，每次变异延迟约2.3秒/案例（对比确定性方法64–191毫秒/案例）。Qwen-2.5-72B成本更高（$0.42/M输入），Llama因JSON解析失败回退到确定性基线。
- **训练**：论文未涉及任何模型训练，所有方法均是直接使用预训练模型或纯算法，因此无训练算力开销。

### 5. 实验充分性
- **实验数量**：进行了大量实验，包括：1）385案例对抗套件上13种配置的完整对比；2）模板套件（1000案例）上4种主要系统的准确率；3）多种嵌入消融（英语与多语言嵌入）；4）跨LLM钩子消融（DeepSeek、Qwen、Llama）；5）跨架构钩子消融（LETHE vs. LangGraph）；6）10位标注者的多标注一致性（Fleiss' κ=0.958）；7）3个不同法官（Qwen、DeepSeek、Claude）与标注者一致性对比；8）外部盲审子集（77案例）验证；9）FactConsolidation和Memora交叉验证；10）多种第三方系统（A-MEM、Graphiti、HippoRAG等）的适配评估；11）子串评分器可靠性审计；12）真实世界失败模式映射等。
- **充分性与公平性**：实验覆盖面广，从不同放置、不同后端、不同LLM家族、不同基准交叉验证到人工标注和外部盲审，设计严谨。所有系统使用统一适配器协议和确定性子串评分，减少了主观性。论文也明确指出了大语言模型起草案例可能带来的循环论证风险，并通过人工编写核心集、外部盲审子集以及多个法官交叉验证予以缓解。

### 6. 主要结论与发现
- **三种放置机制部分互补**：
  - **确定性原语**足以处理词汇/时间类别（>95%），但在规范化上失败（如标识符混淆仅5%，跨语言0%）。
  - **写入时LLM**可完全恢复规范化（100%），但无法协助意图感知删除（前缀冲突和复合事实均为0%）。
  - **变异时钩子**能恢复意图感知删除（78-85%），并同时提升几乎所有类别，整体准确率达91.7-93.2%（$0.17/次），延迟仅增加变异时间，召回路径不变。
- **联合放置提升显著**：外部77案例子集上，联合放置（标语+钩子）提升27.8个百分点，证实互补性。
- **架构无关性**：变异时钩子的提升效果在不同后端（LETHE、LangGraph、Letta）和不同LLM家族之间具有一致性，提升由JSON契约驱动，而非特定存储后端。
- **基准双轴翻转**：MEM PALACE在召回轴上得分60/150，但在遗忘轴上得分为0/385，说明现有召回基准无法预测遗忘控制能力。
- **真实世界故障模式映射**：基准的类别设计（如前缀冲突、标识符混淆、复合事实）直接对应GDPR泄露、凭证残留、矛盾信息等生产故障。

### 7. 优点
- **填补关键空白**：首次系统研究记忆控制平面的遗忘能力评估，将关注点从“召回”转向“遗忘”。
- **基准设计全面**：ForgetEval从遗忘家族分解、对抗攻击类别、确定性子串评分到适配器协议，形成了一个可扩展、公平的评估体系。
- **实验严谨**：大规模系统对比、多样本消融、人工标注验证、外部盲审、多法官交叉验证，确保了结论的可靠性。
- **实用性强**：钩子方法成本低廉、延迟可接受，且不影响召回路径，适合生产环境集成；开源发布基准和适配器，方便社区拓展。
- **洞察深刻**：揭示了控制平面放置的互补规律，并给出了架构选择与权衡（如召回–遗忘权衡图）。

### 8. 不足与局限
- **英文主导**：基准主要针对英文，跨语言类别虽包含多脚本但样本量少（如泰文仅1个案例），多语言覆盖不足。
- **子串评分器盲点**：约30%的确定性失败是评分器伪影（如合法ID是禁止ID的子串），可能导致对确定性系统低估，但对LLM钩子的提升并无虚高。
- **大语言模型起草案例的循环论证风险**：虽然通过人工核心集和外部盲审缓解，但253个LLM起草案例与钩子LLM同家族可能引入部分偏见。
- **外部盲审子集尚小**：77个案例覆盖了10类但部分类别数量过少，不能完全替代大规模独立测试。
- **大语言模型质量敏感性**：钩子提升依赖于LLM的JSON遵循能力，较弱模型可能无效果或退化。
- **记忆系统覆盖**：仅评估了部分开源系统（Graphiti、Cognee等因API不兼容或版本限制未能全面对比），商业系统未纳入。
- **真实世界实证缺乏**：仅做了结构映射，未在真实部署中验证系统遗忘绩效与基准得分的相关性。

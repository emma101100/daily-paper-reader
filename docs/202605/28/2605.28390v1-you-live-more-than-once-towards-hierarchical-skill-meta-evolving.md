## 摘要
测试时技能演化被视为增强已部署智能体系统的新范式。现有工作主要关注硬编码的技能演化策略或依赖底层大语言模型昂贵参数更新的参数化学习。在本文中，我们证明了针对不同下游场景，对技能演化框架本身进行测试时优化对于智能体系统的持续改进是必要的，并且轻量级的算法自适应是可行的。具体而言，我们提出了HiSME，一种轻量级的分层技能元演化解决方案，通过从智能体的任务执行轨迹中学习元技能，联合优化技能和技能演化策略。在多种智能体基准上的实验表明，元演化能产生比纯技能演化更高质量的技能库，并能为不同场景衍生出多样的元技能，从而促进未来的持续经验学习。我们的代码暂时公开于 https://anonymous.4open.science/r/HiSME-BD45。

## Abstract
Test-time skill evolving is regarded as a new paradigm for enhancing deployed agentic systems. Existing works mainly focus on hard-coded skill evolving strategies or parametric learning that rely on expensive parameter updates in the underlying LLMs. In this paper, we demonstrate that test-time refinement of the skill evolving framework itself is necessary for continuous improvement of the agent systems in different downstream scenarios, and lightweight algorithmic adaptation is feasible. Specifically, we propose HiSME, a lightweight hierarchical skill meta-evolving solution that jointly optimizes skills and the skill evolving strategy by learning meta-skills from agents' task execution traces. Experiments on diverse agentic benchmarks show that meta-evolving can produce a higher-quality skill library than pure skill evolving and can derive diverse meta-skills for different scenarios, thereby facilitating future continual experience learning. Our code is temporarily public at https://anonymous.4open.science/r/HiSME-BD45.

### 2. 方法论：HiSME 框架

*   **核心思想**：将技能演化建模为一个**多层级的残差优化问题**。
    *   **第 0 阶**：智能体执行器（Executor）本身是一个有缺陷的先验。
    *   **第 1 阶**：技能演化算法（EVO）作为后验优化器，通过学习“技能”来修正执行器的缺陷，优化执行效用。
    *   **第 2 阶**：技能演化算法本身也并非完美，HiSME 的核心思想是引入“元技能”（meta-skills），作为后验优化器来**修正技能演化算法自身的缺陷**，从而优化技能学习过程的效用。这是一种二阶优化。
*   **关键技术细节与流程**：
    1.  **技能生成 (Skill Generation)**：
        *   **提取器 (Extractor)**：从单条执行轨迹中识别可复用的片段或经验。
        *   **重构器 (Refactorer)**：通过构建轨迹片段的语义相似度图并提取团（clique），从多条相关轨迹或技能中抽象出共享结构。最终生成三类技能：可调用函数、工作流框架和领域知识。
    2.  **技能评估与维护 (Skill Evaluation and Maintenance)**：
        *   **评估**：
            *   **短期效用**：为大语言模型中的每个技能维护一个“测试套装”（bundle），通过单元测试和集成测试验证其契约。
            *   **长期效用**：使用“信用分配器”（credit assigner）判断技能在真实任务中是有益、有害还是中性，并记录使用统计。
        *   **维护**：
            *   **精炼器 (Refiner)**：当证据表明技能的范围、接口或实现可改进时，对其进行修改，并通过测试套装验证后发布新版本。
            *   **过滤器 (Filter)**：当一个技能的有害证据超过阈值且缺乏足够的正面证据时，将其禁用。
    3.  **元技能生成 (Meta-skill Generation)**：
        *   **元评估**：为每个大语言模型驱动的算法角色（提取、重构、精炼）维护一个**经验回放缓冲区（replay buffer）**，存储其产生的技能（或技能组）及该技能的后续效用证据。
        *   **元演化**：定期从回放缓冲区采样经验，调用大语言模型代理（META EVO）总结出经验法则，形成“元技能”。这些元技能被注入到对应角色的上下文中，作为“习得的后验知识”来指导未来的技能生成和维护。元技能数量保持精简（最多5条）。

## 摘要
现有的记忆增强型LLM智能体通常将记忆视为具有预定义表示和固定检索管道的静态存储库，这在动态的智能体环境中是脆弱的，因为反馈、任务变化和异构信号不断重塑应该记住什么以及如何连接。为了解决这个问题，我们提出了FluxMem，一个连接演化的记忆框架，将记忆建模为异构图，并通过三个阶段逐步细化其拓扑结构：初始连接形成、反馈驱动的细化和长期巩固。在执行过程中，FluxMem修复缺失的链接，修剪干扰，对齐抽象粒度，并将反复出现的成功轨迹提炼为可重用的程序电路，由记忆泛化性和进化成熟度这一指标来指导。在包括LoCoMo、Mind2Web和GAIA在内的三个截然不同的基准测试中，FluxMem持续实现了最先进的性能，展示了在复杂智能体环境中的强大适应性和泛化能力。代码将开源在https://github.com/zjunlp/LightMem。

## Abstract
Existing memory-augmented LLM agents often treat memory as a static repository with pre-defined representations and fixed retrieval pipelines, which is brittle in dynamic agentic environments where feedback, task variation, and heterogeneous signals continuously reshape what should be remembered and how it should be connected. To address this, we propose FluxMem, a connectivity-evolving memory framework that models memory as a heterogeneous graph and progressively refines its topology through three stages: initial connection formation, feedback-driven refinement, and long-term consolidation. During execution, FluxMem repairs missing links, prunes interference, aligns abstraction granularity, and distills recurrent successful trajectories into reusable procedural circuits, guided by one metric for memory generalizability and evolutionary maturity. Across three fundamentally distinct benchmarks including LoCoMo, Mind2Web, and GAIA, FluxMem achieves consistent state-of-the-art performance, demonstrating strong adaptation and generalization in complex agentic environments. The code will be open-sourced in https://github.com/zjunlp/LightMem.

## 1. 论文的核心问题与整体含义

- **核心问题**：现有记忆增强的LLM智能体大多将记忆视为静态仓库，使用预定义的表示和固定的检索流水线。在动态、长程、任务多变的智能体环境中，反馈、任务变化和异构信号会持续重塑“该记住什么”以及“记忆如何被连接”，静态记忆设计会带来两类关键失败：
  - **自适应连接失败**：固定连接方式导致检索时“欠连接”（丢失关键上下文）或“过连接”（引入噪声与幻觉），且记忆单元内容以单一抽象层次存储，无法灵活调整粒度。
  - **连接巩固失败**：缺乏将成功经验逐渐固化为稳定程序性结构的机制，导致智能体不断重复构建相似关联，无法让记忆网络自组织优化。

- **整体含义**：论文提出**将记忆视为持续演化的连接结构**，而非静态存储。通过模拟人脑的突触可塑性和系统巩固过程，让记忆单元及其连接在交互中动态生长、修剪与重组，最终提高智能体在复杂任务上的适应性与泛化能力。

## 2. 方法论：核心思想与技术细节

- **核心思想**：将记忆抽象为一个可动态编辑的**异构图**，包含三层节点：
  - **语义层** \( V_{\text{sem}} \)：存储静态事实知识（文档、工具API等）。
  - **情节层** \( V_{\text{epi}} \)：记录具体任务的状态-动作轨迹。
  - **程序层** \( V_{\text{proc}} \)：封装从成功情节中提炼出的可复用推理模板或技能。
  边表示层间关联（grounding边从语义到情节，distill边从情书到程序）。智能体的工作上下文被形式化为当前任务激活的**子图**，通过拓扑编辑来优化上下文内容。

- **三阶段记忆演化流程**：
  1. **阶段一：初始连接形成**
     - 根据当前观察 \( o_t \) 进行混合检索（稠密向量相似度、BM25稀疏匹配、LLM检验），从语义层召回Top-k事实节点。
     - 通过嵌入相似度从情节层召回最相关的k个历史情节。
     - 沿已有distill边从召回的情情节继承对应的程序技能节点。
     - 组合以上节点构成初始子图，序列化后作为初始上下文 \( S_t(q) \)。

  2. **阶段二：反馈驱动的连接精炼**
     - 执行任务后根据环境反馈或自检信号 \( f_t \) 分析失败原因，进行拓扑编辑：
       - **连接扩展**：若缺失关键信息，建立新边，连接未被激活但语义相关的节点。
       - **连接修剪**：若上下文拥挤或产生幻觉，切断干扰边。
       - **单元内容重塑**：当检索充分但粒度不匹配时，对记忆单元的内部内容进行细化或抽象，调整粒度。
     - 更新子图并产生精炼后的上下文 \( S'_t(q) \)，循环直至执行成功或达到最大轮次 \( T \)。

  3. **阶段三：长期连接巩固**
     - 完成后将轨迹存入情节节点。离线阶段，根据轨迹嵌入相似度对情节进行聚类。
     - 对每个簇用LLM归纳出共性的技能或推理模式，生成新的程序节点 \( v_{\text{proc}}^{(m)} \)。
     - 引入**程序进化成熟度评分 (PEMS)**，迭代测-打-修循环：重新运行源情节测试技能效果，计算成功率 \( \eta^{(k)} \)、技能文本长度 \( \ell^{(k)} \)、当前与历史版本的嵌入差异 \( \delta^{(k)} \) 综合评分。LLM重写低分技能，直至评分改进低于阈值 \( \epsilon \)，技能被认为稳定且有用。

- **公式**：上下文完全由激活节点和边决定，如 \( S_t(q) = \text{Concat}(q, Obs_t, V_{\text{sem}}^t, V_{\text{epi}}^t, V_{\text{proc}}^t) \)。

## 摘要
编程智能体在解决软件工程任务时会生成丰富的执行轨迹。为了实现智能体的自演化，这些轨迹可以被提炼为可复用的过程性技能，这些技能以紧凑的方式编码经验，以指导未来行为。然而，现有的技能构建和维护方法通常依赖于固定的提示和启发式更新规则，尚不清楚应如何选择、抽象和维护知识才能最好地服务下游智能体。我们提出 CODESKILL，这是一个基于大语言模型的框架，将技能提取和技能库维护重构为一种可学习的管理策略。CODESKILL 从编程智能体的执行轨迹中提取多粒度的过程性技能，利用新经验演化技能，并维护一个紧凑的技能库以用于未来任务求解。我们使用强化学习训练 CODESKILL，采用混合奖励，结合基于评分细则的密集技能质量反馈与来自冻结的下游智能体的稀疏可验证执行反馈。在 EnvBench、SWE-Bench Verified 和 Terminal-Bench 2 上的实验表明，CODESKILL 相比无技能基线平均通过率提升 9.69，相比最强基于提示或记忆的基线提升 4.01，同时在迭代构建过程中保持技能库大小稳定。

## Abstract
Coding agents produce rich trajectories while solving software-engineering tasks. To enable agent self-evolution, these trajectories can be distilled into reusable procedural skills that compactly encode experience to guide future behavior. However, existing skill construction and maintenance methods often rely on fixed prompts and heuristic update rules, leaving it unclear how knowledge should be selected, abstracted, and maintained to best serve downstream agents. We propose CODESKILL, an LLM-based framework that reformulates skill extraction and skill-bank maintenance as a learnable management policy. CODESKILL extracts multi-granularity procedural skills from coding-agent trajectories, evolves skills with new experience, and maintains a compact skill bank for future task solving. We train CODESKILL with reinforcement learning, using a hybrid reward that combines dense rubric-based skill-quality feedback with sparse verifiable execution feedback from the frozen downstream agent. Experiments on EnvBench, SWE-Bench Verified, and Terminal-Bench 2 show that CODESKILL improves average pass rate by 9.69 over the no-skill baseline and by 4.01 over the strongest prompt-based or memory baseline, while maintaining the skill bank at a stable size during iterative construction.

### 2. 论文提出的方法论
- **核心思想**：将技能管理（抽取、演化、维护）统一建模为策略 \( M_\theta \)，以冻结的下游编码策略 \(\pi\) 的任务成功率为优化目标，使用强化学习训练该策略。
- **关键技术细节**：
    - **多粒度技能库**：
        - **任务级技能**：面向任务或任务族的高层流程策略（如仓库检查、定位、修复验证）。
        - **事件驱动技能**：针对局部执行事件（命令失败、错误信息）的触发-响应规则。
    - **技能管理操作**：
        - **提取**：从单条或几条相关轨迹中生成任务级或事件驱动技能。
        - **演化**：根据新轨迹（尤其是失败轨迹）修订已有技能的触发条件或规则。
        - **维护**：对候选技能决定**添加**（add）、**合并**（merge）或**丢弃**（drop），以保持技能库紧凑。
    - **训练流程**：
        1. **有监督预热**：使用教师模型（GPT-5.4-mini）从轨迹与已有技能上下文中生成结构化操作目标，微调基座模型（Qwen3.5-4B）。
        2. **强化学习优化（GRPO）**：采用混合奖励。
            - **质量奖励 \(R_Q\)**：基于评分细则的 LLM-as-judge 打分（根基性、可复用性、具体性等），密集监督。
            - **执行奖励 \(R_E\)**：对于生成的有效技能，反向检索评估任务 \(x_u\)，计算技能注入后下游智能体成绩相比无技能基线的增量。
            - **对齐奖励 \(R_A\)**：判断下游智能体的实际行为是否遵循了技能指引，用于改善信用分配。
            - 最终奖励：\(R = \lambda R_Q + R_A \cdot R_E\)。
        3. **三阶段课程**：先训练提取，再加入演化，最后加入维护，使技能生命周期数据可复用。
- **公式/算法流程**：核心是最大化更新的技能库 \(B'\) 在未来任务 \(x'\) 上的表现，并通过 GRPO 更新 \(M_\theta\)（详见论文 Algorithm 1）。

### 6. 论文的主要结论与发现
- CODESKILL 在三大基准上**平均通过率相对无技能基线提升 9.69**（约33%相对提升），**相对最强基线提升 4.01**（约11%相对提升）。
- 学习的技能管理策略能**跨任务分布（包括未见的 Terminal-Bench 2）和跨下游智能体模型（如 GPT-5.4-mini）泛化**。

- 强化学习使技能管理质量（密集奖励）和下游效用（稀疏执行奖励）同步改善，且技能库在面对新任务类型时能自适应新增，饱和后趋于稳定。

### 7. 优点
- **将启发式维护转为可学习策略**：首次将基于下游反馈的强化学习引入技能库管理，解决了固定规则难以适应下游需求的问题。
- **混合奖励设计精巧**：结合密集的规则评分（缓解稀疏执行反馈）与对齐系数（改善信用分配），使策略有效收敛。
- **跨任务、跨模型泛化验证**：不仅在训练域内效果好，也在全新的 Terminal-Bench 2 和不同下游智能体上有效，证明技能管理行为的可迁移性。
- **生命周期消融透彻**：清晰地展示了提取、演化、维护各阶段对性能与技能库规模的影响，强调维护对控制冗余的关键作用。

### 8. 不足与局限
- **技能表示形式受限**：仅支持自然语言指令式技能，无法封装可执行脚本、API 等更丰富的结构化资源，限制了技能的表达力和直接工具扩展能力。
- **操作空间单步限制**：管理策略一次只能输出一种操作作用于一个候选技能，无法表达联合修订多个相关技能、拆分过泛技能等复杂决策。
- **实验覆盖**：仅在三种基于 bash 命令交互的软件工程基准上验证，未探索更复杂或不规则的工具使用环境（如多工具编排、GUI 操作）。
- **有监督预热依赖强教师模型**：SFT 数据由 GPT-5.4-mini 生成，可能存在教师偏差，且该方法在完全无教师信号下的可行性未被检验。
- **奖励稀疏与长时延**：尽管混合奖励缓解了部分问题，但执行奖励仍依赖长程下游 rollout，环境反馈噪声大，可能限制策略在更大规模或更稀疏任务上的学习效率。

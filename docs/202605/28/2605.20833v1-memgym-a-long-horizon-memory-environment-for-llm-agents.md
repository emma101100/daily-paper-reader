## 摘要
记忆是LLM智能体执行长周期任务的核心能力。现有的记忆基准测试主要评估多轮对话场景中个性化信息的保留，忽视了智能体长时间执行过程中发生的动态记忆形成。因此，这些基准产生的记忆系统在编码和网页导航等实际智能体环境中的迁移效果较差。我们提出MemGym，一个用于智能体记忆的基准测试，它将现有的智能体训练场和内部基于记忆的管道统一在一个记忆-推理接口之后。MemGym包含五个评估轨道，分为四个智能体领域：工具使用对话（tau2-bench）、多轮深度研究搜索（MEMGYM-DR）、编码（SWE-Gym 和 MEMGYM-CODEQA）以及计算机使用（WebArena-Infinity）。MemGym报告记忆隔离得分，将记忆性能与推理、检索和工具使用能力解耦，因此可以在没有这些混杂因素的情况下对记忆策略进行排名。我们为MEMGYM-CODEQA和MEMGYM-DR构建的合成管道长度可控，每个阶段都经过消融验证，并与下游场景紧密对齐。为了使编码环境上的评估在学术上易于处理，我们训练了MemRM，一个轻量级的奖励模型（使用QLoRA微调的Qwen3-1.7B），它将压缩质量评分为一个快速的标量读数，以替代完整的Docker部署。

## Abstract
Memory is a central capability for LLM agents operating across long-horizon tasks. Existing memory benchmarks predominantly evaluate retention of personalized information in multi-turn chat scenarios, overlooking the dynamic memory formation that occurs during extended agent execution. Consequently, the memory systems they produce transfer poorly to realistic agentic environments, such as coding and web navigation. We present MemGym, a benchmark for agentic memory that unifies existing agent gyms and in-house memory-grounded pipelines behind one memory-reasoning interface. MemGym spans five evaluation tracks grouped into four agentic regimes: tool-use dialogue (tau2-bench), multi-turn deep-research search (MEMGYM-DR), coding (SWE-Gym and MEMGYM-CODEQA), and computer use (WebArena-Infinity). MemGym reports memory-isolated scores that decouple memory performance from reasoning, retrieval, and tool-use ability, so memory strategies can be ranked without those confounders. Our synthetic pipelines for MEMGYM-CODEQA and MEMGYM-DR are length-controllable, ablation-verified at every stage, and tightly aligned with downstream scenarios. To make evaluation on coding environments academically tractable, we train MemRM, a lightweight reward model (Qwen3-1.7B fine-tuned with QLoRA) that scores compression quality as a fast scalar read in place of full Docker rollouts.

### 2. 方法论：核心思想与关键技术细节

MemGym的核心思想是**分离**与**统一**，其关键技术体现在以下几个层面：

*   **核心思想：记忆-推理分离的评估范式**
    *   MemGym在所有环境中，将一个**内存模块（Memory Module）**与执行任务的**推理模型（Reasoning Model）** 显式分开。
    *   评估时采用 **“配对运行”（Paired Runs）** ：对同一任务，固定推理模型，分别在使用特定记忆策略（`With-Memory`）和不使用记忆策略（`No-Memory`）的场景下运行。两者的得分差值（`memory gain`）即为**记忆隔离得分**，纯粹反映了该记忆策略的贡献，排除了推理、工具使用等能力的干扰。

*   **关键技术细节**
    *   **统一记忆基础设施**：为五个不同的评估轨道提供了一个共享的接口契约（`BaseMemoryEnvironment`, `BaseAgent`, `BaseMemoryManager`），定义了标准的单步执行循环：`env.reset() → memory_manager.manage_context → agent.act → env.step`。这使得任何记忆策略都能以即插即用的方式在不同环境中测试。
    *   **可控的合成记忆测试轨道**：
        *   **MemGym-CodeQA**：基于SWE-smith中的Bug和补丁，提取“仅靠记忆”才能回答的事实，并通过注入四种干扰（如主题推断、模式匹配等）来构建问答对，长度（Token预算）可控。
        *   **MemGym-DR（深度研究）**：通过迭代搜索构建“桥梁事实链”，测试智能体在多跳检索中对中间信息的保持能力。其关键在于**虚构化（Fictionalization）**，将所有实体（名称、数字等）替换为虚构内容，以防止模型通过预训练知识回答，从而保证测试的是真实的记忆能力。
    *   **轻量级评估信号：MemRM奖励模型**
        *   为解决完整Docker部署评估成本高的问题，论文训练了一个名为**MemRM**的轻量级二元分类器（Qwen3-1.7B + QLoRA）。
        *   **训练数据**：通过重放记录的轨迹并进行六类“反事实扰动”（如随机删除摘要句子、注入错误信息等），自动生成大量`SAFE`（安全压缩，任务仍可解）和`HARMFUL`（有害压缩，导致任务失败）标签对。
        *   **功能**：MemRM输入“压缩前上下文-压缩后上下文-候选动作”三元组，能在亚秒级时间内预测一个压缩操作是否`SAFE`，从而替代耗时的完整Docker执行过程，作为策略迭代和强化学习后训练的信号。其损失函数为类别加权的交叉熵，权重基于类别平衡计算并设有上限。

## 摘要
尽管人工智能智能体在推理和工具使用方面展现出卓越能力，但它们本质上是反应式的：仅在用户明确提示后才计算响应。这种范式忽视了一个关键机会：交互之间的空闲时间大部分被浪费，使得智能体无法为未来的用户需求做准备。为弥补这一差距，我们引入了ProAct，一种主动型智能体架构，它利用空闲时间计算来预判并满足可能出现的未来用户需求。通过分析不断演进的对话历史以及持久记忆，ProAct预测即将到来的需求并迭代获取信息，使智能体能够在用户发起查询之前填补知识空白并准备证据。为严格评估主动能力，我们还推出了ProActEval，一个涵盖40个领域、包含200个场景的综合基准，其中包含可预测的需求链和多样化的用户认知画像。实证结果表明，与反应式基线相比，ProAct具有显著优势：在ProActEval上，ProAct通过减少14.8%的对话轮次加速了任务完成，降低了11.7%的用户精力，并将幻觉率削减了28.1%。此外，MemBench评估证实ProAct达到了最先进的反思准确性，凸显了其持续且稳健的性能。

## Abstract
While AI agents demonstrate remarkable capabilities in reasoning and tool use, they remain fundamentally reactive: they compute responses only after explicit user prompts. This paradigm ignores a critical opportunity: the idle time between interactions is largely wasted, leaving agents unable to prepare for future user needs. To bridge this gap, we introduce ProAct, a proactive agent architecture that leverages idle-time compute to anticipate and fulfill likely upcoming user needs. By analyzing evolving dialogue history together with persistent memory, ProAct predicts upcoming needs and iteratively acquires information, allowing the agent to resolve knowledge gaps and prepare evidence before the user initiates a query.To rigorously evaluate proactive capabilities, we also introduce ProActEval, a comprehensive benchmark comprising 200 scenarios across 40 domains, featuring predictable need chains and diverse user cognitive profiles. Empirical results demonstrate significant advantages over reactive baselines. ProAct accelerates task completion by reducing required turns by 14.8%, decreases user effort by 11.7%, and cuts hallucination rates by 28.1% on ProActEval. Furthermore, MemBench evaluations confirm that ProAct achieves state-of-the-art reflective accuracy, underscoring its sustained and robust performance.

### 2. 论文提出的方法论

论文提出了名为 **ProAct** 的主动型智能体架构，其核心是一个在空闲时间运行的“预判-获取”闭环，由两大紧密耦合的模块驱动。

*   **核心思想**：
    *   **将空闲计算力前置**：将原本集中在用户请求时刻的计算压力转移到非高峰时段，用于预测和准备。
    *   **记忆驱动的预判**：利用持久记忆中的用户画像、事实知识、对话摘要等，结合当前对话历史，预测用户未来可能的信息需求。
    *   **价值感知的投递**：并非将所有准备好的信息都推送给用户，而是通过一个价值门控机制，仅在信息的预期效用大于其打断用户的成本时才进行主动通知。

*   **关键技术细节与流程**：
    *   **形式化定义**：将主动交互形式化为一个闭环决策问题。引入`Zt = fpred(Ht, Mt)`，其中`Ht`是对话历史，`Mt`是持久记忆，`Zt`是预测的未来需求候选集。每个候选`z`包含需求、理由、置信度和检索计划。系统通过优化一个综合未来效用、打断成本、计算成本和幻觉风险的策略来运作。
    *   **未来状态预测 (Future-State Prediction)**：实例化`fpred`函数。它并非无边界探索，而是从两个来源生成紧凑的候选需求集：1) **局部场景预测**，根据最近的对话推断近期的后续需求；2) **关联扩展**，根据记忆中的用户画像、持久任务和未解决目标提出相关主题。此外，记忆维护模块识别到的陈旧或不完整的知识缺口也会被转化为候选需求。该模块最终会过滤掉低置信度的候选项，并进行去重和优先级排序。
    *   **空闲时间获取与交付 (Idle-Time Acquisition and Delivery)**：实现策略中的获取和交付部分。
        1.  **价值评估**：对每个候选需求`z`计算一个价值分数`S(z)`，该分数由用户相关性、知识缺口、增量价值和时效性四个维度加权（默认权重各25%）得出。仅对分数超过阈值的候选项进行后续操作。
        2.  **感知记忆的获取**：对于接受的候选项，先检查记忆是否已包含足够证据，以决定是直接复用、增量搜索还是完全重新检索。这实现了增量式的研究，避免重复工作。
        3.  **效用感知的交付**：为生成的每份准备材料（Artifact）选择交付模式：`push`（立即推送）、`queue`（排队，稍后集成到回复中）或`store`（静默存入记忆）。该决策通过比较信息的预期价值和打断成本来实现。

### 6. 论文的主要结论与发现

*   **主动计算显著优于被动反应**：在ProActEval上，完整的ProAct系统对比反应式基线，实现了交互轮次减少14.8%，用户负担降低11.7%，同时幻觉率大幅下降28.1%。

*   **存在成本-效率权衡**：搜索预算分析表明，增加空闲时间的搜索量（k值）虽能提升预判召回率，但收益是递减的，并且与最终的对话效率和用户精力减少并非单调线性关系。主动计算是一个需要权衡的“操作点”，而非简单的最大化。
*   **记忆是主动行为的基石**：ProAct的记忆系统在MemBench上取得了最高的反思准确率（10k tokens下84.3%，100k tokens下86.3%），证明其稳固的记忆能力为成功的主动预判和准备提供了坚实支撑。

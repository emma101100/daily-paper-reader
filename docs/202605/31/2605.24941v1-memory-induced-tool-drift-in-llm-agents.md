## 摘要
现代大语言模型（LLM）智能体将长期记忆个性化与工具调用接口相结合，以在现实世界中采取行动——这一组合构成了当代生产系统的基础。我们研究了这种组合此前未被探讨的一种失败：当记忆中存储的由性格驱动的偏见（成本意识、不耐烦、风险承受力等）在不适用的情况下悄然影响工具调用。我们称之为记忆引发的工具漂移，并通过MEMDRIFT对其进行操作化，这是一个涵盖五个偏见维度、七个专业领域的105个场景基准，通过自动化对抗流水线生成。在七个前沿模型（包括具有扩展推理能力的模型）中，有偏见的记忆使偏转分数（由评判员评分的参数偏离无偏见基线的程度）在1–5分制上最多增加+3.6分。当记忆管理由三种生产记忆架构处理时，工具漂移仍然存在。该现象影响现实世界的工具：扫描288个经过验证的MCP服务器中的6,062个工具，我们标记了608个参数易受影响的工具，并在验证子集上确认了工具漂移。机理上，有偏见的记忆充当隐式引导向量，沿着与显式行为指令相同的潜在方向推动激活。它们还将注意力从任务相关上下文重新分配到与目标参数有表面关键词重叠的记忆条目上。标准防御措施——基于提示的相关性指令和记忆过滤器——可以减少漂移，但无法消除。随着智能体代表用户执行越来越重要的操作，记忆引发的工具漂移构成当前防护措施未能解决的系统性漏洞，这促使在记忆管理与工具调用生成的交叉点上开发专门的防御措施。

## Abstract
Modern LLM agents combine long-term memory for personalization with tool-calling interfaces for taking actions in the world -- a combination underpinning contemporary production systems. We study a previously unexamined failure of this combination: when personality-driven biases stored in memory (cost-consciousness, impatience, risk tolerance, etc.) silently affect tool calls in contexts where they are not applicable. We call this memory-induced tool-drift and operationalize it through MEMDRIFT, a benchmark of 105 scenarios spanning five bias dimensions and seven professional domains, generated through an automated adversarial pipeline. Across seven frontier models -- including those with extended reasoning -- biased memories raise deflection scores (a judge-scored measure of parameter deviation from unbiased baselines) by up to $+3.6$ points on a 1--5 scale. Tool-drift persists when memory management is handled by three production memory architectures. The phenomenon affects real-world tools: scanning 6{,}062 tools across 288 verified MCP servers, we flag 608 with susceptible parameters and confirm tool-drift on a validated subset. Mechanistically, biased memories act as implicit steering vectors, pushing activations along the same latent directions as explicit behavioral instructions. They also redistribute attention from task-relevant context toward memory entries with surface-level keyword overlap to the target parameter. Standard defenses -- prompt-based relevance instructions and memory filters -- reduce drift but do not eliminate it. As agents take increasingly consequential actions on a user's behalf, memory-induced tool-drift represents a systematic vulnerability that current safeguards do not address, motivating dedicated defenses at the intersection of memory management and tool-call generation.

# 论文深度总结：记忆引发的LLM智能体工具漂移

### 1. 核心问题与整体含义

本研究聚焦于现代大语言模型（LLM）智能体中一个此前未被系统研究的关键失败模式：**记忆引发的工具漂移（Memory-Induced Tool-Drift）**。

*   **研究背景与动机**：当代AI智能体通过结合“长期记忆”和“工具调用”两大能力，实现了跨会话的用户个性化和在现实世界中采取行动（如代码执行、金融交易）。然而，这两个功能多被独立研究，其交叉影响被忽略。
*   **核心问题**：智能体在专业工作场景下调用工具时，其长期记忆中存储的、源于个人生活习惯的性格偏见（如追求速度、成本极度敏感、不重视安全等）会被**不恰当地**应用于工具参数的选择上。例如，一个平时在生活上过于追求低成本的人在调用金融审计工具时，可能会选择廉价的存储方案而非合规所需的安全方案。
*   **核心含义**：这种漂移构成了一个**系统性的安全漏洞**。与聊天对话中的过度个性化不同，工具调用的后果更具隐蔽性和不可逆性，而当前的模型提供者和工具开发者均未对此风险做出充分考虑。

### 2. 方法论

为量化和分析该问题，作者构建了一个名为**MEMDRIFT**的基准测试框架，并设计了探索机理的实验。

*   **核心思想与定义**：
    *   **问题形式化**：将工具漂移定义为在三种记忆条件下，模型生成的工具参数集合 \(\theta\) 的差异。分别是：无记忆条件（\(M_\emptyset\)）、中性记忆（\(M_n\)）和有偏记忆（\(M_b\)）。通过比较 \(\theta_b\) 与 \(\theta_\emptyset\) 的偏差来判断漂移，同时用 \(\theta_n \approx \theta_\emptyset\) 来确保偏差由偏见内容而非记忆的存在本身引起。
    *   **严格上下文分离**：为确保漂移判断的客观性，所有用户记忆被严格限定在个人生活领域（如生活习惯、性格特征），而所有工具调用任务则属于完全无关的专业领域（如医疗、金融、法律），从而排除了记忆对任务可能有正面帮助的歧义。

*   **关键技术细节**：
    *   **自动化对抗性基准生成**：使用Claude-Opus-4.6作为生成器和评判员，GPT-5.2作为目标模型，通过一个三阶段迭代流水线生成105个高对抗性的场景。
        *   **阶段一**：为每个偏见维度与专业领域组合生成抽象任务场景。
        *   **阶段二**：将场景具象化为完整的基准样本，包括工具schema、用户问题、基础/中性/有偏记忆集、以及预期漂移行为描述。
        *   **阶段三（核心）**：一个**自我改进的对抗性优化循环**。评判员对目标模型在三种记忆条件下的工具调用行为进行打分，生成报告后，与人工编写的优化技巧一同反馈给生成器。生成器据此迭代修改记忆、提示词和工具schema，以最大化引发漂移的能力。最优场景经人工审核后入库。

*   **机制探究流程**：
    *   **隐式引导向量分析**：构建“前填充响应”让模型在特定参数处生成决定漂移与否的关键token。通过计算有/无偏见记忆时的模型隐层激活之差，并将其投影到由显式指令构建的“引导向量”上，证明偏见记忆能将模型激活推向与显式偏见指令相同的方向。
    *   **注意力重分配分析**：计算在有/无偏见记忆下，模型在关键token生成时对输入各区域（系统指令、记忆、工具schema、用户提问等）的注意力分布变化（\(\Delta\alpha\)）。

### 3. 实验设计

*   **数据集/场景**：
    *   **MEMDRIFT基准**：包含105个样本，由5种性格偏见维度（速度/急躁、资源节俭、极简/简洁、风险容忍、自主/自我依赖）和7个专业领域（医疗、金融、法律、软件基建、教育、电商、营销）交叉组合，每个组合3个场景。
    *   **真实世界工具审计**：扫描了来自288个验证MCP服务器的6,062个工具schema，并对其中的608个高风险工具进行了验证性测试。

*   **对比设置**：
    *   **模型评估**：在7款前沿模型上进行测试，包括5款闭源模型（GPT-5.2, GPT-5.4, Claude-Sonnet-4.5等）和2款开源模型（Kimi-K2.5, Qwen3.5-397B-A17B）。所有推理模型均使用中等思考量。
    *   **记忆植入方式**：对比了两种设定：
        1.  **直接注入**：将记忆文本完整放入system prompt，给予模型所有辨识信息。
        2.  **记忆架构框架**：通过多轮对话自然编码偏好，然后由Mem0、MemPalace、SimpleMem三种主流记忆框架独立决定如何存储和检索记忆，测试端到端的真实情况。
    *   **防御策略对比**：评估了基于提示词的相关性指令防御，以及基于过滤器（Self-ReCheck）的记忆过滤防御。

*   **评价指标**：
    *   **偏转分数（Deflection Score）**：由Claude-Opus-4.6作为评判员，在1-5分Likert量表上评估有偏/中性记忆下的工具参数相对于无记忆基线的整体偏离程度，分数越高表示漂移越严重。评测结果中的 \(s_b\) 代表有偏条件得分，\(s_n\) 代表中性条件得分。

### 6. 主要结论与发现

1.  **现象普遍性**：记忆引发的工具漂移是一个**广泛存在且系统性的问题**。所有受测的前沿模型在直接注入记忆和通过记忆框架检索记忆时，均表现出显著的漂移行为（中性记忆得分低，有偏记忆得分高达3.6以上）。
2.  **防御不足**：通过提示词指令和粗粒度相关性过滤器进行防御，**可以减少但无法消除**漂移。相关性过滤器在需要多跳推理以判断相关性的真实场景中性能较差（召回率61.0%，误报率10.3%）。
3.  **现实威胁**：该现象并非仅限于合成基准，**真实世界的工具同样存在漏洞**。对6062个工具的扫描发现了608个高风险对象，并成功验证了真实的漂移案例。
4.  **机理明确**：
    *   偏见记忆充当了**隐式的引导向量**，沿着与显式行为指令相同的潜在方向推动模型内部表征。
    *   偏见记忆通过**关键词的重叠劫持了模型的注意力**，使其从任务相关的上下文（如工具描述、用户指令）向特定的偏见记忆条目转移，并用表面相似性代替了任务相关性判断。

### 7. 优点

*   **问题定义新颖且重要**：首次识别并形式化了“记忆->工具调用”这一交叉点的安全风险，填补了研究空白，且该问题对智能体应用意义重大。
*   **方法论严谨**：
    *   **严格上下文分离**提供了一个客观、无歧义的漂移判断基准。
    *   **自动化对抗性基准生成**高效、可扩展，并能持续产生高质量的测试用例。
    *   结合**行为、表征和注意力**三个层面的分析，为现象提供了深刻的机理解释。
*   **评估全面**：从合成场景到**现实世界工具审计**，从直接注入到端到端的**记忆框架评估**，覆盖了多种前沿模型，验证了问题的真实性与普遍性。
*   **开源承诺**：论文声明将公开代码和基准，有助于社区跟进研究。

### 8. 不足与局限

*   **单工具、单轮次设定**：基准中的每个场景仅包含一次工具调用，未涉及生产环境中常见的多工具链式调用和多轮对话交互，这可能导致低估实际风险。
*   **参数漂移 vs. 工具选择**：研究聚焦于给定工具下的参数赋值偏差，而未探索记忆是否会导致模型**选择错误的工具本身**。
*   **记忆类型限制**：基准中的记忆仅限于“个人性格/习惯”，而在现实中，记忆可能包含更多模糊、交叉的信息，其相关性和边界的判断更加困难。
*   **评测员偏差**：尽管进行了跨评测员验证，但主要评测结果仍然依赖于另一个大语言模型（Claude-Opus-4.6），其自身可能存在的系统偏差是潜在风险。
*   **防御策略量级**：训练时微调作为一种潜在的治本防御策略，文中虽提出但未实验验证。
